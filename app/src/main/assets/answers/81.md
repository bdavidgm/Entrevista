## Concepto
CoroutineScope define el ámbito y ciclo de vida de las [coroutines](pregunta:78) hijas: agrupa jobs y aplica structured concurrency (cancelación en cascada). En Android, scopes típicos son [viewModelScope](pregunta:131), [lifecycleScope](pregunta:130) y applicationScope.

## Explicación detallada
Un scope = [CoroutineContext](pregunta:30) (Job + Dispatcher + opcional name). Al cancelar el Job del scope, todas las [coroutines](pregunta:78) hijas reciben cancelación cooperativa. [viewModelScope](pregunta:131) usa [SupervisorJob](pregunta:143) en implementación que permite un hijo fallar sin cancelar hermanos según configuración.

[lifecycleScope](pregunta:130) en [Activity](pregunta:31)/[Fragment](pregunta:32) cancela al destruir DESTROYED. repeatOnLifecycle(Lifecycle.State.STARTED) para collect [Flow](pregunta:84) solo cuando visible.

No crear scopes globales sin Job vinculado a lifecycle. CoroutineScope([Dispatchers.Main](pregunta:87) + job) manual requiere cancelar en onDestroy.

Testing: TestScope y runTest controlan tiempo virtual. backgroundScope en runTest para trabajo async.

En el contexto de proyectos Android reales, combina esta idea con las guías oficiales de Android Developers y con métricas de calidad (crashes, [ANR](pregunta:95), Vitals) cuando aplique. En code review, pregunta siempre por ciclo de vida, testabilidad y si la solución escala cuando el equipo crece.

## Cuándo usarlo
Siempre que uses launch/async; nunca GlobalScope en features de app. Elige scope según duración del trabajo: pantalla vs app vs test.

Ejemplo:
class MyViewModel : ViewModel() {
    fun save() = viewModelScope.launch {
        repository.save()
    }
}

// Activity
lifecycleScope.launch {
    repeatOnLifecycle(Lifecycle.State.STARTED) {
        viewModel.uiState.collect { render(it) }
    }
}

class MyViewModel : [ViewModel](pregunta:68)() { fun save() = [viewModelScope](pregunta:131).launch { [repository](pregunta:63).save() } } usa expresión = con launch: save() devuelve Job implícito del launch (Unit en practice si no se captura). [viewModelScope](pregunta:131).launch { [repository](pregunta:63).save() } ejecuta save [suspend](pregunta:79) en scope del [ViewModel](pregunta:68). [lifecycleScope](pregunta:130).launch { repeatOnLifecycle(Lifecycle.State.STARTED) { [viewModel](pregunta:68).uiState.collect { render(it) } } } anida launch en [Activity](pregunta:31)/[Fragment](pregunta:32) con repeatOnLifecycle [suspend](pregunta:79) que reinicia collect cuando lifecycle >= STARTED. collect { render(it) } es terminal [Flow](pregunta:84) operator con lambda it del uiState emitido. Contrasta scope de [ViewModel](pregunta:68) (sobrevive rotación) con [lifecycleScope](pregunta:130) (ligado a [LifecycleOwner](pregunta:44)). render(it) method call en lambda.

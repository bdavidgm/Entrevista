Concepto:
rememberSaveable es como remember pero, además de sobrevivir a recomposiciones, guarda automáticamente su valor en el savedInstanceState del componente. Así el estado sobrevive a cambios de configuración y a process death de la UI cuando el sistema mata el proceso.

Funciona out-of-the-box con tipos serializables (primitivos, String, Parcelable). Para tipos personalizados se puede proporcionar un Saver custom.

Ejemplo:
@Composable
fun Counter() {
    var count by rememberSaveable { mutableStateOf(0) }
    Button(onClick = { count++ }) { Text("Clicks: $count") }
}

// Tipo custom con Saver
val saver = listSaver<MyState, Any>(
    save = { listOf(it.id, it.name) },
    restore = { MyState(it[0] as String, it[1] as String) }
)

Consejo:
Limita rememberSaveable a UI pequeña y serializable. Para estado grande o de dominio, mejor el ViewModel con StateFlow.

## Concepto
La interpolación de strings permite insertar variables o expresiones dentro de un literal usando $variable o ${expresion}. Es más legible y eficiente que concatenar con +, y se traduce a llamadas a StringBuilder en JVM.

Las expresiones complejas requieren llaves; los $ literales se escapan con \$.

Ejemplo:
val name = "Ana"
val items = listOf(1, 2, 3)

val greeting = "Hola, $name"
val summary = "Tienes ${items.size} elementos, total ${items.sum()}"
val price = "Precio: \$${9.99}"

ExplicaciónKotlin:
Las propiedades `name` e `items` se declaran con `val` e inferencia de tipos. `"Hola, $name"` interpola una variable simple: el compilador la traduce a concatenación eficiente vía `StringBuilder`. `"Tienes ${items.size} elementos, total ${items.sum()}"` usa `${...}` para expresiones compuestas: acceso a propiedad `.size`, llamada a función de extensión `sum()` sobre la lista, todo evaluado en runtime al construir la cadena. `"Precio: \$${9.99}"` escapa el símbolo `$` con `\$` para mostrarlo literalmente, mientras `${9.99}` interpola el literal numérico (tipo `Double`). Las plantillas son expresiones, no solo sintaxis de impresión, y respetan la nullabilidad: si `name` fuera `String?`, la interpolación seguiría funcionando convirtiéndolo a `String`.

Consejo:
Compara con concatenación; recuerda usar llaves para expresiones (${user.name}) y \$ para mostrar el símbolo literal.

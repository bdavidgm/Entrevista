Concepto:
inline indica al compilador que copie el cuerpo de la función en el lugar de la llamada en vez de generar una invocación normal. En funciones de orden superior evita crear un objeto Function por cada lambda, lo que ahorra allocations en bucles y APIs frecuentes.

Su uso típico es en utilidades pequeñas con lambdas (forEach, scope functions) y es obligatorio para usar reified, que conserva el tipo genérico en runtime. crossinline y noinline modulan el comportamiento de las lambdas que recibe.

Ejemplo:
inline fun <T> measureMs(block: () -> T): Pair<T, Long> {
    val start = System.currentTimeMillis()
    val result = block()
    return result to (System.currentTimeMillis() - start)
}

inline fun <reified T> Bundle.getOrNull(key: String): T? =
    get(key) as? T

Consejo:
No marques inline funciones grandes: aumentan el bytecode. Reified solo funciona en inline. Aprovecha para hablar de crossinline cuando se necesita evitar non-local returns.

## Concepto
SupervisorJob es un Job cuyos hijos fallan de forma aislada: si un hijo lanza excepción y no se captura, los demás siguen ejecutándose. Un Job normal cancela a todos los hermanos ante un fallo no controlado.

Se usa en el contexto raíz de scopes de larga vida ([Application](pregunta:128), [ViewModel](pregunta:68) con varias tareas independientes).

Ejemplo:
val scope = CoroutineScope(SupervisorJob() + Dispatchers.Main)

scope.launch { throw RuntimeException("A") } // no cancela B
scope.launch { delay(100); println("B sigue") }


ExplicaciónKotlin:
`CoroutineScope(SupervisorJob() + Dispatchers.Main)` construye scope con contexto combinado mediante operador `+` de `CoroutineContext`: `SupervisorJob()` es job raíz que no cancela hermanos ante fallo, y `Dispatchers.Main` fija hilo principal. `scope.launch { throw RuntimeException("A") }` inicia [corrutina](pregunta:78) hija; la excepción no cancelada puede propagarse según [handler](pregunta:159). El segundo `scope.launch { delay(100); println("B sigue") }` muestra que `delay` es [suspend](pregunta:79) y libera el hilo; `println` es función estándar de Kotlin. Dos llamadas independientes a `launch` crean jobs hermanos bajo el mismo supervisor. Sin `SupervisorJob`, un fallo en A cancelaría B. El ejemplo ilustra composición declarativa de contexto de [corrutinas](pregunta:78) en pocas líneas.

Consejo:
Combínalo con CoroutineExceptionHandler en el contexto raíz para loguear errores sin tumbar el resto. [viewModelScope](pregunta:131) ya usa SupervisorJob internamente.

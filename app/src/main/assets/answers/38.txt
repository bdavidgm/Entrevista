Concepto:
Un ContentProvider es un componente que expone datos estructurados a otras apps mediante una URI (content://) y un conjunto de operaciones tipo CRUD. Es el mecanismo estándar de Android para compartir datos entre aplicaciones (contactos, fotos, calendario).

Dentro de la propia app rara vez se usa directamente: para datos internos suele bastar con Room o DataStore. Se sigue usando como cliente vía ContentResolver para acceder a MediaStore, ContactsContract, etc.

Ejemplo:
val cursor = contentResolver.query(
    ContactsContract.Contacts.CONTENT_URI,
    arrayOf(ContactsContract.Contacts.DISPLAY_NAME),
    null, null, null
)
cursor?.use {
    while (it.moveToNext()) {
        Log.d(TAG, it.getString(0))
    }
}

ExplicaciónKotlin:
`contentResolver.query(...)` invoca la API de ContentProvider del framework; los argumentos `null` son parámetros opcionales de selección/orden que aquí no se usan. `arrayOf(...)` crea un arreglo Kotlin de proyección de tipos (`Array<String>`) para las columnas proyectadas. `cursor?.use { ... }` combina safe call con la función de extensión estándar `use`: si `query` devuelve null, no entra al bloque; si hay cursor, lo cierra automáticamente al salir (como try-with-resources). Dentro del lambda, `it` es el `Cursor`; `while (it.moveToNext())` itera filas. `it.getString(0)` accede a la columna por índice. `Log.d(TAG, ...)` registra cada nombre. Este patrón es idiomático en Kotlin para recursos que implementan `Closeable`, evitando fugas de cursores.

Consejo:
"Para compartir datos entre apps; dentro de la app, Room". Necesario para integrarse con APIs del sistema como MediaStore y SAF.

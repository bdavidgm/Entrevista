Concepto:
ContentResolver es la puerta de entrada para acceder a datos expuestos por ContentProviders: consultas, inserciones, actualizaciones y borrados mediante URIs content:// con cursores o contratos tipados.

Explicación detallada:
En Android, los datos de otras apps o del sistema (contactos, calendario, media, downloads) no se leen abriendo archivos ajenos directamente, sino a través de un ContentProvider que abstrae permisos y columnas. Tu app usa contentResolver.query(uri, projection, selection, selectionArgs, sortOrder) y obtiene un Cursor (o versión moderna con ContentResolver.query con Bundle en API 30+).

Operaciones CRUD vía insert, update, delete con ContentValues. openInputStream(uri) para leer bytes de un documento SAF o MediaStore.

Ejemplos de URIs conocidos:
- ContactsContract.Contacts.CONTENT_URI
- MediaStore.Images.Media.EXTERNAL_CONTENT_URI
- CalendarContract.Events.CONTENT_URI
- Settings.System (limitado)

Permisos: READ_CONTACTS, READ_EXTERNAL_STORAGE (legacy), o permisos granulares READ_MEDIA_IMAGES en Android 13+. SAF no requiere permiso de almacenamiento amplio: el usuario elige el archivo.

ContentResolver también resuelve sync adapters y notifica cambios con ContentObserver para UI reactiva.

Buenas prácticas: cerrar Cursor (o usar extensiones), projection mínima, selection parametrizada (evitar SQL injection en providers custom), mover queries pesadas fuera del main thread, usar ContentUris.withAppendedId.

Cuándo usarlo:
- Leer/escribir contactos, galería, calendario del sistema.
- Integrar con providers de tu propia app multi-módulo.
- Abrir streams de URIs devueltos por picker o cámara.

Ejemplo:
fun loadRecentImages(context: Context): List<Uri> {
    val resolver = context.contentResolver
    val collection = MediaStore.Images.Media.EXTERNAL_CONTENT_URI
    val projection = arrayOf(MediaStore.Images.Media._ID)
    val sort = "${MediaStore.Images.Media.DATE_ADDED} DESC"
  resolver.query(collection, projection, null, null, sort)?.use { cursor ->
        val idCol = cursor.getColumnIndexOrThrow(MediaStore.Images.Media._ID)
        return buildList {
            while (cursor.moveToNext()) {
                val id = cursor.getLong(idCol)
                add(ContentUris.withAppendedId(collection, id))
            }
        }
    }
    return emptyList()
}

Consejo para entrevista:
Explica el patrón Provider + Resolver + URI. Contrasta acceso directo a paths (obsoleto) con MediaStore y SAF. Menciona que Room no usa ContentResolver para tu BD privada; solo para datos compartidos o del sistema.

Concepto:
tailrec es un modificador que indica al compilador que una función recursiva debe optimizarse a un bucle iterativo si su última operación es la llamada recursiva. Evita el StackOverflowError en recursiones profundas.

Requiere que la recursión sea estrictamente en cola: nada de operaciones tras el return de la llamada recursiva, ni try/catch alrededor de ella.

Ejemplo:
tailrec fun factorial(n: Long, acc: Long = 1): Long =
    if (n <= 1) acc else factorial(n - 1, acc * n)

tailrec fun findRoot(view: View): View =
    if (view.parent !is View) view
    else findRoot(view.parent as View)

ExplicaciónKotlin:
`tailrec fun factorial(n: Long, acc: Long = 1): Long` marca la función para optimización de recursión en cola: el compilador la transforma en un bucle iterativo si la última operación es la llamada recursiva. `acc: Long = 1` es un parámetro con valor por defecto que acumula el resultado (estilo fold). La rama `if (n <= 1) acc else factorial(n - 1, acc * n)` devuelve directamente la recursión sin operaciones posteriores, cumpliendo la restricción tailrec. `tailrec fun findRoot(view: View)` recorre la jerarquía: `view.parent !is View` usa smart cast implícito en la negación; en el else, `view.parent as View` es un cast clásico (unsafe) porque el compilador no puede promover el tipo tras el `!is`. Si la recursión no fuera en cola, el compilador emitiría un warning y no optimizaría.

Consejo:
Si el compilador no puede optimizar avisa con warning. Útil para recorridos lineales sin riesgo de stack overflow; en lógica de negocio compleja, valora un loop explícito.

Concepto:
tailrec (recursión de cola) es un modificador de Kotlin que indica al compilador que una función recursiva puede optimizarse a un bucle iterativo si la llamada recursiva es la última operación de la función (tail call). Evita StackOverflowError en recursión profunda y es idiomático para algoritmos lineales.

Explicación detallada:
El compilador reescribe la recursión en un while bajo el capó en JVM. Debe ser exactamente tail call: no puedes hacer return n * factorial(n-1) con tailrec sin acumulador. Patrón: llevar estado en parámetros por defecto o función interna.

Ejemplo clásico: factorial y fibonacci educativos; en Android real, recorrer árboles de categorías, paginación recursiva de APIs con acumulador, o parsers. Muchas veces en producción Android prefieres while/for explícito por legibilidad del equipo.

Si la recursión no es en cola, el compilador error en compile time al marcar tailrec. Kotlin no garantiza tail call optimization en lambdas o funciones no tailrec.

Comparación con bucle while: mismo rendimiento; tailrec expresa intento recursivo claro. En entrevistas piden reescribir factorial con acumulador tailrec.

Cuándo usarlo:
Algoritmos naturalmente recursivos con profundidad potencial grande y tail call clara. Si la lógica no es tail, no fuerces tailrec.

Ejemplo:
tailrec fun factorial(n: Long, acc: Long = 1): Long {
    return when {
        n <= 1 -> acc
        else -> factorial(n - 1, acc * n)
    }
}

tailrec fun findCategoryPath(
    node: Category,
    targetId: String,
    path: List<Category> = emptyList()
): List<Category>? {
  if (node.id == targetId) return path + node
  for (child in node.children) {
    val found = findCategoryPath(child, targetId, path + node)
    if (found != null) return found
  }
  return null
}

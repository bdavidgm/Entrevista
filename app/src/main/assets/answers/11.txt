Concepto:
when es la expresión de control de flujo de Kotlin equivalente a un switch enriquecido. Puede usarse como sentencia o como expresión (que devuelve valor) y soporta condiciones arbitrarias, rangos, tipos (is) y deconstrucción.

Si se usa como expresión sobre un tipo sealed o enum, el compilador exige exhaustividad y avisa al añadir una variante nueva, lo que ayuda a mantener el código alineado con el dominio.

Ejemplo:
fun describe(value: Any): String = when (value) {
    is Int -> "Entero $value"
    in 1..9 -> "Dígito"
    "hola" -> "Saludo"
    null -> "Vacío"
    else -> "Otro"
}

ExplicaciónKotlin:
`fun describe(value: Any): String = when (value) { ... }` declara una función cuyo cuerpo es una expresión `when` que devuelve `String` (no hace falta `return` explícito). Cada rama es una condición: `is Int` comprueba el tipo y hace smart cast de `value` a `Int` dentro de la rama, permitiendo interpolar `$value` en la plantilla. `in 1..9` usa un range inclusivo para comprobar pertenencia. `"hola" ->` compara igualdad estructural con un literal. `null ->` maneja explícitamente el caso anulable de `Any?` implícito cuando el argumento puede ser null (aunque aquí el parámetro es `Any`, la rama null es válida si el tipo fuera `Any?`). `else ->` cubre el resto cuando `when` no es exhaustivo sobre un tipo abierto como `Any`. El smart cast tras `is` evita casts manuales dentro de cada rama.

Consejo:
Subraya when como expresión exhaustiva con sealed/enum (sin else). Compáralo con switch de Java y menciona el smart cast tras is.

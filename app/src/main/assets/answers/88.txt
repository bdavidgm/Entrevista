Concepto:
JUnit es el framework estándar de pruebas unitarias en JVM y Android (JUnit 4 y JUnit 5 en proyectos Kotlin). Proporciona anotaciones @Test, assertions, reglas y organización de suites para verificar comportamiento de forma automatizada.

Explicación detallada:
@Test marca métodos ejecutables por Gradle testDebugUnitTest. Assertions: assertEquals, assertTrue, assertThrows. @Before y @After preparan estado. Parameterized tests para muchos inputs.

En Android Kotlin, kotlinx-coroutines-test ofrece runTest en lugar de runBlocking en tests de suspend. InstantTaskExecutorRule legacy para LiveData; MainDispatcherRule para coroutines.

JUnit 5 (Jupiter) con Kotlin: @ExtendWith, tags, nested classes; adopción variable en Android por tooling.

Integración CI: fallo de test bloquea merge. Naming descriptivo: backticks `when X then Y`.

En el contexto de proyectos Android reales, combina esta idea con las guías oficiales de Android Developers y con métricas de calidad (crashes, ANR, Vitals) cuando aplique. En code review, pregunta siempre por ciclo de vida, testabilidad y si la solución escala cuando el equipo crece.

Durante la entrevista, estructura la respuesta en definición, ejemplo concreto de tu experiencia o proyecto demo, y trade-offs frente a alternativas. Mencionar versiones recientes de API o biblioteca (Material3, Paging 3, Hilt 2.x) demuestra que sigues el ecosistema, sin necesidad de memorizar números de versión exactos.

Cuándo usarlo:
Toda lógica testeable sin emulador. Combina con MockK, Turbine, Truth según necesidad.

Ejemplo:
class DiscountTest {
    @Test
    fun `applies ten percent`() {
        val result = calculateDiscount(100, 0.10)
        assertEquals(90.0, result, 0.001)
    }
}

Menciona runTest y reglas Android. Diferencia JUnit test vs androidTest source set.

Concepto:
Las extension properties son propiedades asociadas a un tipo receptor (val Tipo.nombre) declaradas fuera de la clase. Al igual que las extension functions, se compilan como getters/setters estáticos, por lo que no tienen backing field: solo definen cómo calcular o asignar el valor.

Sirven para exponer valores derivados con sintaxis de propiedad. No deben usarse para almacenar estado por instancia (no hay campo); para eso conviene un delegate dentro de la clase.

Ejemplo:
val View.isVisibleNow: Boolean
    get() = visibility == View.VISIBLE

var View.isVisibleNow: Boolean
    get() = visibility == View.VISIBLE
    set(value) { visibility = if (value) View.VISIBLE else View.GONE }

Consejo:
Respuesta clave: "no tienen campo, solo get/set". Si necesitas estado por instancia, usa un delegated property dentro de la clase, no una extension.

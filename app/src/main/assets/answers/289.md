## Concepto
TopAppBar es la barra superior de [Material Design](pregunta:105): muestra el título, un icono de navegación (atrás o menú) y acciones a la derecha. Se coloca en el slot topBar de un [Scaffold](pregunta:59).

Ejemplo:
Scaffold(
    topBar = {
        TopAppBar(
            title = { Text("Detalle") },
            navigationIcon = {
                IconButton(onClick = onBack) {
                    Icon(Icons.AutoMirrored.Filled.ArrowBack, contentDescription = "Volver")
                }
            },
            actions = {
                IconButton(onClick = onShare) {
                    Icon(Icons.Filled.Share, contentDescription = "Compartir")
                }
            }
        )
    }
) { innerPadding ->
    // contenido, aplicando innerPadding
}

ExplicaciónKotlin:
TopAppBar recibe slots [composable](pregunta:46): title, navigationIcon y actions (este último con scope RowScope, por eso puedes poner varios [IconButton](pregunta:283)). Va dentro de topBar del [Scaffold](pregunta:59), que a su vez entrega un PaddingValues (innerPadding) que debes aplicar al contenido para no quedar debajo de la barra. En Material3 requiere @OptIn(ExperimentalMaterial3Api::class).

Consejo:
Aplica siempre el innerPadding del Scaffold al contenido. Para que la barra reaccione al scroll usa un TopAppBarScrollBehavior.

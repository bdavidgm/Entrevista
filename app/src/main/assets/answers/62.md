## Concepto
MVVM (Model-View-[ViewModel](pregunta:68)) separa la app en tres capas: Model (datos y lógica de negocio), View (UI pasiva que observa estado y emite eventos) y [ViewModel](pregunta:68) (mantiene el estado de la pantalla y expone APIs para que la View actúe). En Android, [ViewModel](pregunta:68) sobrevive a cambios de configuración y suele exponer [StateFlow](pregunta:86)/[LiveData](pregunta:69).

Beneficios: testabilidad ([ViewModel](pregunta:68) puro y sin Android), separación clara de responsabilidades y un flujo unidireccional natural (state down, events up).

Ejemplo:
class HomeViewModel(repo: Repository) : ViewModel() {
    val state: StateFlow<HomeState> = repo.users()
        .map(HomeState::Success)
        .stateIn(viewModelScope, SharingStarted.WhileSubscribed(5_000), HomeState.Loading)
}

@Composable
fun HomeScreen(vm: HomeViewModel = hiltViewModel()) {
    val state by vm.state.collectAsStateWithLifecycle()
    HomeContent(state)
}

ExplicaciónKotlin:
HomeViewModel(repo: [Repository](pregunta:63)) : [ViewModel](pregunta:68)() recibe [Repository](pregunta:63) por constructor sin anotaciones DI en este ejemplo simplificado. repo.users().map(HomeState::Success).stateIn(...) replica el patrón [Flow](pregunta:84) → [StateFlow](pregunta:86) con method reference HomeState::Success. HomeScreen(vm: HomeViewModel = [hiltViewModel](pregunta:141)()) resuelve el [ViewModel](pregunta:68); [hiltViewModel](pregunta:141)() es extensión [composable](pregunta:46). val state by vm.state.[collectAsStateWithLifecycle](pregunta:134)() usa delegación para suscribirse con lifecycle awareness. HomeContent(state) renderiza según HomeState sellado o [data class](pregunta:3). MVVM aquí se expresa en Kotlin mediante propiedades val inmutables expuestas, funciones composables sin estado propio y property delegates (by, [hiltViewModel](pregunta:141), [collectAsStateWithLifecycle](pregunta:134)).

Consejo:
MVVM en Android suele combinarse con [Repository](pregunta:63), UDF y, opcionalmente, [Use Case](pregunta:66)s. Comparte conceptos con [MVI](pregunta:65): la diferencia clave es cómo se modela el "[intent](pregunta:33)".

Concepto:
SharedPreferences es un almacén clave-valor persistente basado en archivos XML, síncrono en su API antigua y típicamente envuelto en apply() para escritura asíncrona. Hoy Google recomienda migrar a DataStore (Preferences o Proto) por seguridad de hilos y API moderna con Flow.

Sigue apareciendo en código legado y en preferencias simples de SDK.

Ejemplo:
val prefs = getSharedPreferences("settings", MODE_PRIVATE)
prefs.edit { putBoolean("dark", true) }

ExplicaciónKotlin:
`getSharedPreferences("settings", MODE_PRIVATE)` es una función de extensión o miembro de `Context` que devuelve un `SharedPreferences`; `MODE_PRIVATE` es la constante entera `0` del companion de `Context`. La variable `prefs` se declara con `val` y tipo inferido. La llamada `prefs.edit { ... }` usa la función de extensión `edit` de Android KTX: recibe una lambda con receptor (`SharedPreferences.Editor.() -> Unit`), de modo que dentro del bloque `putBoolean` se invoca como si fuera método del editor sin nombrar `apply()` ni `commit()` explícitamente — KTX aplica los cambios al salir del lambda. `"dark"` es un literal `String` y `true` un literal booleano; la clave-valor queda tipada de forma estática. Este patrón sustituye el boilerplate Java de `editor.put...(); editor.apply();` por una expresión única y más legible, aprovechando las higher-order functions de Kotlin.

Consejo:
Evita leer prefs en hot paths del main thread con get* síncronos masivos. Para datos estructurados o observables, DataStore.

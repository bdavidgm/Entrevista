## Concepto
Una clase abstracta puede tener constructor, estado y miembros implementados o abstractos, pero solo se hereda de una clase. Una interfaz define un contrato, puede aportar implementaciones por defecto y una clase puede implementar varias; no almacena estado con backing field.

Ejemplo:
abstract class BaseRepository(protected val api: Api) {
    abstract suspend fun refresh()
}

interface Cacheable {
    fun clearCache()
    fun isCacheEnabled() = true
}

class UserRepository(api: Api) : BaseRepository(api), Cacheable {
    override suspend fun refresh() { /* ... */ }
    override fun clearCache() { /* ... */ }
}

ExplicaciónKotlin:
`abstract class BaseRepository(protected val api: Api)` tiene constructor con parámetro promovido a propiedad `protected val`, visible en subclases pero no fuera. `abstract suspend fun refresh()` declara función sin implementación que subclases deben proveer; `suspend` permite llamarla desde corrutinas. `interface Cacheable` define contrato adicional con `clearCache()` abstracto y `isCacheEnabled() = true` con implementación default en interfaz Kotlin. `class UserRepository(api: Api) : BaseRepository(api), Cacheable` hereda una clase e implementa interfaz; orden `: BaseRepository(api), Cacheable`. Debe implementar `refresh` y `clearCache`; puede omitir override de `isCacheEnabled` por default. Kotlin no permite herencia múltiple de clases, sí múltiples interfaces. `/* ... */` son comentarios de bloque placeholder. Este diseño mezcla estado compartido (`api`) en abstract class y capacidad opcional (`Cacheable`) en interfaz.

Consejo:
Usa interfaz para capacidades y desacoplamiento; clase abstracta cuando las implementaciones comparten estado o inicialización.

## Concepto
SAF (Storage Access Framework) permite al usuario conceder acceso a documentos o árboles mediante ACTION_OPEN_DOCUMENT, ACTION_CREATE_DOCUMENT o ACTION_OPEN_DOCUMENT_TREE; obtienes URIs persistentes con takePersistableUriPermission. Es la forma correcta de leer carpetas arbitrarias sin MANAGE_EXTERNAL_STORAGE.

Funciona con proveedores de documentos del sistema y de terceros.

Ejemplo:
val intent = Intent(Intent.ACTION_OPEN_DOCUMENT).apply {
    addCategory(Intent.CATEGORY_OPENABLE)
    type = "application/pdf"
}
startActivityForResult(intent, REQ_CODE)

ExplicaciónKotlin:
`Intent(Intent.ACTION_OPEN_DOCUMENT)` construye un intent explícito con la acción de documento del SAF; el constructor de Kotlin delega en la clase Java `Intent`. El bloque `.apply { ... }` configura el receptor: `addCategory(Intent.CATEGORY_OPENABLE)` restringe a documentos abribles y `type = "application/pdf"` usa la propiedad Kotlin generada para el setter Java `setType`. `val intent` fija la referencia inmutable al intent configurado. `startActivityForResult(intent, REQ_CODE)` es un método de `Activity` (deprecado en favor de Activity Result API, pero válido en el ejemplo) que lanza el selector del sistema; `REQ_CODE` se asume una constante `Int` del companion o del archivo. No hay lambdas ni corrutinas: el flujo es imperativo clásico de Activity. La sintaxis `apply` con asignaciones de propiedad es el patrón idiomático Kotlin para inicializar objetos Android verbosos en pocas líneas.

Consejo:
Persiste permisos con takePersistableUriPermission tras FLAG_GRANT_PERSISTABLE_URI_PERMISSION. No asumas rutas de archivo absolutas.

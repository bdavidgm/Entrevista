## Concepto
remember almacena un valor en la composition para que sobreviva a las recomposiciones, pero no a la recreación del Composable (cambio de configuración, navegación, process death). Sirve para cachear cálculos o conservar estado interno que no necesita persistencia.

remember { mutableStateOf(...) } es el patrón habitual para crear estado observable: el value cambia, Compose recompone los lectores. Si necesitas sobrevivir cambios de configuración usa rememberSaveable.

Ejemplo:
@Composable
fun ExpandableCard(title: String) {
    var expanded by remember { mutableStateOf(false) }
    Column(Modifier.clickable { expanded = !expanded }) {
        Text(title)
        if (expanded) Text("Detalle…")
    }
}

ExplicaciónKotlin:
ExpandableCard es una función @Composable con parámetro title: String. var expanded by remember { mutableStateOf(false) } crea estado booleano local con delegación: el compilador genera getter/setter que redirigen a mutableStateOf.value. Modifier.clickable { expanded = !expanded } encadena un modificador con lambda que invierte el booleano usando el operador lógico !. Column recibe un Modifier como primer parámetro posicional y un trailing lambda con el contenido. if (expanded) Text("Detalle…") es un condicional de Kotlin a nivel de expresión dentro del DSL composable: solo se compone Text cuando expanded es true. El string "Detalle…" usa el carácter Unicode de puntos suspensivos. No hay nullable types aquí, pero el patrón remember + mutableStateOf es la base del estado local en Compose con sintaxis idiomática by.

Consejo:
"remember = sobrevive recomposiciones; rememberSaveable = sobrevive cambios de configuración". Si la key cambia, remember reinicia su valor.

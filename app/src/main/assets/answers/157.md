## Concepto
Un módulo Gradle es una unidad compilable (app, library Android, Kotlin JVM) con su propio build.gradle.kts, código fuente y dependencias. Modularizar separa features, acelera builds incrementales y permite límites de API claros entre equipos.

Los módulos se declaran en settings.gradle.kts e incluyen :app, :core:network, :feature:login, etc.

Ejemplo:
// settings.gradle.kts
include(":app", ":core:model", ":feature:home")

// :app/build.gradle.kts
dependencies {
    implementation(project(":feature:home"))
}


ExplicaciónKotlin:
Los fragmentos Kotlin en Gradle usan sintaxis de archivo `.gradle.kts`: `include(":app", ":core:model", ":feature:home")` invoca función `include` con strings de rutas de módulo — DSL de settings. En `dependencies { implementation(project(":feature:home")) }`, el bloque `dependencies` es lambda con receptor; `implementation` es función infix-like del plugin Android. `project(":feature:home")` referencia dependencia de módulo local. Los comentarios `//` son línea única Kotlin. No hay clases ni funciones definidas por el desarrollador: es script Kotlin ejecutado por Gradle. La modularización se declara declarativamente; el compilador Kotlin de Gradle resuelve classpath entre módulos.

Consejo:
Menciona API vs implementation para ocultar dependencias transitivas. Los módulos dynamic feature reducen tamaño inicial del descargable.

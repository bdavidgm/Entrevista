Concepto:
Una value class (antes llamada inline class) envuelve un único valor sin generar una instancia en runtime cuando es posible: el compilador “inlinea” el valor primitivo, lo que evita la asignación de memoria y conserva la seguridad de tipos.

Sirve para tipar con fuerza primitivos que de otra forma quedarían como String o Int sueltos (UserId, Email, Money), reduciendo errores al pasar parámetros del mismo tipo en orden equivocado.

Ejemplo:
@JvmInline
value class UserId(val raw: String)

@JvmInline
value class Email(val raw: String) {
    init { require("@" in raw) }
}

fun load(id: UserId) { /* ... */ }
load(UserId("42"))

ExplicaciónKotlin:
`@JvmInline value class UserId(val raw: String)` declara una value class (inline class): en runtime, cuando es posible, el valor se representa directamente como `String` sin wrapper, pero en compile-time es un tipo distinto que evita confusiones. `@JvmInline` es necesaria para interoperabilidad JVM. `value class Email(val raw: String)` incluye un bloque `init { require("@" in raw) }` que valida en construcción; `"@" in raw` usa el operador `in` sobre String. `fun load(id: UserId)` fuerza que no pases un `String` crudo ni un `Email` por error. `load(UserId("42"))` requiere envolver el literal en el constructor de la value class. Las value classes no pueden tener múltiples propiedades almacenadas ni herencia de clases, pero pueden implementar interfaces.

Consejo:
"@JvmInline value class = primitivo tipado sin coste". Hay restricciones (no herencia, una sola propiedad val). Útil para evitar primitive obsession.

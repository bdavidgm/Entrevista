Concepto:
Una value class (antes llamada inline class) envuelve un único valor sin generar una instancia en runtime cuando es posible: el compilador “inlinea” el valor primitivo, lo que evita la asignación de memoria y conserva la seguridad de tipos.

Sirve para tipar con fuerza primitivos que de otra forma quedarían como String o Int sueltos (UserId, Email, Money), reduciendo errores al pasar parámetros del mismo tipo en orden equivocado.

Ejemplo:
@JvmInline
value class UserId(val raw: String)

@JvmInline
value class Email(val raw: String) {
    init { require("@" in raw) }
}

fun load(id: UserId) { /* ... */ }
load(UserId("42"))

Consejo:
"@JvmInline value class = primitivo tipado sin coste". Hay restricciones (no herencia, una sola propiedad val). Útil para evitar primitive obsession.

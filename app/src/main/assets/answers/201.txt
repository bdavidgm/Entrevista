Concepto:
DiffUtil calcula la diferencia mínima entre dos listas y emite actualizaciones granulares (insert/remove/move/change) para RecyclerView, evitando notifyDataSetChanged() y animaciones incorrectas.

Explicación detallada:
Algoritmo de Eugene Myers: encuentra el patch más pequeño entre lista antigua y nueva. Requiere implementar:
- areItemsTheSame(old, new): identidad estable (id).
- areContentsTheSame(old, new): igualdad de contenido visible.
- Opcional getChangePayload: actualización parcial en bind.

Uso manual:
val diff = DiffUtil.calculateDiff(object : DiffUtil.Callback() {
    override fun getOldListSize() = old.size
    override fun getNewListSize() = new.size
    override fun areItemsTheSame(o, n) = old[o].id == new[n].id
    override fun areContentsTheSame(o, n) = old[o] == new[n]
})
diff.dispatchUpdatesTo(adapter)

ListAdapter (RecyclerView) ejecuta DiffUtil en background thread automáticamente al llamar submitList(newList).

Beneficios: mejor rendimiento en listas grandes, animaciones por item, preservación de scroll si aplica, menos parpadeo.

Cuándo usarlo:
- Siempre que la lista cambie por datos de red o base de datos.
- Chat, feeds, catálogos con actualizaciones parciales.

Evitar: comparaciones costosas en areContentsTheSame (usar campos relevantes).

Ejemplo:
val callback = object : DiffUtil.ItemCallback<Message>() {
    override fun areItemsTheSame(a: Message, b: Message) = a.id == b.id
    override fun areContentsTheSame(a: Message, b: Message) =
        a.text == b.text && a.timestamp == b.timestamp
}
val adapter = object : ListAdapter<Message, MsgVH>(callback) { ... }
adapter.submitList(newMessages)

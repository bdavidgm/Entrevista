## Concepto
Koin es un framework de DI en Kotlin puro: declara módulos con un DSL (`single`, `factory`, `viewModel`) y resuelve dependencias en runtime, sin procesador de anotaciones.

Ejemplo:
val appModule = module {
    single {
        Room.databaseBuilder(androidContext(), AppDatabase::class.java, "app.db").build()
    }
    single { get<AppDatabase>().noteDao() }
    single { NoteRepository(get()) }
    viewModel { NotesViewModel(get()) }
}

class MyApp : Application() {
    override fun onCreate() {
        super.onCreate()
        startKoin {
            androidContext(this@MyApp)
            modules(appModule)
        }
    }
}

class NotesScreen : ComponentActivity() {
    private val vm: NotesViewModel by viewModel()
}

ExplicaciónKotlin:
`module { ... }` es un DSL: `single { }` registra singleton; `get()` resuelve por tipo. `androidContext()` entrega el [Context](pregunta:127) de la app. `viewModel { }` integra con el ciclo de vida Android. `by viewModel()` es una [delegated property](pregunta:24) que obtiene el [ViewModel](pregunta:68). Comparación con [Hilt](pregunta:74) en [Koin vs Hilt](pregunta:214).

Consejo:
Ideal para prototipos y KMP; en apps grandes muchos equipos prefieren [Hilt](pregunta:74) por errores en compile-time.

Concepto:
remember es una función de Compose que almacena un valor en el Composition y lo conserva entre recomposiciones del mismo Composable en la misma posición del árbol. Sin remember, valores calculados o state se recrearían en cada recomposition perdiendo estado.

Explicación detallada:
remember { mutableStateOf(0) } crea estado una vez. remember(key1, key2) { } invalida y recalcula cuando cambian keys (útil al cambiar userId). remember con objetos costosos evita realloc (parser, AnnotatedString).

No sobrevive process death ni rotación fuera de rememberSaveable. rememberCoroutineScope recuerda scope para callbacks UI. Contraste: ViewModel para lógica de pantalla y datos de negocio; remember para estado UI local efímero.

En entrevistas: remember vs rememberSaveable; qué pasa si el Composable sale del árbol (se olvida). Side effects en remember block: permitidos una vez al crear.

Cuándo usarlo:
Estado UI local, objetos derivados costosos, AnimationState, SnackbarHostState. Keys cuando dependes de parámetros de identidad.

Ejemplo:
@Composable
fun UserForm(userId: String) {
    val repository = remember(userId) { UserRepository(userId) }
    var name by rememberSaveable { mutableStateOf("") }
    val scope = rememberCoroutineScope()
    Button(onClick = {
        scope.launch { repository.save(name) }
    }) { Text("Guardar") }
}

Consejo para entrevista:
remember sin keys vs con keys. No uses remember para datos de red que deben vivir en ViewModel. rememberSaveable para sobrevivir rotación.

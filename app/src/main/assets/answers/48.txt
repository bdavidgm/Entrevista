Concepto:
remember almacena un valor en la composition para que sobreviva a las recomposiciones, pero no a la recreación del Composable (cambio de configuración, navegación, process death). Sirve para cachear cálculos o conservar estado interno que no necesita persistencia.

remember { mutableStateOf(...) } es el patrón habitual para crear estado observable: el value cambia, Compose recompone los lectores. Si necesitas sobrevivir cambios de configuración usa rememberSaveable.

Ejemplo:
@Composable
fun ExpandableCard(title: String) {
    var expanded by remember { mutableStateOf(false) }
    Column(Modifier.clickable { expanded = !expanded }) {
        Text(title)
        if (expanded) Text("Detalle…")
    }
}

Consejo:
"remember = sobrevive recomposiciones; rememberSaveable = sobrevive cambios de configuración". Si la key cambia, remember reinicia su valor.

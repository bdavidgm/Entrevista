## Concepto
MVI (Model-View-[Intent](pregunta:33)) es un patrón con flujo unidireccional estricto: la View emite [Intents](pregunta:33) (acciones del usuario), un reducer/[ViewModel](pregunta:68) produce un nuevo Model (estado inmutable) y la View lo renderiza. El estado es una sola estructura inmutable, lo que facilita razonar, persistir y reproducir bugs.

A diferencia de [MVVM](pregunta:62), en MVI las acciones son explícitas ([sealed class](pregunta:19) de [Intents](pregunta:33)) y el estado se transforma como datos puros.

Ejemplo:
sealed interface CounterIntent { data object Inc : CounterIntent; data object Dec : CounterIntent }
data class CounterState(val count: Int = 0)

class CounterViewModel : ViewModel() {
    private val _state = MutableStateFlow(CounterState())
    val state = _state.asStateFlow()

    fun process(intent: CounterIntent) {
        _state.update { s -> when (intent) {
            CounterIntent.Inc -> s.copy(count = s.count + 1)
            CounterIntent.Dec -> s.copy(count = s.count - 1)
        }}
    }
}

ExplicaciónKotlin:
[sealed interface](pregunta:19) CounterIntent declara jerarquía cerrada; data object Inc y data object Dec son singletons de Kotlin 1.9+ (objetos de datos) implementando la interfaz sin estado. [data class](pregunta:3) CounterState(val count: Int = 0) es inmutable con valor por defecto 0 en el constructor. class CounterViewModel : [ViewModel](pregunta:68)() { private val _state = [MutableStateFlow](pregunta:86)(CounterState()); val state = _state.asStateFlow() } encapsula mutabilidad: _state es privado [MutableStateFlow](pregunta:86), state es [StateFlow](pregunta:86) de solo lectura via asStateFlow(). fun process([intent](pregunta:33): CounterIntent) recibe [intents](pregunta:33) tipados. _state.update { s -> when ([intent](pregunta:33)) { ... } } usa update con lambda que recibe estado anterior s y when exhaustivo sobre [sealed interface](pregunta:19): CounterIntent.Inc -> s.copy(count = s.count + 1) crea nueva instancia con copy de [data class](pregunta:3). CounterIntent.Dec decrementa simétricamente.

Consejo:
"Estado inmutable + [intents](pregunta:33) explícitos". Útil en pantallas complejas con muchas transiciones; en pantallas simples puede ser overkill.

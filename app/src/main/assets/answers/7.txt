Concepto:
Un companion object es un singleton declarado dentro de una clase y ligado a ella, que sirve para definir miembros accesibles a través del nombre de la clase, similares a los static de Java. Tiene acceso a los miembros privados de la clase contenedora.

Suele usarse para factories, constantes, fábricas de Fragment con argumentos y para implementar interfaces estáticas (Parcelable.Creator, etc.).

Ejemplo:
class UserFragment : Fragment() {
    companion object {
        private const val ARG_ID = "id"

        fun newInstance(id: String) = UserFragment().apply {
            arguments = bundleOf(ARG_ID to id)
        }
    }
}

UserFragment.newInstance("42")

Consejo:
Compáralo con static de Java (no es exactamente lo mismo: es un objeto real). Si necesitas verdadero static para Java interop, anota con @JvmStatic.

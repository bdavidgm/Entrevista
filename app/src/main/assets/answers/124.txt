Concepto:
La delegación de clase (class A : I by impl) permite que una clase implemente una interfaz reenviando todas sus llamadas a otra instancia, sin escribir manualmente cada método. Es la forma idiomática de Kotlin para aplicar el principio "composición sobre herencia".

El delegado tiene que implementar la interfaz; los miembros se pueden sobreescribir individualmente si se necesita ajustar algún comportamiento.

Ejemplo:
interface Repository {
    suspend fun load(): List<User>
}

class RemoteRepository : Repository { ... }

class LoggingRepository(
    private val inner: Repository
) : Repository by inner {
    override suspend fun load(): List<User> {
        Log.d(TAG, "load()")
        return inner.load()
    }
}

Consejo:
Distinguir de la delegación de propiedades (val x by lazy). Útil para decoradores (logging, cache, métrica) sin extender una clase concreta.

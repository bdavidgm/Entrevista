Concepto:
La delegación por clase en Kotlin (class delegation con by) implementa una interfaz delegando todas o parte de sus métodos a otra instancia, reduciendo boilerplate respecto a reenviar manualmente cada método.

Explicación detallada:
Sintaxis:
class PrinterImpl : Printer by DefaultPrinter()

El compilador genera métodos que invocan al delegado DefaultPrinter. Puedes sobrescribir métodos concretos y usar super<Interface> o el delegado según necesites.

Diferencia con delegated properties (by lazy, by viewModels):
- Class delegation: patrón de diseño a nivel de clase/interfaz.
- Property delegation: getters/setters generados para una propiedad.

Casos de uso:
- Decorator: class LoggingRepository(repo: UserRepository) : UserRepository by repo { override suspend fun getUser() { log(); return repo.getUser() } } — en Kotlin el override explícito reenvía al delegado.
- Composición sobre herencia: cumples interfaz sin extender clase concreta.
- Android: delegar en wrappers de sistema poco frecuente; más común en dominio y testing.

Limitaciones:
- Solo interfaces (no delegar implementación de clase concreta con by en la misma forma).
- Métodos no declarados en la interfaz no se delegan.
- Múltiples interfaces: class Foo : A by a, B by b

Cuándo usarlo:
Cuando una clase debe cumplir una interfaz y la mayoría de métodos son idénticos al de un colaborador interno; decoradores, proxies y fakes parciales.

Ejemplo:
interface Repository {
    suspend fun load(): List<Item>
    suspend fun save(item: Item)
}

class DatabaseRepository @Inject constructor(
    private val dao: ItemDao
) : Repository {
    override suspend fun load() = dao.getAll()
    override suspend fun save(item: Item) { dao.insert(item) }
}

class CachingRepository(
    private val inner: Repository,
    private val cache: MutableList<Item> = mutableListOf()
) : Repository by inner {
    override suspend fun load(): List<Item> {
        if (cache.isEmpty()) cache.addAll(inner.load())
        return cache
    }
}

Consejo para entrevista:
No confundas class delegation (by en clase) con property delegation (by lazy). Menciona patrón Decorator y que el compilador genera el forwarding.

Concepto:
Un Use Case (o Interactor) encapsula una operación de negocio única en una clase con un solo método público, normalmente invoke. Vive en la capa de dominio y orquesta uno o varios repositorios sin conocer detalles de UI ni de framework.

Ventajas: nombres expresivos del dominio, fácil de testear, reutilización entre ViewModels y separación clara entre "qué hace el sistema" y "cómo se presenta".

Ejemplo:
class GetActiveUsersUseCase(
    private val users: UserRepository,
    private val sessions: SessionRepository
) {
    suspend operator fun invoke(): List<User> = users.all()
        .filter { sessions.isActive(it.id) }
}

// En el ViewModel
viewModelScope.launch { _state.value = getActiveUsers() }

Consejo:
Crea Use Cases cuando hay lógica de negocio repetida o compleja; para llamadas triviales, llamar al Repository desde el ViewModel también es válido.

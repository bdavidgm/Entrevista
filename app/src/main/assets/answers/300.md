## Concepto
Gson es una librería de Google para convertir JSON ↔ objetos Java/Kotlin. Sigue siendo habitual en proyectos legacy; en código nuevo muchos equipos eligen [Moshi](pregunta:262) o [kotlinx.serialization](pregunta:263).

Ejemplo:
data class NoteDto(val id: String, val title: String)

val gson = Gson()
val dto = gson.fromJson(jsonString, NoteDto::class.java)
val json = gson.toJson(dto)

val retrofit = Retrofit.Builder()
    .baseUrl("https://api.example.com/")
    .addConverterFactory(GsonConverterFactory.create(gson))
    .build()

ExplicaciónKotlin:
`Gson()` crea el motor. `fromJson(json, NoteDto::class.java)` deserializa usando la clase Java; `toJson(dto)` serializa. Con [Retrofit](pregunta:207), `GsonConverterFactory.create(gson)` registra el converter. Cuidado con nullability de Kotlin: Gson no la respeta igual que Moshi. Ver [Gson](pregunta:261).

Consejo:
Si usas Gson con Kotlin, documenta campos nullables y considera `SerializedName` para nombres distintos.

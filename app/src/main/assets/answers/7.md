## Concepto
Un companion object es un singleton declarado dentro de una clase y ligado a ella, que sirve para definir miembros accesibles a través del nombre de la clase, similares a los static de Java. Tiene acceso a los miembros privados de la clase contenedora.

Suele usarse para factories, constantes, fábricas de [Fragment](pregunta:32) con argumentos y para implementar interfaces estáticas ([Parcelable](pregunta:129).Creator, etc.).

Ejemplo:
class UserFragment : Fragment() {
    companion object {
        private const val ARG_ID = "id"

        fun newInstance(id: String) = UserFragment().apply {
            arguments = bundleOf(ARG_ID to id)
        }
    }
}

UserFragment.newInstance("42")

ExplicaciónKotlin:
`class UserFragment : Fragment()` hereda de `Fragment` con la sintaxis kotliniana `:` en lugar de `extends`. El `companion object` es un singleton ligado a la clase, equivalente conceptual a miembros estáticos pero siendo un objeto real. `private const val ARG_ID` declara una constante de compilación visible solo dentro del companion. `fun newInstance(id: String)` es una factory que devuelve `UserFragment` con expresión de cuerpo único. La cadena `.apply { arguments = bundleOf(ARG_ID to id) }` usa la scope function `apply`: el receptor es la instancia recién creada (`this` implícito), se configura `arguments` y `apply` devuelve el mismo objeto. `bundleOf(ARG_ID to id)` es una función de utilidad que crea un `Bundle` con pares clave-valor usando el infix `to`. La llamada final `UserFragment.newInstance("42")` accede al companion como si fuera estático.

Consejo:
Compáralo con static de Java (no es exactamente lo mismo: es un objeto real). Si necesitas verdadero static para Java interop, anota con @JvmStatic.

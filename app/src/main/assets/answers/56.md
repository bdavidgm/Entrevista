## Concepto
derivedStateOf crea un State cuyo valor se calcula a partir de otros States y se "memoriza": el [Composable](pregunta:46) lector solo se recompone si el resultado derivado realmente cambia, aunque las fuentes cambien con frecuencia. Es la receta para evitar [recomposiciones](pregunta:47) excesivas en valores derivados.

Caso típico: mostrar un botón solo cuando el scroll ha pasado un umbral, sin recomponer en cada píxel.

Ejemplo:
val listState = rememberLazyListState()
val showFab by remember {
    derivedStateOf { listState.firstVisibleItemIndex > 0 }
}

if (showFab) ScrollToTopFab { /* ... */ }

ExplicaciónKotlin:
val listState = rememberLazyListState() conserva el estado del scroll entre [recomposiciones](pregunta:47) con [remember](pregunta:48). val showFab by [remember](pregunta:48) { derivedStateOf { listState.firstVisibleItemIndex > 0 } } anida dos mecanismos: [remember](pregunta:48) evita recrear derivedStateOf en cada [recomposición](pregunta:47), y derivedStateOf { } recibe una lambda que calcula un Boolean leyendo listState.firstVisibleItemIndex. La comparación > 0 devuelve true cuando el usuario ha scrolleado. by delega hacia el State<Boolean> resultante. if (showFab) ScrollToTopFab { /* ... */ } usa if como statement [composable](pregunta:46) condicional. derivedStateOf memoriza el último valor calculado y solo invalida lectores cuando el resultado cambia, no cuando firstVisibleItemIndex cambia sin cruzar el umbral. El comentario /* ... */ en ScrollToTopFab indica lambda vacía o placeholder.

Consejo:
"derivedStateOf cuando la fuente cambia mucho pero el resultado pocas veces". Sin él, leer firstVisibleItemIndex causaría [recomposiciones](pregunta:47) por cada frame de scroll.

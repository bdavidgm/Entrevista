## Concepto
derivedStateOf crea un State cuyo valor se calcula a partir de otros States y se "memoriza": el Composable lector solo se recompone si el resultado derivado realmente cambia, aunque las fuentes cambien con frecuencia. Es la receta para evitar recomposiciones excesivas en valores derivados.

Caso típico: mostrar un botón solo cuando el scroll ha pasado un umbral, sin recomponer en cada píxel.

Ejemplo:
val listState = rememberLazyListState()
val showFab by remember {
    derivedStateOf { listState.firstVisibleItemIndex > 0 }
}

if (showFab) ScrollToTopFab { /* ... */ }

ExplicaciónKotlin:
val listState = rememberLazyListState() conserva el estado del scroll entre recomposiciones con remember. val showFab by remember { derivedStateOf { listState.firstVisibleItemIndex > 0 } } anida dos mecanismos: remember evita recrear derivedStateOf en cada recomposición, y derivedStateOf { } recibe una lambda que calcula un Boolean leyendo listState.firstVisibleItemIndex. La comparación > 0 devuelve true cuando el usuario ha scrolleado. by delega hacia el State<Boolean> resultante. if (showFab) ScrollToTopFab { /* ... */ } usa if como statement composable condicional. derivedStateOf memoriza el último valor calculado y solo invalida lectores cuando el resultado cambia, no cuando firstVisibleItemIndex cambia sin cruzar el umbral. El comentario /* ... */ en ScrollToTopFab indica lambda vacía o placeholder.

Consejo:
"derivedStateOf cuando la fuente cambia mucho pero el resultado pocas veces". Sin él, leer firstVisibleItemIndex causaría recomposiciones por cada frame de scroll.

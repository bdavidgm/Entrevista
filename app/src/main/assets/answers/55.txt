Concepto:
derivedStateOf crea un State cuyo valor se recalcula solo cuando cambian las dependencias de estado leídas durante el cálculo, evitando recomposiciones innecesarias cuando un estado intermedio cambia pero el resultado derivado no. Optimiza cálculos costosos o condiciones booleanas compuestas en Compose.

Explicación detallada:
Ejemplo: lista con scroll; firstVisibleIndex cambia a menudo pero boolean "showButton = index > 0" solo cambia en 0→1. derivedStateOf { listState.firstVisibleItemIndex > 0 } evita recomponer hijos que solo dependen del boolean cuando el índice sigue >0.

Debe usarse dentro de remember { derivedStateOf { } } para persistir el cálculo entre recomposiciones. Lee estados snapshot durante la lambda; si el resultado es igual (equals), no notifica lectores.

No confundir con map de Flow ni transform en ViewModel: derivedStateOf es optimización de capa UI local. Para lógica de negocio derivada, ViewModel + StateFlow.distinctUntilChanged puede ser más testeable.

Cuándo usarlo:
Filtros derivados de estado que cambia frecuentemente, flags UI calculados, formatos derivados costosos. No uses para todo: añade complejidad; perfilar primero.

Ejemplo:
@Composable
fun ChatList(messages: List<Message>) {
    val listState = rememberLazyListState()
    val showScrollToBottom by remember {
        derivedStateOf {
            listState.firstVisibleItemIndex > 0
        }
    }
  Column {
        LazyColumn(Modifier.weight(1f), state = listState) {
            items(messages, key = { it.id }) { MessageBubble(it) }
        }
        if (showScrollToBottom) {
            FloatingActionButton(onClick = {
                // scroll to bottom
            }) { Icon(Icons.Default.ArrowDownward, null) }
        }
    }
}

Cuándo derivedStateOf vs calcular directo en Composable. Ejemplo firstVisibleItemIndex. remember { derivedStateOf }. distinctUntilChanged en Flow como análogo en ViewModel.

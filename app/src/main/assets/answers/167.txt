Concepto:
La ubicación se obtiene con FusedLocationProviderClient (Google Play services), que fusiona GPS, red y sensores. Pide permisos ACCESS_FINE_LOCATION / ACCESS_COARSE_LOCATION (y BACKGROUND desde Android 10 si necesitas ubicación con app en segundo plano).

Para apps modernas usa el contrato de permisos y comprueba si el usuario eligió “precisa” o “aproximada”.

Ejemplo:
fused.lastLocation.addOnSuccessListener { loc ->
    loc?.let { println("${it.latitude},${it.longitude}") }
}

val request = LocationRequest.Builder(Priority.PRIORITY_HIGH_ACCURACY, 10_000).build()
fused.requestLocationUpdates(request, callback, Looper.getMainLooper())


ExplicaciónKotlin:
`fused.lastLocation.addOnSuccessListener { loc -> ... }` registra callback Java con lambda Kotlin SAM-conversion; parámetro `loc` es `Location?` nullable. `loc?.let { println("${it.latitude},${it.longitude}") }` ejecuta bloque solo con fix válido; template string interpola coordenadas. `LocationRequest.Builder(Priority.PRIORITY_HIGH_ACCURACY, 10_000).build()` usa builder con prioridad enum y intervalo Long con separador `_`. `fused.requestLocationUpdates(request, callback, Looper.getMainLooper())` pasa callback y looper explícito del hilo principal. `println` es función estándar para demo. Las APIs Play Services son Java-first; Kotlin las consume con lambdas y safe calls.

Consejo:
Menciona justificación en UI, intervalos de muestreo y Foreground Service + tipo location si rastreas en background. Alternativa sin GMS: LocationManager.

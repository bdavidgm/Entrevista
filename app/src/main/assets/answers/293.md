## Concepto
Hilt es la librería oficial de inyección de dependencias en Android, basada en [Dagger 2](pregunta:269). Genera el grafo en compile-time y lo amarra a [Application](pregunta:128), [Activity](pregunta:31) y [ViewModel](pregunta:68) con anotaciones.

Ejemplo:
@HiltAndroidApp
class MyApp : Application()

@Module
@InstallIn(SingletonComponent::class)
object DatabaseModule {
    @Provides
    @Singleton
    fun provideDb(@ApplicationContext context: Context): AppDatabase =
        Room.databaseBuilder(context, AppDatabase::class.java, "app.db").build()

    @Provides
    fun provideNoteDao(db: AppDatabase): NoteDao = db.noteDao()
}

@HiltViewModel
class NotesViewModel @Inject constructor(
    private val noteDao: NoteDao
) : ViewModel() {
    val notes = noteDao.observeAll().stateIn(
        viewModelScope, SharingStarted.WhileSubscribed(5_000), emptyList()
    )
}

ExplicaciónKotlin:
`@HiltAndroidApp` arranca el componente raíz. `@Module` + `@InstallIn(SingletonComponent::class)` registra proveedores en el scope de app. `@Provides @Singleton` crea una sola instancia de [Room](pregunta:70). `@HiltViewModel` + `@Inject constructor` inyecta el DAO en el [ViewModel](pregunta:68). `stateIn` convierte el [Flow](pregunta:84) en [StateFlow](pregunta:86) listo para Compose. Ver también [Hilt](pregunta:74).

Consejo:
Prefiere inyección por constructor; usa `@Binds` cuando expongas interfaces.

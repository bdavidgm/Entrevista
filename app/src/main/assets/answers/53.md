## Concepto
LaunchedEffect es el side effect estándar de Compose para lanzar trabajo suspendido durante la composition. Crea un [CoroutineScope](pregunta:81) ligado a la composición; cuando alguna de sus keys cambia, cancela la [coroutine](pregunta:78) en curso y lanza otra nueva. Al salir el [Composable](pregunta:46), también se cancela.

Sirve para iniciar trabajo asíncrono que depende del ciclo de vida del [Composable](pregunta:46): colectar Flows, lanzar animaciones, hacer requests on-screen.

Ejemplo:
@Composable
fun UserScreen(userId: String, viewModel: UserViewModel = hiltViewModel()) {
    LaunchedEffect(userId) {
        viewModel.load(userId)
    }
    val user by viewModel.user.collectAsStateWithLifecycle()
    UserContent(user)
}

ExplicaciónKotlin:
UserScreen combina parámetros composables y side effects. userId: String es key de LaunchedEffect; [viewModel](pregunta:68): UserViewModel = [hiltViewModel](pregunta:141)() resuelve dependencias con valor por defecto. LaunchedEffect(userId) { [viewModel](pregunta:68).load(userId) } es una función [composable](pregunta:46) que lanza una [coroutine](pregunta:78) suspendida ligada a la composición: cuando userId cambia, cancela la [coroutine](pregunta:78) anterior y ejecuta load de nuevo. [viewModel](pregunta:68).load(userId) presumiblemente es [suspend](pregunta:79) o lanza trabajo interno. val user by [viewModel](pregunta:68).user.[collectAsStateWithLifecycle](pregunta:134)() observa un [Flow](pregunta:84)/[StateFlow](pregunta:86) con delegación by. UserContent(user) renderiza según el estado; si user es nullable, el tipo se reflejaría en la firma de UserContent. El patrón separa efecto (carga) de render (observación), usando scope de [coroutine](pregunta:78) gestionado por Compose en lugar de [lifecycleScope](pregunta:130) manual.

Consejo:
"Las keys controlan cuándo reinicia". Si no debe reiniciar nunca, usa Unit como key. Si depende de algo que cambia, pásalo como key.

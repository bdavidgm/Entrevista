## Concepto
LaunchedEffect es el side effect estándar de Compose para lanzar trabajo suspendido durante la composition. Crea un CoroutineScope ligado a la composición; cuando alguna de sus keys cambia, cancela la coroutine en curso y lanza otra nueva. Al salir el Composable, también se cancela.

Sirve para iniciar trabajo asíncrono que depende del ciclo de vida del Composable: colectar Flows, lanzar animaciones, hacer requests on-screen.

Ejemplo:
@Composable
fun UserScreen(userId: String, viewModel: UserViewModel = hiltViewModel()) {
    LaunchedEffect(userId) {
        viewModel.load(userId)
    }
    val user by viewModel.user.collectAsStateWithLifecycle()
    UserContent(user)
}

ExplicaciónKotlin:
UserScreen combina parámetros composables y side effects. userId: String es key de LaunchedEffect; viewModel: UserViewModel = hiltViewModel() resuelve dependencias con valor por defecto. LaunchedEffect(userId) { viewModel.load(userId) } es una función composable que lanza una coroutine suspendida ligada a la composición: cuando userId cambia, cancela la coroutine anterior y ejecuta load de nuevo. viewModel.load(userId) presumiblemente es suspend o lanza trabajo interno. val user by viewModel.user.collectAsStateWithLifecycle() observa un Flow/StateFlow con delegación by. UserContent(user) renderiza según el estado; si user es nullable, el tipo se reflejaría en la firma de UserContent. El patrón separa efecto (carga) de render (observación), usando scope de coroutine gestionado por Compose en lugar de lifecycleScope manual.

Consejo:
"Las keys controlan cuándo reinicia". Si no debe reiniciar nunca, usa Unit como key. Si depende de algo que cambia, pásalo como key.

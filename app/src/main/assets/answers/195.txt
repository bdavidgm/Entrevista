Concepto:
Git es control de versiones distribuido: ramas, merges, pull requests, bisect y etiquetas de release. En Android permite colaborar en paralelo, revisar cambios en Gradle/manifestes y revertir regresiones con precisión.

Se integra con CI (GitHub Actions, Bitrise) para lint, tests y builds por commit.

Ejemplo:
git checkout -b feature/login
git commit -am "Add login screen"
git push -u origin feature/login

Consejo:
Buenas prácticas: commits pequeños, .gitignore de build/, keystores fuera del repo o cifrados, y conventional commits si el equipo los usa.

Concepto:
Un Intent es un objeto de mensajería que describe una acción a realizar: iniciar Activity, Service, entregar broadcast, o pedir resultado a otra pantalla. Transporta componente objetivo (explícito) o acción genérica (implícito) más extras (Bundle) con datos primitivos, Parcelable, etc.

Explicación detallada:
Intent explícito: clase concreta, usado dentro de la app para navegación conocida. Intent implícito: action + category + data URI, resuelto por el sistema contra Manifest de apps (compartir, abrir enlace, cámara). PendingIntent envuelve Intent para ejecutar en nombre de tu app más tarde (notificaciones, widgets, alarmas) con permisos y mutabilidad controlada.

Extras tipados con constantes; evita strings mágicos dispersos. Activity Result API reemplaza startActivityForResult deprecado. Deep links y App Links asocian intents con URLs.

Seguridad: no exportar components innecesarios; validar intents entrantes; PendingIntent FLAG_IMMUTABLE en Android 12+. En entrevistas preguntan flags (NEW_TASK, CLEAR_TOP), task affinity y intent filters.

Cuándo usarlo:
Navegación entre pantallas, compartir contenido, abrir settings, lanzar chooser, programar notificaciones. Para navegación interna grande, Navigation Component o Compose Navigation reducen intents manuales.

Ejemplo:
val intent = Intent(this, DetailActivity::class.java).apply {
    putExtra(DetailActivity.EXTRA_ID, productId)
}
startActivity(intent)

// Activity Result
private val pickImage = registerForActivityResult(
    ActivityResultContracts.GetContent()
) { uri -> viewModel.onImagePicked(uri) }

fun selectPhoto() = pickImage.launch("image/*")

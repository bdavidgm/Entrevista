Concepto:
Un Intent es un mensaje que describe una operación a realizar (abrir una pantalla, enviar un broadcast, arrancar un Service) y se entrega al sistema, que decide qué componente lo atiende. Puede llevar datos a través de extras y una acción + categoría que describe la intención.

Es la pieza central de la comunicación entre componentes Android, tanto dentro de la misma app como entre apps distintas.

Ejemplo:
val intent = Intent(this, DetailActivity::class.java).apply {
    putExtra("userId", "42")
}
startActivity(intent)

val shareIntent = Intent(Intent.ACTION_SEND).apply {
    type = "text/plain"
    putExtra(Intent.EXTRA_TEXT, "Hola")
}
startActivity(Intent.createChooser(shareIntent, null))

ExplicaciónKotlin:
`Intent(this, DetailActivity::class.java)` construye un intent explícito: `this` es el `Context` (Activity), y `DetailActivity::class.java` obtiene la clase destino vía reflexión Kotlin. `.apply { putExtra("userId", "42") }` configura extras con scope function; `putExtra` es API Java del Intent. `Intent(Intent.ACTION_SEND).apply { ... }` crea un intent implícito por acción; asignación a `type` y `putExtra` con constantes de `Intent`. `Intent.createChooser(shareIntent, null)` envuelve el intent para mostrar selector de apps; el segundo parámetro es título `CharSequence?`. `startActivity(...)` lanza la navegación. Las extensiones y scope functions de Kotlin reducen el ruido respecto a Java, pero las APIs subyacentes siguen siendo las del framework Android.

Consejo:
Distinguir explícito vs implícito y conocer PendingIntent para notificaciones/alarmas. En Android 12+ los intents implícitos requieren atención por las restricciones de exported.

## Concepto
State hoisting es el patrón de "subir" el estado a un ancestro común, dejando los hijos sin estado propio (stateless) que reciben el valor y un callback para cambiarlo. Mejora la testabilidad, la reutilización y mantiene una única fuente de verdad.

Forma canónica de un componente hoisted: value: T y onValueChange: (T) -> Unit. El padre puede tener el estado en [remember](pregunta:48) o, mejor, en un [ViewModel](pregunta:68).

Ejemplo:
@Composable
fun SearchScreen(viewModel: SearchViewModel = hiltViewModel()) {
    val query by viewModel.query.collectAsStateWithLifecycle()
    SearchField(value = query, onChange = viewModel::onQueryChange)
}

@Composable
fun SearchField(value: String, onChange: (String) -> Unit) {
    TextField(value = value, onValueChange = onChange)
}

ExplicaciónKotlin:
SearchScreen muestra state hoisting hacia un [ViewModel](pregunta:68). El parámetro [viewModel](pregunta:68): SearchViewModel = [hiltViewModel](pregunta:141)() usa valor por defecto y [hiltViewModel](pregunta:141)() como función [composable](pregunta:46) de [Hilt](pregunta:74) que resuelve el [ViewModel](pregunta:68) del NavBackStackEntry o componente actual. val query by [viewModel](pregunta:68).query.[collectAsStateWithLifecycle](pregunta:134)() combina delegación by con [collectAsStateWithLifecycle](pregunta:134), que convierte el [Flow](pregunta:84) query en State observable respetando el Lifecycle. SearchField(value = query, onChange = [viewModel](pregunta:68)::onQueryChange) pasa el estado hacia abajo y usa method reference [viewModel](pregunta:68)::onQueryChange para el callback, equivalente a { [viewModel](pregunta:68).onQueryChange(it) } si la firma lo permite. SearchField es stateless: recibe value: String y onChange: (String) -> Unit, tipando explícitamente la función lambda que recibe String y devuelve Unit. TextField reenvía onValueChange = onChange, conectando la API de Material con el callback del padre sin lógica intermedia.

Consejo:
"State down, events up": flujo unidireccional. Hoist al nivel más bajo que necesite compartirlo. El [ViewModel](pregunta:68) suele ser el hoist final.

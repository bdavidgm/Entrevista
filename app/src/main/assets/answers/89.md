## Concepto
JUnit es el framework estándar de pruebas unitarias en JVM y Android (JUnit 4 y JUnit 5 en proyectos Kotlin). Proporciona anotaciones @Test, assertions, reglas y organización de suites para verificar comportamiento de forma automatizada.

## Explicación detallada
@Test marca métodos ejecutables por [Gradle](pregunta:109) testDebugUnitTest. Assertions: assertEquals, assertTrue, assertThrows. @Before y @After preparan estado. Parameterized tests para muchos inputs.

En Android Kotlin, kotlinx-[coroutines](pregunta:78)-test ofrece runTest en lugar de [runBlocking](pregunta:142) en tests de [suspend](pregunta:79). InstantTaskExecutorRule legacy para [LiveData](pregunta:69); [MainDispatcherRule](pregunta:148) para [coroutines](pregunta:78).

JUnit 5 (Jupiter) con Kotlin: @ExtendWith, tags, nested classes; adopción variable en Android por tooling.

Integración CI: fallo de test bloquea merge. Naming descriptivo: backticks `when X then Y`.

En el contexto de proyectos Android reales, combina esta idea con las guías oficiales de Android Developers y con métricas de calidad (crashes, [ANR](pregunta:95), Vitals) cuando aplique. En code review, pregunta siempre por ciclo de vida, testabilidad y si la solución escala cuando el equipo crece.

## Cuándo usarlo
Toda lógica testeable sin emulador. Combina con [MockK](pregunta:90), [Turbine](pregunta:93), Truth según necesidad.

Ejemplo:
class DiscountTest {
    @Test
    fun `applies ten percent`() {
        val result = calculateDiscount(100, 0.10)
        assertEquals(90.0, result, 0.001)
    }
}


ExplicaciónKotlin:
`class DiscountTest` define una clase de prueba JUnit en Kotlin; no requiere modificador `public` porque la visibilidad por defecto es pública a nivel de módulo. `@Test` anota el método `applies ten percent`, cuyo identificador entre backticks es válido en Kotlin y se lee como frase en español/inglés. La función invoca `calculateDiscount(100, 0.10)`, donde `100` es `Int` y `0.10` es `Double`; Kotlin promueve o exige tipos compatibles según la firma. `assertEquals(90.0, result, 0.001)` es la sobrecarga para `Double` con delta de tolerancia por precisión de coma flotante. No hay `runTest` porque la función bajo prueba no es `suspend`; es lógica pura JVM. El patrón es unit test clásico: Arrange (entrada 100, 10%), Act (llamada), Assert (resultado 90.0).

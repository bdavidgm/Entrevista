## Concepto
Timber es un wrapper de logging sobre `Log` de Android que añade árboles plantables, tags automáticos y facilita silenciar logs en release.

Ejemplo:
class App : Application() {
    override fun onCreate() {
        super.onCreate()
        if (BuildConfig.DEBUG) Timber.plant(Timber.DebugTree())
    }
}
Timber.d("Usuario %s", userId)

ExplicaciónKotlin:
`Timber.plant(DebugTree())` registra el árbol de debug solo en DEBUG. `Timber.d` formatea el mensaje. En release no plantas árboles o usas uno que no imprima.

Consejo:
Mejor que `Log.d` repartidos: un solo punto de configuración y menos riesgo de logs en producción.

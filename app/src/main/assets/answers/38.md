## Concepto
Un ContentProvider es un componente que expone datos estructurados a otras apps mediante una URI (content://) y un conjunto de operaciones tipo CRUD. Es el mecanismo estándar de Android para compartir datos entre aplicaciones (contactos, fotos, calendario).

Dentro de la propia app rara vez se usa directamente: para datos internos suele bastar con [Room](pregunta:70) o [DataStore](pregunta:73). Se sigue usando como cliente vía [ContentResolver](pregunta:178) para acceder a [MediaStore](pregunta:179), ContactsContract, etc.

Ejemplo:
val cursor = contentResolver.query(
    ContactsContract.Contacts.CONTENT_URI,
    arrayOf(ContactsContract.Contacts.DISPLAY_NAME),
    null, null, null
)
cursor?.use {
    while (it.moveToNext()) {
        Log.d(TAG, it.getString(0))
    }
}

ExplicaciónKotlin:
`contentResolver.query(...)` invoca la API de ContentProvider del framework; los argumentos `null` son parámetros opcionales de selección/orden que aquí no se usan. `arrayOf(...)` crea un arreglo Kotlin de proyección de tipos (`Array<String>`) para las columnas proyectadas. `cursor?.use { ... }` combina [safe call](pregunta:17) con la [función de extensión](pregunta:9) estándar `use`: si `query` devuelve null, no entra al bloque; si hay cursor, lo cierra automáticamente al salir (como try-with-resources). Dentro del lambda, `it` es el `Cursor`; `while (it.moveToNext())` itera filas. `it.getString(0)` accede a la columna por índice. `Log.d(TAG, ...)` registra cada nombre. Este patrón es idiomático en Kotlin para recursos que implementan `Closeable`, evitando fugas de cursores.

Consejo:
"Para compartir datos entre apps; dentro de la app, [Room](pregunta:70)". Necesario para integrarse con APIs del sistema como [MediaStore](pregunta:179) y SAF.

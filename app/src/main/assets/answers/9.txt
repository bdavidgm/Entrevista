Concepto:
Las extension functions permiten añadir nuevos métodos a clases existentes sin modificarlas ni heredarlas, declarándolos con la sintaxis fun Tipo.nombre(...). Bajo el capó se compilan como funciones estáticas que reciben el receptor como primer parámetro.

Como son estáticas, no son polimórficas: se resuelven por el tipo declarado, no por el tipo en runtime. Si hay un método miembro con la misma firma, el miembro gana.

Ejemplo:
fun String.toSlug(): String =
    trim().lowercase().replace(" ", "-")

fun Int.dpToPx(context: Context): Int =
    (this * context.resources.displayMetrics.density).toInt()

"Hola Mundo".toSlug() // "hola-mundo"

Consejo:
Resalta que no son polimórficas y que ayudan a construir DSLs y APIs idiomáticas. Útiles para extender SDK/Android (View.show(), Context.toast()).

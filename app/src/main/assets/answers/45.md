## Concepto
[ViewModel](pregunta:68) sobrevive a los cambios de configuración (rotación, [modo oscuro](pregunta:107), cambio de idioma) porque está asociado al ViewModelStore del componente, que la [Activity](pregunta:31)/[Fragment](pregunta:32) retiene cuando se destruye y recrea por configuración. Así se evita perder estado y volver a pedir datos a la red.

[ViewModel](pregunta:68) no sobrevive a process death directamente; para eso se combina con [SavedStateHandle](pregunta:140).

Ejemplo:
@HiltViewModel
class HomeViewModel @Inject constructor(
    private val repo: HomeRepository
) : ViewModel() {
    val state: StateFlow<HomeState> = repo.observe()
        .stateIn(viewModelScope, SharingStarted.WhileSubscribed(5_000), HomeState.Loading)
}

class HomeFragment : Fragment() {
    private val viewModel: HomeViewModel by viewModels()
}

ExplicaciónKotlin:
[@HiltViewModel](pregunta:141) es una anotación de [Hilt](pregunta:74) que marca la clase para que el contenedor DI genere el factory del [ViewModel](pregunta:68). class HomeViewModel @Inject constructor(...) indica inyección por constructor: [Hilt](pregunta:74) proveerá HomeRepository al crear la instancia. private val repo limita la visibilidad y declara una propiedad inmutable de solo lectura. : [ViewModel](pregunta:68)() hereda de la clase base de Jetpack con el constructor vacío de Kotlin. val state: [StateFlow](pregunta:86)<HomeState> declara un flujo caliente tipado con el genérico HomeState. La cadena repo.observe().stateIn(...) encadena operadores de [Flow](pregunta:84): observe() devuelve un [Flow](pregunta:84), y stateIn lo convierte en [StateFlow](pregunta:86) dentro de [viewModelScope](pregunta:131) (scope del [ViewModel](pregunta:68) cancelado en onCleared). SharingStarted.WhileSubscribed(5_000) usa un literal numérico con separador de miles (_) válido en Kotlin. HomeFragment usa by viewModels() que es una propiedad delegada (property delegate) de KTX que resuelve el [ViewModel](pregunta:68) lazy y ligado al [Fragment](pregunta:32).

Consejo:
"Rotación = [ViewModel](pregunta:68); muerte de proceso = [SavedStateHandle](pregunta:140)". Esto resuelve la pérdida de datos sin recargar y separa la lógica de la UI.

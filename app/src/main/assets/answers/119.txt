Concepto:
Un deep link (enlace profundo) es un URI o App Link que abre directamente una pantalla o flujo específico dentro de la app, en lugar de solo lanzar la Activity principal.

Explicación detallada:
Tipos:
- Deep link clásico: intent-filter con scheme custom (myapp://product/42). Puede mostrar selector de apps si varias coinciden.
- App Links (Android 6+): https con autoVerify="true" y Digital Asset Links en el dominio. Abren tu app sin diálogo si la verificación es correcta.
- Dynamic Links (Firebase, legacy) y enlaces de marketing que redirigen al destino in-app.

Configuración en AndroidManifest:
- intent-filter con action VIEW, categorías DEFAULT y BROWSABLE.
- data: scheme, host, pathPrefix o pathPattern.

Navigation Component:
- <deepLink app:uri="https://example.com/item/{id}" /> en el destino.
- navController.handleDeepLink(intent) en onCreate/onNewIntent.

Flujo típico: usuario toca enlace en email o web -> sistema resuelve Intent -> Activity host del NavHost -> grafo navega al destino con argumentos.

Consideraciones:
- Estado cold start: la app puede no tener sesión; redirigir a login y guardar destino pendiente.
- onNewIntent si la Activity existe con launchMode singleTop.
- Pruebas con adb shell am start -a android.intent.action.VIEW -d "https://..."

Cuándo usarlo:
Compartir contenido, campañas, recuperación de carrito, notificaciones que abren pantalla concreta, integración web-to-app.

Ejemplo:
// AndroidManifest.xml dentro de activity
<intent-filter android:autoVerify="true">
    <action android:name="android.intent.action.VIEW" />
    <category android:name="android.intent.category.DEFAULT" />
    <category android:name="android.intent.category.BROWSABLE" />
    <data android:scheme="https"
          android:host="example.com"
          android:pathPrefix="/product" />
</intent-filter>

class MainActivity : AppCompatActivity() {
    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        setContent { App(navController) }
        navController.handleDeepLink(intent)
    }
}


ExplicaciónKotlin:
El XML del manifest no es Kotlin; define intent-filter para App Links. `class MainActivity : AppCompatActivity()` en Kotlin hereda Activity. `override fun onCreate(savedInstanceState: Bundle?)` — `Bundle?` nullable. `setContent { App(navController) }` usa Compose Activity API: lambda `@Composable` configura UI; `navController` parámetro de grafo. `navController.handleDeepLink(intent)` procesa URI del Intent entrante (cold start o `onNewIntent`). `intent` es propiedad de Activity. El flujo combina interoperabilidad XML manifest + Kotlin Activity + Navigation Compose; `handleDeepLink` es extensión o método del NavController que parsea URI y navega al destino con argumentos.

Consejo para entrevista:
Diferencia deep link custom vs App Links verificados. Menciona handleDeepLink, cold start y prueba con adb. Si hablas de notificaciones, enlaza con PendingIntent que lleva el URI.

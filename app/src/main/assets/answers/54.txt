Concepto:
SideEffect ejecuta un bloque después de cada recomposición exitosa, útil para publicar valores hacia código no-Compose (analytics, librerías que esperan estado actualizado). DisposableEffect ejecuta un bloque al entrar en la composition y permite definir un onDispose que se ejecuta al salir o al cambiar las keys, ideal para registrar/desregistrar listeners.

Ambos son side effects "manejados" por Compose, en contraste con escribir código no controlado en el cuerpo del Composable.

Ejemplo:
SideEffect {
    analytics.set("isPremium", user.isPremium)
}

DisposableEffect(systemUiController, useDark) {
    systemUiController.setStatusBarColor(if (useDark) Color.Black else Color.White)
    onDispose { /* limpieza si hace falta */ }
}

Consejo:
"SideEffect = tras recomposición, sin limpieza"; "DisposableEffect = entrar/salir con onDispose". LaunchedEffect, para trabajo suspendido.

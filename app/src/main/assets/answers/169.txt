Concepto:
CameraX es una biblioteca Jetpack que simplifica el uso de la cámara en Android unificando Camera2 bajo casos de uso (Preview, ImageCapture, ImageAnalysis) y enlazándolos automáticamente al ciclo de vida.

Explicación detallada:
CameraX resuelve el problema histórico de fragmentación: distintos OEMs implementan Camera2 de forma inconsistente. Ofrece una API estable orientada a casos de uso en lugar de gestionar manualmente CaptureSession, Surfaces y estados.

Componentes clave:
- ProcessCameraProvider: punto de entrada singleton por contexto.
- CameraSelector: cámara frontal/trasera, LENS_FACING, filtros por nivel de hardware.
- Use cases:
  - Preview: stream en PreviewView (Compose: AndroidView con PreviewView).
  - ImageCapture: fotos JPEG con flash, relación de aspecto, rotación EXIF.
  - ImageAnalysis: frames para ML Kit, escáner QR, detección de rostro (en background thread/analyzer).
- bindToLifecycle(owner, selector, ...useCases): abre/cierra cámara según STARTED/STOPPED del LifecycleOwner.

Extensiones opcionales: camera-camera2, camera-lifecycle, camera-view, camera-extensions (HDR, Night, Bokeh) donde el dispositivo lo soporte.

Ventajas: menos código, pruebas en emulador limitadas pero API coherente, compatibilidad retroactiva vía quirk layer, integración con ListenableFuture y executors.

Limitaciones: no sustituye apps de cámara pro que necesitan control manual total; algunas funciones avanzadas siguen requiriendo Camera2 directo.

Cuándo usarlo:
- Cualquier app que muestre vista previa y capture fotos o analice frames.
- Proyectos nuevos en lugar de APIs camera deprecadas (android.hardware.Camera).
- Cuando quieres reducir bugs entre Samsung, Xiaomi, Pixel, etc.

Ejemplo:
implementation("androidx.camera:camera-camera2:1.3.x")
implementation("androidx.camera:camera-lifecycle:1.3.x")
implementation("androidx.camera:camera-view:1.3.x")

@Composable
fun CameraScreen() {
    val context = LocalContext.current
    val lifecycleOwner = LocalLifecycleOwner.current
    AndroidView(
        factory = { ctx ->
            PreviewView(ctx).apply {
                val future = ProcessCameraProvider.getInstance(ctx)
                future.addListener({
                    val provider = future.get()
                    val preview = Preview.Builder().build()
                    preview.setSurfaceProvider(surfaceProvider)
                    provider.bindToLifecycle(
                        lifecycleOwner,
                        CameraSelector.DEFAULT_BACK_CAMERA,
                        preview
                    )
                }, ContextCompat.getMainExecutor(ctx))
            }
        },
        modifier = Modifier.fillMaxSize()
    )
}

Consejo para entrevista:
Enlaza CameraX con LifecycleOwner, nombra los tres use cases y di que encapsula Camera2. Menciona ImageAnalysis para ML y que Extensions son opcionales. Contrasta con intent ACTION_IMAGE_CAPTURE para casos triviales.

Concepto:
Para saber si hay red usable se usa ConnectivityManager: activeNetwork + getNetworkCapabilities o, mejor, registerDefaultNetworkCallback para recibir onAvailable/onLost/onCapabilitiesChanged. NET_CAPABILITY_VALIDATED indica Internet real (no solo Wi‑Fi asociado sin portal).

El permiso ACCESS_NETWORK_STATE es normal (manifest).

Ejemplo:
val cm = getSystemService(ConnectivityManager::class.java)
val online = cm.activeNetwork?.let { net ->
    cm.getNetworkCapabilities(net)
        ?.hasCapability(NetworkCapabilities.NET_CAPABILITY_INTERNET) == true
} == true


ExplicaciónKotlin:
`getSystemService(ConnectivityManager::class.java)` usa reified class literal Java para obtener servicio del sistema — patrón Android KTX. `cm.activeNetwork?.let { net -> ... }` safe call encadenado: si no hay red activa, expresión completa es null. Dentro, `cm.getNetworkCapabilities(net)?.hasCapability(...) == true` combina safe call, comparación booleana y equality estricta con `true` para convertir nullable Boolean a Boolean false por defecto. La expresión externa `== true` convierte todo el `let` nullable en booleano de presencia de internet. `NET_CAPABILITY_INTERNET` es constante int de `NetworkCapabilities`. Kotlin evita NPE con operadores `?.` en cadena profunda.

Consejo:
No uses APIs deprecadas tipo NetworkInfo. Para políticas de sync, WorkManager Constraints.setRequiredNetworkType es la integración natural.

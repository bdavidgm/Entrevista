## Concepto
Reducir el tamaño del APK/AAB mejora descargas, conversión en mercados con datos caros y límites de almacenamiento. Estrategias incluyen App Bundle, splits ABI/densidad, R8 shrinking, recursos optimizados y auditoría de dependencias.

## Explicación detallada
Android App Bundle: Play genera APKs optimizados por dispositivo (abi, density, language). enableSplit en bundle. R8 minifyEnabled y shrinkResources eliminan código y recursos no usados con reglas ProGuard correctas.

WebP/AVIF para imágenes; vector drawables donde aplique. Evita bibliotecas duplicadas (BOM Compose alinea versiones). abiFilters o splits si distribuyes fuera de Play.

Analiza APK Analyzer en Android Studio. dynamic feature modules para features opcionales. Elimina assets no usados y traducciones no necesarias con resConfigs.

En el contexto de proyectos Android reales, combina esta idea con las guías oficiales de Android Developers y con métricas de calidad (crashes, ANR, Vitals) cuando aplique. En code review, pregunta siempre por ciclo de vida, testabilidad y si la solución escala cuando el equipo crece.

Durante la entrevista, estructura la respuesta en definición, ejemplo concreto de tu experiencia o proyecto demo, y trade-offs frente a alternativas. Mencionar versiones recientes de API o biblioteca (Material3, Paging 3, Hilt 2.x) demuestra que sigues el ecosistema, sin necesidad de memorizar números de versión exactos.

## Cuándo usarlo
Siempre antes de release; monitorea tamaño en CI. Balance: no sobre-agresivo shrinking que rompa reflexión sin reglas.

Ejemplo:
android {
    bundle { language { enableSplit = true } }
    buildTypes {
        release {
            isMinifyEnabled = true
            isShrinkResources = true
        }
    }
}


ExplicaciónKotlin:
El bloque `android { }` es extensión AGP en Gradle Kotlin. `bundle { language { enableSplit = true } }` activa splits de idioma en App Bundle mediante propiedades asignadas con `=`. `buildTypes { release { ... } }` configura variante release. `isMinifyEnabled = true` y `isShrinkResources = true` son propiedades booleanas del DSL que activan R8 y eliminación de recursos huérfanos. No hay código Kotlin de app: es script de build compilado como `.gradle.kts`. La sintaxis usa asignación Kotlin (`=`) en lugar de Groovy. `release` es objeto de configuración tipado; el IDE autocompleta propiedades.

Consejo para entrevista:
AAB + R8 + recursos + dependencias. Menciona APK Analyzer. Seguimiento: ¿Qué es dex? Métodos multidex si >64K.

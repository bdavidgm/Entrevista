## Concepto
Un Service "started" se arranca con startService/startForegroundService y vive hasta que él mismo se detiene (stopSelf) o lo detiene quien lo creó (stopService); ejecuta trabajo aunque el cliente desaparezca. Un Service "bound" se enlaza con bindService desde uno o varios clientes, expone un IBinder para llamarlo como API y se destruye cuando se desligan todos los clientes.

Un mismo Service puede ser ambos a la vez. El started es típico para procesos largos (reproductor, descargas); el bound, para comunicación cliente-servicio (música controlada desde UI).

Ejemplo:
// Started
startForegroundService(Intent(this, MusicService::class.java))

// Bound
val conn = object : ServiceConnection {
    override fun onServiceConnected(name: ComponentName?, binder: IBinder?) { /* ... */ }
    override fun onServiceDisconnected(name: ComponentName?) { }
}
bindService(Intent(this, MyService::class.java), conn, Context.BIND_AUTO_CREATE)

ExplicaciónKotlin:
`startForegroundService(Intent(this, MusicService::class.java))` crea un `Intent` explícito usando `MusicService::class.java`, la forma de obtener `Class<T>` desde Kotlin para APIs Android escritas en Java. `this` en un `Activity` o `Context` es el receptor implícito. El servicio started no necesita `IBinder` si solo ejecuta trabajo en segundo plano. Para el bound service, `object : ServiceConnection { ... }` declara un objeto anónimo que implementa la interfaz Java `ServiceConnection`; Kotlin traduce `override fun onServiceConnected(...)` en métodos que deben coincidir con la firma Java. Los parámetros `name: ComponentName?` e `ibinder: IBinder?` son nullable porque el sistema puede desconectar sin aviso. `bindService(..., conn, Context.BIND_AUTO_CREATE)` pasa flags como entero; `BIND_AUTO_CREATE` crea el servicio si aún no existe. `Intent(this, MyService::class.java)` reutiliza la misma sintaxis de clase que en el started. Un mismo `Service` puede combinar ambos modos implementando `onStartCommand` y `onBind`. La diferencia clave en Kotlin es el estilo declarativo del `object` frente a clases internas verbosas de Java.

Consejo:
"Started vive por sí solo; bound vive mientras haya clientes". Conoce que startForegroundService obliga a llamar startForeground en 5 s.

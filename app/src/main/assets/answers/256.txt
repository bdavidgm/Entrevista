Concepto:
Tipo::class es un class literal que devuelve un KClass<T>, la representación de una clase en el sistema de reflexión de Kotlin. Permite consultar su nombre, comprobar instancias, usarla como clave o pasar el tipo sin crear un objeto.

Ejemplo:
val type: KClass<User> = User::class

println(type.simpleName)       // User
println(type.qualifiedName)    // com.example.User
println(type.isInstance(user)) // true

fun register(type: KClass<*>) { /* ... */ }
register(User::class)

ExplicaciónKotlin:
`val type: KClass<User> = User::class` usa class literal Kotlin; `::class` devuelve metadatos de tipo sin instanciar. Anotación de tipo explícita `KClass<User>` documenta genérico concreto. `type.simpleName` y `type.qualifiedName` son propiedades de reflexión Kotlin (requieren kotlin-reflect en runtime para algunas operaciones). `type.isInstance(user)` equivale a `user is User` pero dinámico cuando el tipo viene como valor. `fun register(type: KClass<*>)` usa star projection: acepta cualquier KClass sin fijar parámetro de tipo. `register(User::class)` pasa literal en compile time. `KClass` es contraparte Kotlin de `Class<?>` Java, pero modela nullability y generics Kotlin mejor. No confundir con referencia a constructor `::User` (usada en callables). Útil en registries, DI manual o serializers que necesitan tipo en runtime.

Consejo:
KClass<*> representa una clase cuyo tipo concreto no conoces. No confundas User::class con ::User: el segundo es una referencia al constructor.

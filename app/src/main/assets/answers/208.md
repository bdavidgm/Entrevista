## Concepto
OkHttp es el cliente HTTP de Square usado por Retrofit: connection pooling, HTTP/2, interceptors (logging, auth token refresh), timeouts y caché de respuestas opcional.

También puedes usarlo solo sin Retrofit para llamadas muy custom.

Ejemplo:
val client = OkHttpClient.Builder()
    .addInterceptor { chain ->
        val req = chain.request().newBuilder().addHeader("Authorization", "Bearer $token").build()
        chain.proceed(req)
    }
    .connectTimeout(15, TimeUnit.SECONDS)
    .build()

ExplicaciónKotlin:
`OkHttpClient.Builder()` inicia el patrón builder de OkHttp. `.addInterceptor { chain -> ... }` registra un interceptor con lambda Kotlin: `chain` es el parámetro tipado implícitamente como `Interceptor.Chain`. Dentro, `chain.request()` obtiene el `Request` original; `.newBuilder()` clona editable; `.addHeader("Authorization", "Bearer $token")` usa template string con interpolación `$token` para insertar el token en el header; `.build()` produce el request modificado. `chain.proceed(req)` continúa la cadena y devuelve `Response`. La lambda debe devolver `Response` como última expresión (semántica de expresión única implícita). `.connectTimeout(15, TimeUnit.SECONDS)` fija timeout con literal `15` y enum Java `TimeUnit`. `.build()` finaliza el cliente inmutable asignable a `val client`. Es interop Kotlin-Java con SAM-like lambda sobre interfaz funcional `Interceptor`.

Consejo:
Distingue application vs network interceptors. Para cert pinning y metrics, OkHttp es el punto único de configuración.

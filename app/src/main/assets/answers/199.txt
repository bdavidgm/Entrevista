Concepto:
ConstraintLayout es un ViewGroup plano que posiciona hijos con restricciones (constraints) a otros widgets o al padre, soportando bias, chains, barriers y guidelines para layouts flexibles con un solo nivel de anidamiento.

Explicación detallada:
Cada vista tiene constraints horizontales y verticales (start/end/top/bottom) anclados a referencias. Ventajas:
- Layouts planos → mejor rendimiento que muchos LinearLayout anidados.
- Responsive: percent, ratio, dimension ratio en ImageView.
- Chains: distribuyen espacio horizontal/vertical (spread, packed, spread_inside).
- Barrier: alinea múltiples vistas al borde más extremo (útil con textos de altura variable).
- Guideline: línea virtual al 50% para centrar.
- Group: visibilidad conjunta.

Editor visual en Android Studio facilita diseño. Programático: ConstraintSet para animaciones o cambios dinámicos.

Compose: ConstraintLayout también existe en androidx.constraintlayout.compose para casos complejos, aunque muchas UIs usan Box/Column/Row.

Cuándo usarlo:
- Pantallas XML complejas sin migrar a Compose.
- Formularios, cards con elementos alineados en distintas posiciones.
- Reemplazo de RelativeLayout/LinearLayout anidados.

Ejemplo:
<androidx.constraintlayout.widget.ConstraintLayout
    android:layout_width="match_parent"
    android:layout_height="match_parent">

    <Button
        android:id="@+id/btn"
        android:layout_width="wrap_content"
        android:layout_height="wrap_content"
        android:text="OK"
        app:layout_constraintBottom_toBottomOf="parent"
        app:layout_constraintEnd_toEndOf="parent"
        app:layout_constraintStart_toStartOf="parent"
        android:layout_marginBottom="24dp" />
</androidx.constraintlayout.widget.ConstraintLayout>

Consejo para entrevista:
Enfatiza “layout plano”, chains y barrier. Contrasta con LinearLayout anidado. Si el proyecto es Compose-first, di que ConstraintLayout XML es conocimiento legacy pero ConstraintLayout compose sirve para pantallas híbridas.

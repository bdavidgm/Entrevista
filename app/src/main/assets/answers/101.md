## Concepto
Los datos sensibles (tokens, contraseñas, claves API, PII) no deben almacenarse en texto plano en [SharedPreferences](pregunta:177) sin cifrar, archivos externos legibles, logs ni código fuente. Se usan [Android Keystore](pregunta:103), EncryptedSharedPreferences/[DataStore](pregunta:73), servidor remoto y principio de mínimo privilegio.

## Explicación detallada
Keystore genera claves hardware-backed difíciles de extraer en dispositivos compatibles. EncryptedSharedPreferences (Security crypto) cifra valores. Para tokens OAuth: almacenar refresh token cifrado; access token corta vida en memoria si posible.

Nunca hardcodear API secrets en APK (reverse engineering con apktool). Usa backend proxy o Play [App Signing](pregunta:219) + remote config con rotación. [certificate pinning](pregunta:154) para MITM.

Logs: desactivar body en [OkHttp](pregunta:208) producción. excludeFromBackup en manifest para backups cloud. [BiometricPrompt](pregunta:185) antes de mostrar datos ultra sensibles.

En el contexto de proyectos Android reales, combina esta idea con las guías oficiales de Android Developers y con métricas de calidad (crashes, [ANR](pregunta:95), Vitals) cuando aplique. En code review, pregunta siempre por ciclo de vida, testabilidad y si la solución escala cuando el equipo crece.

Durante la entrevista, estructura la respuesta en definición, ejemplo concreto de tu experiencia o proyecto demo, y trade-offs frente a alternativas. Mencionar versiones recientes de API o biblioteca (Material3, [Paging 3](pregunta:76), [Hilt](pregunta:74) 2.x) demuestra que sigues el ecosistema, sin necesidad de memorizar números de versión exactos.

## Cuándo usarlo
Cualquier credencial, salud, financiero. [DataStore](pregunta:73) normal para tema oscuro; Keystore para secrets.

Ejemplo:
val masterKey = MasterKey.Builder(context)
    .setKeyScheme(MasterKey.KeyScheme.AES256_GCM)
    .build()

val securePrefs = EncryptedSharedPreferences.create(
    context, "secure_prefs", masterKey,
    EncryptedSharedPreferences.PrefKeyEncryptionScheme.AES256_SIV,
    EncryptedSharedPreferences.PrefValueEncryptionScheme.AES256_GCM
)


ExplicaciónKotlin:
`val masterKey = MasterKey.Builder(context)` crea builder de AndroidX Security; `context` es `Context` Android requerido. Encadenamiento `.setKeyScheme(MasterKey.KeyScheme.AES256_GCM).build()` selecciona algoritmo y construye clave maestra en Keystore. `EncryptedSharedPreferences.create(...)` es función estática con múltiples parámetros posicionales: `context`, nombre del archivo, `masterKey`, y dos enums de esquema de cifrado para claves y valores. `val securePrefs` infiere tipo `SharedPreferences` cifrado. No hay [corrutinas](pregunta:78): operación síncrona de setup. Los enums `PrefKeyEncryptionScheme` y `PrefValueEncryptionScheme` son tipos Kotlin/Java interop. El resultado se usa como `SharedPreferences` normal con `edit { putString(...) }` (extensión KTX opcional).

Consejo para entrevista:
Keystore + encrypted prefs + no logs + no hardcode. OWASP Mobile mención breve. Root bypass: defensa en profundidad, no absoluta.

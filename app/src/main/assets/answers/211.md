## Concepto
Firebase Authentication ofrece login con email/contraseña, teléfono (SMS), Google, Facebook, etc., gestionando tokens y refresco en el SDK. Los datos de perfil viven en Firebase; tu backend puede verificar ID tokens JWT.

Se integra con UI de login y reglas de seguridad en otras piezas de Firebase.

Ejemplo:
Firebase.auth.signInWithEmailAndPassword(email, pass)
    .addOnCompleteListener { task ->
        if (task.isSuccessful) navigateHome()
    }

ExplicaciónKotlin:
`Firebase.auth` es propiedad de extensión o entry point del SDK KTX que devuelve `FirebaseAuth`. `signInWithEmailAndPassword(email, pass)` recibe dos variables Kotlin (`String`, asumidas no nulas) y devuelve `Task<AuthResult>` de las Tasks API de Google Play services, no una corrutina nativa. `.addOnCompleteListener { task -> ... }` registra lambda con parámetro `task`; Kotlin infiere `Task<AuthResult>`. Dentro, `if (task.isSuccessful) navigateHome()` usa propiedad Java booleana y llama función del ámbito sin else explícito. Este patrón callback es Java-first; con Firebase KTX alternativo se puede usar `await()` dentro de corrutinas (`try { auth.signIn...().await() }`). Las variables `email` y `pass` deben validarse antes para evitar enviar strings vacíos. No hay manejo de excepciones en el snippet: fallos quedan en `task.exception`.

Consejo:
Nunca confíes solo en el cliente: valida tokens en servidor con las claves públicas de Google. Habilita verificación de email y rate limits donde aplique.

## Concepto
En Compose el estado se modela combinando tres piezas: [mutableStateOf](pregunta:49) crea un State<T> observable; [remember](pregunta:48) conserva ese State entre [recomposiciones](pregunta:47) del mismo [Composable](pregunta:46); [rememberSaveable](pregunta:58) además lo persiste en el Bundle de la pantalla para sobrevivir a cambios de configuración y a process death de la UI.

Para datos de pantalla pequeños y serializables (texto de un campo, índice de tab) basta [rememberSaveable](pregunta:58). Para datos grandes o lógica de dominio se eleva al [ViewModel](pregunta:68), que de por sí sobrevive a la rotación.

Ejemplo:
@Composable
fun LoginForm() {
    var email by rememberSaveable { mutableStateOf("") }
    var password by remember { mutableStateOf("") }

    TextField(value = email, onValueChange = { email = it })
    TextField(value = password, onValueChange = { password = it }, visualTransformation = PasswordVisualTransformation())
}

ExplicaciónKotlin:
LoginForm mezcla [rememberSaveable](pregunta:58) y [remember](pregunta:48) para distintos requisitos de persistencia. var email by [rememberSaveable](pregunta:58) { [mutableStateOf](pregunta:49)("") } usa delegación by sobre un State que sobrevive a [recomposición](pregunta:47) y a process death/cambio de configuración gracias al Saver interno de [rememberSaveable](pregunta:58). var password by [remember](pregunta:48) { [mutableStateOf](pregunta:49)("") } solo sobrevive [recomposiciones](pregunta:47), no rotación ni muerte de proceso. TextField(value = email, onValueChange = { email = it }) y el equivalente para password usan lambdas con it implícito para asignar al setter delegado. PasswordVisualTransformation() instancia un objeto de Material sin argumentos para ocultar caracteres. Los dos campos comparten el mismo patrón de estado unidireccional: value lee, onValueChange escribe. La diferencia entre [remember](pregunta:48) y [rememberSaveable](pregunta:58) es puramente de API Compose, pero la sintaxis Kotlin (by, var, lambdas) es idéntica.

Consejo:
Regla: efímero local → [remember](pregunta:48); UI pequeña que debe sobrevivir rotación → [rememberSaveable](pregunta:58); estado de pantalla/dominio → [ViewModel](pregunta:68) con [StateFlow](pregunta:86).

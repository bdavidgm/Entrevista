Concepto:
@Insert es la anotación de Room para definir métodos que insertan filas. Admite el parámetro onConflict (OnConflictStrategy.REPLACE / IGNORE / ABORT) para decidir qué hacer si ya existe una fila con la misma clave primaria. Acepta una entidad, varargs o List, y devuelve void/Long/List<Long> según rowIds.

Para "insert-or-update" sin escribir tu propio SQL, hoy es habitual usar @Upsert (Room 2.5+).

Ejemplo:
@Dao
interface UserDao {
    @Insert(onConflict = OnConflictStrategy.REPLACE)
    suspend fun upsertAll(users: List<User>)

    @Upsert
    suspend fun upsert(user: User)
}

Consejo:
"REPLACE puede borrar y reinsertar (cuidado con triggers/foreign keys cascada)". Para insertar y actualizar mejor @Upsert, más explícito.

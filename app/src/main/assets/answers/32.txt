Concepto:
Un Fragment es un componente de UI reutilizable que vive dentro de una Activity (o de otro Fragment) y tiene su propio ciclo de vida acoplado al del contenedor. Permite dividir la UI en piezas independientes que se pueden combinar en pantallas multi-panel (tablets, navigation drawer).

Su ciclo de vida es más complejo que el de Activity (View lifecycle adicional) y conviene gestionarlo con viewLifecycleOwner para no fugar referencias a la vista.

Ejemplo:
class ProfileFragment : Fragment(R.layout.fragment_profile) {
    override fun onViewCreated(view: View, savedInstanceState: Bundle?) {
        viewLifecycleOwner.lifecycleScope.launch {
            viewModel.state.collect { render(it) }
        }
    }
}

Consejo:
Distingue Fragment lifecycle del View lifecycle. Hoy el patrón habitual es single-activity + varios Fragments o pantallas Compose con Navigation.

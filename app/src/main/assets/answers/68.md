## Concepto
ViewModel es una clase de Jetpack que mantiene estado y lógica de presentación de una pantalla y sobrevive a los cambios de configuración (rotaciones, idioma, modo oscuro). Se obtiene mediante un ViewModelStore vinculado al ámbito de la Activity, Fragment o NavBackStackEntry.

Su ciclo es onCleared() cuando el componente desaparece definitivamente; con SavedStateHandle puede preservar estado tras process death.

Ejemplo:
class HomeViewModel(private val repo: Repository) : ViewModel() {
    val state: StateFlow<HomeState> = repo.observe()
        .stateIn(viewModelScope, SharingStarted.WhileSubscribed(5_000), HomeState.Loading)
}

// Fragment
private val viewModel: HomeViewModel by viewModels()

ExplicaciónKotlin:
class HomeViewModel(private val repo: Repository) : ViewModel() hereda ciclo de vida extendido de ViewModel. val state: StateFlow<HomeState> = repo.observe().stateIn(viewModelScope, SharingStarted.WhileSubscribed(5_000), HomeState.Loading) expone flujo caliente sin MutableStateFlow visible: stateIn suscribe upstream en viewModelScope. // Fragment comentario precede private val viewModel: HomeViewModel by viewModels() que es delegación KTX lazy scoped al Fragment. by viewModels() genera getter que obtiene instancia del ViewModelStore. No hay override ni suspend en el snippet, pero StateFlow y SharingStarted son generics de kotlinx.coroutines.flow. HomeState.Loading es valor inicial del enum o sealed class.

Consejo:
"Vive más que la View, menos que el proceso". Nunca guardes Context de Activity dentro de un ViewModel: usa el applicationContext o inyéctalo.

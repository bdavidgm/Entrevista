## Concepto
Un fake es una implementación de prueba de una dependencia con comportamiento simplificado pero realista (en memoria), a diferencia de un mock que verifica llamadas con Mockito/MockK. Ejemplo: FakeUserApi que devuelve listas fijas sin red.

Los fakes hacen tests más legibles y menos frágiles que mocks excesivos.

Ejemplo:
class FakeUserApi : UserApi {
    var users = listOf(UserDto("1", "Ana"))
    override suspend fun fetchUsers() = users
}

val repo = UserRepositoryImpl(FakeUserApi(), dao)


ExplicaciónKotlin:
`class FakeUserApi : UserApi` implementa interfaz con `override suspend fun fetchUsers() = users`: expresión de cuerpo único sin llaves devuelve la propiedad directamente. `var users = listOf(UserDto("1", "Ana"))` es propiedad mutable reasignable en tests; inicializada con `listOf` inmutable de un elemento. `UserDto("1", "Ana")` usa constructor primario de data class con argumentos posicionales. La interfaz `UserApi` exige `suspend`, por lo que la implementación también lo es aunque no haga I/O real. `val repo = UserRepositoryImpl(FakeUserApi(), dao)` muestra composición manual de dependencias sin framework DI. El fake sustituye comportamiento real con estado en memoria manipulable desde el test.

Consejo:
"Fake = implementación sustituta; mock = verificación de interacción". Usa fakes para repositorios y APIs; reserva mocks para bordes puntuales.

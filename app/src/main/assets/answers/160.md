## Concepto
CompositionLocal es el mecanismo de Compose para propagar valores implícitamente por el árbol sin tener que pasarlos por parámetros. Se usa para cosas transversales: tema, densidad, controladores (LocalContext, LocalLifecycleOwner) o configuraciones de UI.

Hay dos variantes: compositionLocalOf (eficiente: solo recompone lectores cuando cambia) y staticCompositionLocalOf (no se observa: recompone toda la rama bajo el Provider). Crea uno propio solo cuando varios composables anidados necesitan el mismo valor y no es práctico pasarlo por parámetro.

Ejemplo:
val LocalUser = compositionLocalOf<User?> { null }

@Composable
fun App(user: User) {
    CompositionLocalProvider(LocalUser provides user) {
        ProfileButton()
    }
}

@Composable
fun ProfileButton() {
    val user = LocalUser.current
    Text(user?.name.orEmpty())
}


ExplicaciónKotlin:
`val LocalUser = compositionLocalOf<User?> { null }` declara `ProvidableCompositionLocal` con factory default nullable; el genérico permite ausencia de usuario. `@Composable fun App(user: User)` recibe usuario no nullable. `CompositionLocalProvider(LocalUser provides user) { ProfileButton() }` usa infix `provides` para empujar valor al árbol; lambda trailing contiene hijos que leen el local. `ProfileButton()` accede con `LocalUser.current`, propiedad composable que lee valor más cercano arriba en el árbol. `user?.name.orEmpty()` encadena safe call con extensión `orEmpty()` de stdlib para `String?`. Si no hay provider, `current` lanzaría error; aquí siempre hay uno en `App`. `compositionLocalOf` recomponer solo lectores cuando cambia el valor.

Consejo:
"Para datos cross-cutting, no para reemplazar parámetros". Abusar de CompositionLocal oculta dependencias y dificulta tests.

Concepto:
Un Looper es un bucle de mensajes asociado a un hilo, y un Handler es la API para encolar Runnables o Messages en ese bucle. Forman la base del modelo de concurrencia clásico de Android: el hilo principal tiene un Looper que procesa los mensajes de UI y entrada.

Hoy en día se prefieren coroutines + dispatchers, pero Handler/Looper aún aparecen en APIs como postDelayed, en el patrón main-thread executor y en código legado.

Ejemplo:
val handler = Handler(Looper.getMainLooper())
handler.postDelayed({
    textView.text = "Hola"
}, 1_000)

Consejo:
"Looper = bucle de mensajes del hilo; Handler = la cola para postear". En código moderno prefiere viewModelScope + Dispatchers.Main.

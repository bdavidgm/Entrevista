## Concepto
El parámetro key en [LazyColumn](pregunta:52)/[LazyRow](pregunta:52) le dice a Compose cómo identificar cada ítem para mantener su estado interno (estado de scroll, animaciones, foco) al reordenar, insertar o eliminar elementos de la lista. Sin key se usa la posición, lo que provoca [recomposiciones](pregunta:47) innecesarias y "saltos" visuales al cambiar la lista.

La key debe ser estable y única por ítem (típicamente el id del dominio).

Ejemplo:
LazyColumn {
    items(users, key = { it.id }) { user ->
        UserRow(user)
    }
}


ExplicaciónKotlin:
El bloque `LazyColumn { }` es un DSL de Compose: la lambda receptora (`LazyListScope`) permite invocar `items` como si fuera una función de nivel superior dentro del scope. La sobrecarga `items(users, key = { it.id }) { user -> ... }` recibe la lista, un parámetro nombrado `key` con lambda `(User) -> Any` y un content lambda `(User) -> Unit` al final (trailing lambda). `{ it.id }` usa `it` implícito como único parámetro de la lambda de key; el valor devuelto debe ser estable y comparable. El lambda de contenido recibe cada `user` tipado por inferencia a partir del tipo de la lista. `UserRow(user)` es invocación de un `@Composable` hijo; Compose infiere el tipo de retorno `Unit`. Sin `key`, Compose identificaría ítems por índice, lo que Kotlin no controla pero afecta [recomposición](pregunta:47).

Consejo:
"key estable por ítem evita reusar la posición". Imprescindible para animateItemPlacement y para conservar el estado de hijos editables al reordenar.

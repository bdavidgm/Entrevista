## Concepto
El back stack (pila de retroceso) es la estructura LIFO que el sistema de navegación de Android usa para gestionar destinos visitados: al pulsar Atrás se destruye o pausa el destino actual y se restaura el anterior.

## Explicación detallada
Cada Task (tarea) suele asociarse a una app en el recents y contiene una o más Activities apiladas. Al iniciar B desde A, A pasa a onPause/onStop y B a onStart/onResume; B queda encima de la pila.

Comportamiento del botón Back:
- Si hay más de un destino en la pila de la task actual, se hace finish() del superior y se reanuda el inferior.
- Si es la raíz de la task, puede ir a la pantalla anterior del launcher o cerrar la app según versión y gestos.

[Navigation Component](pregunta:75):
- navController.navigate() empuja destinos al NavController back stack.
- popUpTo, popUpToInclusive y launchSingleTop controlan si se reutiliza o se limpia la pila.
- [Deep link](pregunta:119)s pueden reconstruir grafos con back stack definido en XML o [Kotlin DSL](pregunta:112).

Fragments:
- FragmentManager mantiene su propia pila de transacciones (add, replace, addToBackStack).
- replace sin addToBackStack no permite volver al [fragment](pregunta:32) anterior con Back.

Compose Navigation:
- NavHostController gestiona la pila de rutas composables de forma análoga.

[launchMode](pregunta:199) de [Activity](pregunta:31) (standard, [singleTop](pregunta:199), [singleTask](pregunta:199), singleInstance) modifica cómo se apilan instancias entre tasks.

## Cuándo usarlo
Al diseñar flujos de login, onboarding, wizard multipaso y salida de pantallas modales. Configurar popUpTo al navegar tras login evita volver a credenciales con Back.

Ejemplo:
// Navigation Component
navController.navigate(R.id.home) {
    popUpTo(R.id.login) { inclusive = true }
    launchSingleTop = true
}

// Fragment
supportFragmentManager.commit {
    replace(R.id.container, DetailFragment())
    addToBackStack("detail")
}


ExplicaciónKotlin:
`navController.navigate(R.id.home) { popUpTo(R.id.login) { inclusive = true }; launchSingleTop = true }` — lambda trailing de `NavOptionsBuilder`: `popUpTo` elimina destinos hasta login inclusive; `launchSingleTop = true` reutiliza instancia superior si existe. `R.id.home` e `R.id.login` son IDs generados del grafo Navigation. [Fragment](pregunta:32) snippet: `supportFragmentManager.commit { replace(R.id.container, DetailFragment()); addToBackStack("detail") }` usa KTX `commit` lambda DSL; `DetailFragment()` constructor Kotlin/Java; `addToBackStack` con string tag. Son APIs Android de navegación invocadas desde Kotlin con lambdas y named arguments en NavOptions.

Consejo para entrevista:
Dibuja la pila A -> B -> C y qué pasa con Back. Diferencia task del sistema vs NavController back stack. Menciona popUpTo tras autenticación como caso clásico.

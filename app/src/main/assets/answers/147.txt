Concepto:
Un test de integración verifica que varias piezas reales o casi reales cooperan: por ejemplo Room + Repository, navegación + ViewModel, o un módulo contra un servidor de pruebas. No sustituye tests unitarios rápidos ni tests de UI end-to-end.

Suelen vivir en androidTest, ser más lentos y usar Hilt test modules o bases de datos en memoria.

Ejemplo:
@Test
fun `dao y repo devuelven mismos datos`() = runTest {
    val db = Room.inMemoryDatabaseBuilder(context, AppDb::class.java).build()
    val repo = UserRepositoryImpl(db.dao(), FakeApi())
    repo.refresh()
    assertEquals(1, repo.observeUsers().first().size)
}

Consejo:
Distingue unit (lógica aislada), integration (varias capas) y UI (Espresso/Compose). Los de integración detectan fallos de wiring y migraciones.

Concepto:
Un Foreground Service (FGS) es un Service que muestra una notificación persistente mientras ejecuta trabajo perceptible para el usuario (música, navegación, llamada, sync visible), con prioridad alta y menos probabilidad de ser matado.

Explicación detallada:
Desde Android 8 debe llamarse startForegroundService(); en pocos segundos startForeground(id, notification) o el sistema detiene la app (ANR/crash).

Android 10+: requiere permiso FOREGROUND_SERVICE. Android 14+: declarar foregroundServiceType en manifest (location, camera, mediaPlayback, dataSync, etc.) y permisos asociados según tipo.

Notificación: canal obligatorio, ongoing, no dismissable hasta stopForeground.

stopForeground(STOP_FOREGROUND_REMOVE) y stopSelf al terminar.

Cuándo usarlo:
- Reproducción audio/video en background.
- Navegación turn-by-turn activa.
- Fitness tracking con ubicación en uso.
- Compartir pantalla o llamadas VoIP.

No usar para: sync silencioso (WorkManager), analytics, prefetch — viola políticas y restricciones.

Alternativa: WorkManager con setForeground() para trabajo largo que debe mostrar progreso ocasional.

Ejemplo:
class PlaybackService : Service() {
    override fun onStartCommand(intent: Intent?, flags: Int, startId: Int): Int {
        val notification = buildNotification()
        if (Build.VERSION.SDK_INT >= Build.VERSION_CODES.Q) {
            startForeground(NOTIF_ID, notification, ServiceInfo.FOREGROUND_SERVICE_TYPE_MEDIA_PLAYBACK)
        } else {
            startForeground(NOTIF_ID, notification)
        }
        startPlayback()
        return START_STICKY
    }
}

// Manifest
// <service android:foregroundServiceType="mediaPlayback" ... />
// <uses-permission android:name="android.permission.FOREGROUND_SERVICE_MEDIA_PLAYBACK"/>

Consejo para entrevista:
Menciona notificación obligatoria, tipos Android 14, permisos FGS y que Play audita abuso. Contrasta con background service prohibido. Ejemplo concreto: mediaPlayback o location.

## Concepto
DiffUtil compara dos listas y calcula un conjunto mínimo de insert/remove/move para actualizar [RecyclerView](pregunta:203) sin recargar todo. [ListAdapter](pregunta:203) en Kotlin delega en AsyncListDiffer para aplicar el resultado en el hilo principal.

Mejora rendimiento y animaciones de cambio.

Ejemplo:
private object UserDiff : DiffUtil.ItemCallback<User>() {
    override fun areItemsTheSame(a: User, b: User) = a.id == b.id
    override fun areContentsTheSame(a: User, b: User) = a == b
}

class UserAdapter : ListAdapter<User, VH>(UserDiff)

ExplicaciónKotlin:
`private object UserDiff : DiffUtil.ItemCallback<User>()` declara un singleton con `object`: una única instancia thread-safe del callback, visibilidad restringida con `private`. Hereda de `ItemCallback<User>()` genérico en el tipo `User`. `override fun areItemsTheSame(a: User, b: User) = a.id == b.id` usa expression body: compara identidad de ítem por `id`, devolviendo `Boolean`; los parámetros tienen tipos explícitos no anulables. `override fun areContentsTheSame(a: User, b: User) = a == b` delega en `equals` estructural de `User` — en Kotlin conviene que `User` sea `data class` para generar `equals`/`hashCode` automáticos. `class UserAdapter : ListAdapter<User, VH>(UserDiff)` pasa el `object` al constructor del adaptador; `ListAdapter` internamente usa `AsyncListDiffer` con ese callback. El patrón combina generics, object declaration y funciones de una línea típicas de Kotlin para [RecyclerView](pregunta:203).

Consejo:
Si solo cambia un campo, implementa getChangePayload para actualizar parcialmente el [ViewHolder](pregunta:96) y evitar rebind completo.

Concepto:
Una migración en Room define cómo evolucionar el esquema de la base de datos de una versión a la siguiente sin perder datos. Cada vez que cambias entidades (nuevas tablas, columnas, índices) debes subir la versión y registrar una Migration que ejecute el SQL necesario.

Sin migración (o con fallbackToDestructiveMigration), Room borra y recrea la BBDD: aceptable en debug, inaceptable en producción.

Ejemplo:
val MIGRATION_1_2 = object : Migration(1, 2) {
    override fun migrate(db: SupportSQLiteDatabase) {
        db.execSQL("ALTER TABLE User ADD COLUMN avatarUrl TEXT")
    }
}

Room.databaseBuilder(context, AppDb::class.java, "app.db")
    .addMigrations(MIGRATION_1_2)
    .build()

Consejo:
Activa exportSchema = true y commitea los JSON del esquema; permite migraciones automáticas en versiones recientes de Room y revisión en CI.

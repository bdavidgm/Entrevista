## Concepto
RecyclerView recicla filas fuera de pantalla: el Adapter crea ViewHolder y enlaza datos en onBindViewHolder; el LayoutManager coloca ítems (Linear, Grid, Staggered). ListAdapter + DiffUtil automatiza animaciones y diffs.

Separar ViewHolder reduce trabajo en scroll.

Ejemplo:
class UserAdapter : ListAdapter<User, UserVH>(UserDiff) {
    override fun onCreateViewHolder(parent: ViewGroup, viewType: Int) =
        UserVH(ItemUserBinding.inflate(LayoutInflater.from(parent.context), parent, false))
    override fun onBindViewHolder(holder: UserVH, position: Int) {
        holder.bind(getItem(position))
    }
}

ExplicaciónKotlin:
`class UserAdapter : ListAdapter<User, UserVH>(UserDiff)` declara una subclase genérica: `User` es el tipo de ítem y `UserVH` el ViewHolder; el constructor primario pasa `UserDiff` al padre `ListAdapter`. `override fun onCreateViewHolder(parent: ViewGroup, viewType: Int)` redefine el método abstracto; el cuerpo es expresión única (`=`) que devuelve `UserVH(...)`. `ItemUserBinding.inflate(LayoutInflater.from(parent.context), parent, false)` usa View Binding generado: `LayoutInflater.from` es estático Java, `parent.context` obtiene el `Context`, y `false` indica que no se adjunta aún al parent. `override fun onBindViewHolder(holder: UserVH, position: Int)` llama `holder.bind(getItem(position))`: `getItem` es método heredado de `ListAdapter` que accede a la lista diffrada internamente. El estilo expression body (`=`) y generics `<User, UserVH>` son idiomas Kotlin puro sobre la API RecyclerView Java.

Consejo:
onBindViewHolder debe ser barato: usa payloads, evita allocations y mueve lógica pesada a background. DiffUtil calcula cambios mínimos.

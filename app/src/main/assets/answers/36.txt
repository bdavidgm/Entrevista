Concepto:
Un Service es un componente sin UI que realiza operaciones de larga duración en segundo plano o expone funcionalidad a otros componentes. Existen tres tipos: started (se inicia con startService), bound (se enlaza con bindService) y foreground (visible al usuario mediante notificación).

A partir de Android 8 hay fuertes restricciones para servicios en background; para trabajo diferible suele preferirse WorkManager, y para tareas visibles (música, navegación, llamada) un Foreground Service.

Ejemplo:
class MusicService : Service() {
    override fun onStartCommand(intent: Intent?, flags: Int, startId: Int): Int {
        startForeground(NOTIF_ID, buildNotification())
        playMusic()
        return START_STICKY
    }
    override fun onBind(intent: Intent?): IBinder? = null
}

Consejo:
"Hoy usa WorkManager para trabajo diferible y Foreground Service solo si hay que ser visible". Conoce los tipos y los límites de background a partir de Android 8.

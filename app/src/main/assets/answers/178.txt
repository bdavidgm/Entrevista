Concepto:
ContentResolver es la fachada para leer/escribir datos expuestos por ContentProviders (propios o del sistema: Contacts, MediaStore, Calendar). Métodos típicos: query, insert, update, delete, openInputStream/openFileDescriptor.

Unifica acceso CRUD sobre URIs content://.

Ejemplo:
val cursor = contentResolver.query(
    MediaStore.Images.Media.EXTERNAL_CONTENT_URI,
    arrayOf(MediaStore.Images.Media.DISPLAY_NAME),
    null, null, null
)

ExplicaciónKotlin:
`contentResolver` es una propiedad de `Context` que expone el `ContentResolver` del proceso. La llamada `query(...)` devuelve un `Cursor?`, que en Kotlin debe tratarse como anulable. El segundo argumento `arrayOf(MediaStore.Images.Media.DISPLAY_NAME)` construye un `Array<String>` con la función estándar `arrayOf`, equivalente a varargs Java pero explícito en tipo. Los tres `null` posteriores (`selection`, `selectionArgs`, `sortOrder`) usan el literal `null` de Kotlin para omitir filtros y ordenación; el compilador los infiere como tipos anulables compatibles con la firma Java. `MediaStore.Images.Media.EXTERNAL_CONTENT_URI` es una constante `Uri` estática del SDK. El resultado debería consumirse con `use { cursor -> ... }` o cerrarse manualmente para aprovechar el contrato `Closeable` y evitar fugas; el ejemplo muestra solo la apertura de la consulta CRUD sobre un ContentProvider del sistema.

Consejo:
Cierra cursores con use. En Android 10+ el almacenamiento con scopes puede requerir SAF o permisos de medios específicos según el tipo de archivo.

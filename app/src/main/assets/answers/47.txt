Concepto:
La recomposición es el proceso por el que el runtime de Compose vuelve a ejecutar las funciones Composable cuyo estado leído ha cambiado, actualizando la UI de forma incremental. Solo se recompone el subárbol afectado; si los parámetros son estables y no cambiaron, Compose puede saltar la ejecución (skipping).

Para evitar recomposiciones innecesarias: colocar bien el estado (state hoisting), usar remember y derivedStateOf, pasar tipos estables y poner key en listas.

Ejemplo:
@Composable
fun Counter() {
    var count by remember { mutableStateOf(0) }
    Button(onClick = { count++ }) { Text("Sumar") }
    Text("Clicks: $count")  // solo este Text se recompone
}

Consejo:
Distingue las 3 fases (composition / layout / draw). Menciona "tipos estables" y herramientas como Layout Inspector con recomposition counts para diagnosticar problemas.

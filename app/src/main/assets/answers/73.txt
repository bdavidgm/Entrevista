Concepto:
DataStore es la API moderna para guardar pequeños datos clave-valor o tipados, sustituta de SharedPreferences. Es asíncrona (basada en coroutines y Flow), thread-safe y soporta dos variantes: Preferences DataStore (clave-valor parecido a Prefs) y Proto DataStore (esquema tipado con protobuf).

Sus ventajas frente a SharedPreferences: no bloquea el hilo principal, ofrece manejo de errores y propaga los cambios como Flow.

Ejemplo:
val Context.dataStore by preferencesDataStore("settings")
val DARK = booleanPreferencesKey("dark")

class SettingsRepository(private val context: Context) {
    val isDark: Flow<Boolean> = context.dataStore.data.map { it[DARK] ?: false }
    suspend fun setDark(value: Boolean) {
        context.dataStore.edit { it[DARK] = value }
    }
}

ExplicaciónKotlin:
val Context.dataStore by preferencesDataStore("settings") es propiedad delegada a nivel de archivo (extension property): extiende Context con dataStore lazy singleton por nombre "settings". val DARK = booleanPreferencesKey("dark") declara clave tipada Preferences.Key<Boolean>. class SettingsRepository(private val context: Context) almacena context para acceder a dataStore. val isDark: Flow<Boolean> = context.dataStore.data.map { it[DARK] ?: false } encadena .data (Flow<Preferences>) con map: lambda { it[DARK] ?: false } usa operador [] sobrecargado y Elvis ?: false si la clave falta. suspend fun setDark(value: Boolean) { context.dataStore.edit { it[DARK] = value } } usa edit suspend con lambda mutable sobre Preferences.Editor implícito. Combina extension properties, delegación by, Flow, suspend, generics y operador Elvis.

Consejo:
"Sustituye SharedPreferences". Para esquemas complejos prefiere Proto DataStore por la validación de tipos.

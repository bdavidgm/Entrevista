## Concepto
Hilt es el framework oficial de inyección de dependencias para Android, construido sobre [Dagger 2](pregunta:269). Reduce el boilerplate generando los componentes asociados al ciclo de vida de [Application](pregunta:128), [Activity](pregunta:31), [Fragment](pregunta:32), [ViewModel](pregunta:68) y [Service](pregunta:36), y proporciona scopes preconfigurados (@Singleton, @ActivityScoped, @ViewModelScoped).

Anotaciones clave: @HiltAndroidApp en [Application](pregunta:128), @AndroidEntryPoint en componentes Android, @Module + @InstallIn para proveer dependencias, @Inject en constructores y [@HiltViewModel](pregunta:141) para ViewModels.

Ejemplo:
@HiltAndroidApp
class MyApp : Application()

@Module
@InstallIn(SingletonComponent::class)
object NetworkModule {
    @Provides @Singleton
    fun provideApi(): Api = Retrofit.Builder()...build().create(Api::class.java)
}

@HiltViewModel
class HomeViewModel @Inject constructor(private val api: Api) : ViewModel()

ExplicaciónKotlin:
@HiltAndroidApp class MyApp : [Application](pregunta:128)() marca [Application](pregunta:128) para generación de componente raíz Hilt. @Module @InstallIn(SingletonComponent::class) object NetworkModule declara módulo [Dagger](pregunta:269) como object singleton Kotlin (no class instanciable). @Provides @Singleton fun provideApi(): Api = [Retrofit](pregunta:207).Builder()...build().create(Api::class.java) es función @Provides: create(Api::class.java) usa class reference Java Class y genérico de [Retrofit](pregunta:207). [@HiltViewModel](pregunta:141) class HomeViewModel @Inject constructor(private val api: Api) : [ViewModel](pregunta:68)() conecta [ViewModel](pregunta:68) con grafo DI. object NetworkModule evita estado; @InstallIn vincula módulo al componente Singleton. Las anotaciones son metadatos compile-time procesados por KAPT/KSP. Api::class.java interop Java-Kotlin para [Retrofit](pregunta:207) proxy.

Consejo:
"Hilt = [Dagger](pregunta:269) fácil con scopes Android". Para apps pequeñas o multiplataforma considera [Koin](pregunta:214), que es más liviano pero resuelve en runtime.

## Concepto
dp (density-independent pixels) y sp (scale-independent pixels) son unidades de medida que se adaptan a la densidad de la pantalla del dispositivo. 1 dp equivale a 1 píxel en una pantalla mdpi (160 dpi); en pantallas de mayor densidad el sistema escala automáticamente.

dp se usa para dimensiones (paddings, márgenes, tamaños), y sp se usa para texto, porque además respeta el tamaño de fuente configurado por el usuario en [accesibilidad](pregunta:106).

Ejemplo:
<TextView
    android:textSize="16sp"
    android:padding="8dp"
    android:layout_marginTop="12dp"/>

// Compose
Text("Hola", fontSize = 16.sp)
Spacer(Modifier.height(12.dp))

ExplicaciónKotlin:
El fragmento XML muestra atributos `android:textSize`, `android:padding` y `android:layout_marginTop` con unidades `sp` y `dp`; no es sintaxis Kotlin sino recursos de layout referenciados desde código Kotlin vía inflado. La sección Compose sí es Kotlin: `Text("Hola", fontSize = 16.sp)` llama a una función [composable](pregunta:46) con argumento nombrado `fontSize`; `16.sp` es una extension property que convierte un número en `TextUnit` con escala tipográfica. `Spacer(Modifier.height(12.dp))` usa otro [composable](pregunta:46); `12.dp` convierte un Int en unidad de densidad independiente mediante extension del paquete Compose. `Modifier.height(...)` encadena el modificador de layout. Estas extensiones (`sp`, `dp`) son API idiomática de Jetpack Compose escrita en Kotlin, reemplazando atributos XML por funciones y tipos seguros en compile-time.

Consejo:
"dp para tamaños, sp para texto". Mencionar [accesibilidad](pregunta:106): si fuerzas textos en dp, los usuarios con texto grande no ven bien tu app.

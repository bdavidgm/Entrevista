Concepto:
El bloque init es un inicializador de la clase que se ejecuta cada vez que se crea una instancia, justo después de asignar las propiedades del constructor primario. Puede haber varios; se ejecutan en el orden de declaración intercalados con las propiedades de la clase.

Sirve para validar parámetros, calcular valores derivados o registrar callbacks que requieren más lógica de la que cabe en la declaración de la propiedad.

Ejemplo:
class User(val email: String) {
    val domain: String

    init {
        require(email.contains("@")) { "Email inválido" }
        domain = email.substringAfter("@")
    }
}

ExplicaciónKotlin:
`class User(val email: String)` declara una clase con constructor primario: `email` es una propiedad `val` inicializada al construir la instancia. `val domain: String` declara una propiedad que debe inicializarse antes de terminar la construcción; aquí no tiene valor en el constructor, así que se asigna en el bloque `init`. El bloque `init { ... }` se ejecuta en cada instancia, en orden de aparición junto con otras propiedades e inits. `require(email.contains("@")) { "Email inválido" }` valida una precondición y lanza `IllegalArgumentException` con mensaje lazy si falla. `domain = email.substringAfter("@")` usa una función de extensión/`String` estándar para derivar un valor y asignarlo a la propiedad. Tras el init, `domain` queda inicializada y no puede reasignarse por ser `val`.

Consejo:
Recuerda que init se ejecuta en cada instancia y que el orden importa: propiedades e inits se mezclan según aparezcan en el código.

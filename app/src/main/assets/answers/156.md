## Concepto
UI adaptativa (tablet / pantallas grandes) reutiliza lógica pero cambia layout según ancho o ventana: paneles maestro-detalle, navigation rail, columnas extra. Jetpack WindowManager y breakpoints (Material3) ayudan a clasificar el espacio (compact, medium, expanded).

Compose facilita el when(windowSizeClass) { ... } sin duplicar ViewModels.

Ejemplo:
@Composable
fun HomeRoute(window: WindowSizeClass) {
    when (window.widthSizeClass) {
        WindowWidthSizeClass.Expanded -> TwoPane(master = { List() }, detail = { Reader() })
        else -> SinglePaneNav()
    }
}


ExplicaciónKotlin:
`@Composable fun HomeRoute(window: WindowSizeClass)` recibe clasificación de ventana como parámetro tipado. `when (window.widthSizeClass) { ... }` es expresión `when` sobre propiedad; debe ser exhaustiva o incluir `else`. `WindowWidthSizeClass.Expanded -> TwoPane(...)` compara con singleton/objeto de Material3 WindowSizeClass. `TwoPane(master = { List() }, detail = { Reader() })` usa parámetros nombrados con lambdas trailing `@Composable` como slots. `else -> SinglePaneNav()` cubre compact y medium. `when` devuelve `Unit` implícito al ser usado como statement dentro del composable. Las lambdas `{ List() }` son funciones composables sin parámetros pasadas donde se espera `@Composable () -> Unit`.

Consejo:
Prueba en emulador tablet y en modo escritorio (freeform). No uses solo “sw600dp”: considera ventanas redimensionables y foldables.

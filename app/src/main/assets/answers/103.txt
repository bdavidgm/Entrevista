Concepto:
Android Keystore es el almacén seguro de claves criptográficas respaldado por hardware (TEE/StrongBox cuando disponible) que impide exportar la clave privada en claro. Se usa para cifrar datos locales, firmar, TLS client certs y autenticación biométrica vinculada.

Explicación detallada:
KeyGenParameterSpec genera clave AES o RSA dentro del Keystore. Las operaciones cifran/descifran sin exponer clave a la app. setUserAuthenticationRequired exige biometría o PIN reciente.

EncryptedSharedPreferences usa claves del Keystore. BiometricPrompt + CryptoObject descifra solo tras huella válida.

Limitaciones: emuladores, dispositivos sin TEE, backup. Rotación de claves y invalidación si cambian huellas registradas (setInvalidatedByBiometricEnrollment).

En el contexto de proyectos Android reales, combina esta idea con las guías oficiales de Android Developers y con métricas de calidad (crashes, ANR, Vitals) cuando aplique. En code review, pregunta siempre por ciclo de vida, testabilidad y si la solución escala cuando el equipo crece.

Cuándo usarlo:
Secretos locales, PIN de app, cifrado de archivos sensibles. No para almacenar contraseña usuario en claro; hash en servidor.

Ejemplo:
val keyGenerator = KeyGenerator.getInstance(
    KeyProperties.KEY_ALGORITHM_AES, "AndroidKeyStore"
)
keyGenerator.init(
    KeyGenParameterSpec.Builder(
        "app_master_key",
        KeyProperties.PURPOSE_ENCRYPT or KeyProperties.PURPOSE_DECRYPT
    )
        .setBlockModes(KeyProperties.BLOCK_MODE_GCM)
        .setEncryptionPaddings(KeyProperties.ENCRYPTION_PADDING_NONE)
        .build()
)


ExplicaciónKotlin:
`KeyGenerator.getInstance(...)` obtiene instancia de factory criptográfica con algoritmo `"AES"` y provider `"AndroidKeyStore"`. `keyGenerator.init(...)` recibe `KeyGenParameterSpec` construido con builder. `KeyGenParameterSpec.Builder("app_master_key", KeyProperties.PURPOSE_ENCRYPT or KeyProperties.PURPOSE_DECRYPT)` usa nombre alias string y flags combinados con `or` a nivel de bits sobre constantes enteras. `.setBlockModes(KeyProperties.BLOCK_MODE_GCM)` y `.setEncryptionPaddings(KeyProperties.ENCRYPTION_PADDING_NONE)` configuran parámetros AES-GCM. `.build()` produce spec inmutable pasado a `init`. No hay corrutinas; APIs java.security desde Kotlin con interop Java directo. Variables `val keyGenerator` con inferencia de tipo.

Consejo para entrevista:
Hardware-backed, no exportable. Contrasta con guardar AES key en prefs (inseguro). Seguimiento: EncryptedSharedPreferences.

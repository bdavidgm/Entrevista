Concepto:
Un Intent explícito nombra directamente el componente destino (paquete + clase) o ComponentName, usado típicamente para navegación interna dentro de tu aplicación. Un Intent implícito declara una acción abstracta (ACTION_SEND, ACTION_VIEW) y deja que el sistema resuelva qué app o componente puede manejarla según intent-filters del Manifest.

Explicación detallada:
Explícito: Intent(context, TargetActivity::class.java) — predecible, solo tu app si no exportas la activity. Implícito: Intent(Intent.ACTION_SEND).setType("text/plain") — puede mostrar chooser con varias apps. El PackageManager queryIntentActivities verifica handlers.

Intent-filters en Manifest declaran action, category (DEFAULT, BROWSABLE), data scheme/host para deep links. Sin match, ActivityNotFoundException. Chooser Intent.createChooser da control al usuario.

En entrevistas contrastan cuándo usar cada uno: interno = explícito; integración sistema/compartir = implícito. Seguridad: intents implícitos entrantes pueden ser spoofed; valida datos. Android 11+ package visibility requiere queries en manifest para ver otras apps.

Cuándo usarlo:
Explícito: flujos propios, notificaciones a pantallas internas con PendingIntent explícito. Implícito: compartir, abrir PDF, mailto, mapas, cámara si no usas contracts modernos.

Ejemplo:
// Explícito
startActivity(Intent(this, SettingsActivity::class.java))

// Implícito con chooser
val send = Intent(Intent.ACTION_SEND).apply {
    type = "text/plain"
    putExtra(Intent.EXTRA_TEXT, shareMessage)
}
startActivity(Intent.createChooser(send, getString(R.string.share_via)))

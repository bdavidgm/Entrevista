## Concepto
Scaffold es un [Composable](pregunta:46) de Material que ofrece el esqueleto de una pantalla con slots predefinidos: topBar, bottomBar, floatingActionButton, snackbarHost y contenido principal. Gestiona automáticamente los insets y el espaciado para que el contenido no quede tapado por las barras.

Es el punto de partida estándar para pantallas con TopAppBar, FAB o snackbars.

Ejemplo:
@Composable
fun HomeScreen() {
    val snackbarHostState = remember { SnackbarHostState() }
    Scaffold(
        topBar = { TopAppBar(title = { Text("Inicio") }) },
        snackbarHost = { SnackbarHost(snackbarHostState) },
        floatingActionButton = { FloatingActionButton(onClick = { }) { } }
    ) { padding ->
        LazyColumn(contentPadding = padding) {
            items(users) { UserRow(it) }
        }
    }
}

ExplicaciónKotlin:
HomeScreen declara val snackbarHostState = [remember](pregunta:48) { SnackbarHostState() } para conservar el host entre [recomposiciones](pregunta:47). Scaffold usa parámetros nombrados topBar, snackbarHost, floatingActionButton y trailing lambda { padding -> ... } para el contenido. Cada slot (topBar, etc.) es una lambda [@Composable](pregunta:46) sin nombre explícito. TopAppBar(title = { Text("Inicio") }) anida otro [composable](pregunta:46) en el parámetro title. FloatingActionButton(onClick = { }) usa lambda vacía con Unit implícito. El lambda { padding -> [LazyColumn](pregunta:52)(contentPadding = padding) { items(users) { UserRow(it) } } } recibe padding: PaddingValues del Scaffold y lo pasa a [LazyColumn](pregunta:52). items(users) { UserRow(it) } usa it como elemento de users en el lambda de items. La estructura demuestra DSL anidado, lambdas composables y parámetros con valores por defecto omitidos.

Consejo:
"Aplica siempre el paddingValues del Scaffold al contenido", si no, queda tapado por la TopAppBar.

Concepto:
Un recompose scope es la unidad mínima que el runtime de Compose puede invalidar y volver a ejecutar de forma independiente. Cada vez que un Composable lee un State, el runtime asocia esa lectura al scope actual; cuando el State cambia, solo se reejecutan los scopes que lo leían.

Conocer la noción ayuda a entender por qué dividir un Composable grande en varios pequeños puede ayudar al skipping y reducir recomposiciones.

Ejemplo:
@Composable
fun Screen() {
    var count by remember { mutableStateOf(0) }
    Button(onClick = { count++ }) { Text("Sumar") }
    Counter(count) // este es el único scope que se invalida cuando count cambia
}

@Composable
fun Counter(count: Int) {
    Text("Clicks: $count")
}

Consejo:
"Cada función @Composable es (idealmente) un scope". Aislar lecturas frecuentes en pequeños Composables ayuda al runtime a recomponer menos.

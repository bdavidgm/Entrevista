Concepto:
El ciclo de vida de una Activity está compuesto por callbacks que indican su estado: onCreate (creada, antes de mostrarse), onStart (visible), onResume (en primer plano e interactiva), onPause (parcialmente tapada, dejará de recibir input), onStop (totalmente oculta), onDestroy (se libera) y onRestart (al volver de onStop a onStart).

Conocer este orden es esencial para liberar recursos en onStop/onDestroy, registrar/desregistrar listeners en onStart/onStop y manejar cambios de configuración.

Ejemplo:
class MainActivity : AppCompatActivity() {
    override fun onStart()  { super.onStart();  sensor.register() }
    override fun onStop()   { sensor.unregister(); super.onStop() }
    override fun onDestroy() { binding = null; super.onDestroy() }
}

ExplicaciónKotlin:
`class MainActivity : AppCompatActivity()` hereda de la Activity de AndroidX. Cada callback usa `override fun` para sobrescribir métodos del ciclo de vida. En `onStart`, `super.onStart()` se invoca antes del registro del sensor: convención de llamar al super primero al entrar en un estado. En `onStop`, el orden se invierte: `sensor.unregister()` antes de `super.onStop()` para liberar recursos mientras la Activity aún es válida. `onDestroy` asigna `binding = null`, lo que implica que `binding` es probablemente `lateinit var` o nullable, liberando la referencia a la vista; luego llama a `super.onDestroy()`. Las llaves compactas en una línea son estilo kotliniano para callbacks cortos. El ejemplo muestra cómo Kotlin expresa orden de operaciones en lifecycle con código conciso.

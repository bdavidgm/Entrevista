## Concepto
Un smart cast es el conversión automática de tipos que el compilador aplica cuando puede demostrar, por análisis de flujo, que un valor pertenece a un tipo más concreto dentro de un bloque. Evita escribir casts manuales tras una comprobación con is o != null.

Solo funciona si el valor no puede cambiar entre la comprobación y el uso: aplica a val locales y a propiedades privadas/no abiertas. En var compartidas (multihilo) no es seguro y el compilador lo prohíbe.

Ejemplo:
fun describe(x: Any) {
    if (x is String) {
        println(x.length) // smart cast a String
    }
}

fun printSize(list: List<Int>?) {
    if (list != null) {
        println(list.size) // smart cast a List<Int>
    }
}


ExplicaciónKotlin:
`fun describe(x: Any)` recibe cualquier tipo. `if (x is String)` — type check que habilita smart cast: dentro del bloque `x` se trata como `String` sin cast manual, el compilador infiere por flujo de control. `x.length` accede a propiedad String. `fun printSize(list: List<Int>?)` acepta lista nullable de enteros. `if (list != null)` — null check smart cast: dentro del bloque `list` es `List<Int>` non-null. `list.size` es propiedad de colección. Smart cast falla si `list` es `var` mutable compartida o propiedad open con getter custom; aquí parámetro val local es seguro. No se usa `as` explícito gracias al análisis de flujo del compilador Kotlin.

Consejo:
Conoce los casos en que NO aplica: var públicas, open properties, custom getters. En esos casos, asigna a una val local: val current = property y opera sobre ella.

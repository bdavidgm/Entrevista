Concepto:
En Compose, el patrón ViewModel mantiene el estado y la lógica fuera de las funciones Composable. El ViewModel expone un StateFlow<UiState> y métodos para procesar intents; la pantalla colecta el estado con collectAsStateWithLifecycle y delega los eventos en métodos del ViewModel.

Esto permite que la UI sea declarativa y pura (UI = f(state)), mientras el ViewModel sobrevive a la rotación y centraliza el side effect / coroutine management.

Ejemplo:
@HiltViewModel
class HomeViewModel @Inject constructor(
    private val repo: HomeRepository
) : ViewModel() {
    val state: StateFlow<HomeUiState> = repo.observe()
        .map(HomeUiState::Success)
        .stateIn(viewModelScope, SharingStarted.WhileSubscribed(5_000), HomeUiState.Loading)
}

@Composable
fun HomeScreen(vm: HomeViewModel = hiltViewModel()) {
    val state by vm.state.collectAsStateWithLifecycle()
    HomeContent(state)
}

Consejo:
"State down, events up". Usa collectAsStateWithLifecycle para colectar con conciencia del ciclo de vida; evita lógica en el Composable.

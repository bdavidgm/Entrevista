Concepto:
Koin es un framework de DI en Kotlin puro: módulos declaran single/factory/viewModel y se arranca con startKoin { modules(...) }. Resuelve dependencias en runtime (sin anotaciones ni procesador de código como Dagger/Hilt).

Ventaja: simplicidad y multiplataforma. Desventaja: errores de grafo en runtime y algo menos de optimización que Dagger.

Ejemplo:
val appModule = module {
    single { Retrofit.Builder()...build().create(Api::class.java) }
    viewModel { HomeViewModel(get()) }
}

startKoin { androidContext(this@App); modules(appModule) }

Consejo:
"Hilt = compile-time + Android integrado; Koin = runtime + ligero". Para apps grandes muchos equipos prefieren Hilt por seguridad de grafo.

Concepto:
Un evento de UI one-shot es una acción que debe ejecutarse una sola vez (navegar, mostrar Snackbar, cerrar pantalla) y no formar parte del estado persistente. Se modela con un canal aparte (Channel + receiveAsFlow o SharedFlow con replay=0), no con StateFlow, para evitar reemisión al rotar.

El consumidor (Compose, Fragment) los recolecta solo cuando está STARTED y los ejecuta como side effects.

Ejemplo:
class LoginViewModel : ViewModel() {
    private val _events = Channel<LoginEvent>(Channel.BUFFERED)
    val events = _events.receiveAsFlow()

    fun onLogin() = viewModelScope.launch {
        _events.send(LoginEvent.NavigateHome)
    }
}

// Compose
LaunchedEffect(Unit) {
    vm.events.collect { event -> when (event) { ... } }
}

Consejo:
"State para datos, Channel/SharedFlow(0) para eventos one-shot". Evita LiveData<Event<T>> con wrappers, ya no hace falta con Flows bien diseñados.

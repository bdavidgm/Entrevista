Concepto:
LifecycleOwner es una interfaz implementada por Activity, Fragment, etc., que posee un Lifecycle observable. Lifecycle es el objeto que rastrea el estado actual (CREATED, STARTED, RESUMED, DESTROYED) y permite que observadores (LifecycleObserver) reaccionen de forma segura sin fugas de memoria.

Explicación detallada:
lifecycleScope y viewModelScope integran corrutinas: cancelación automática al destruir owner. repeatOnLifecycle(Lifecycle.State.STARTED) { collect } evita consumir Flow cuando la UI no está visible, ahorrando batería y evitando crashes.

LiveData.observe(owner) solo entrega si owner está al menos STARTED. DefaultLifecycleObserver simplifica callbacks. En Compose, LocalLifecycleOwner provee el owner para DisposableEffect ligado a lifecycle.

En entrevistas: por qué no usar observeForever sin remover; diferencia lifecycle de fragment vs viewLifecycleOwner. Diagrama estados Lifecycle.State.

Cuándo usarlo:
Siempre que suscribas Flow, LiveData, listeners de sensor o location a una pantalla. Usa viewLifecycleOwner en Fragment para código que toca la vista.

Ejemplo:
class FeedFragment : Fragment() {
    override fun onViewCreated(view: View, savedInstanceState: Bundle?) {
        viewLifecycleOwner.lifecycleScope.launch {
            viewLifecycleOwner.repeatOnLifecycle(Lifecycle.State.STARTED) {
                viewModel.feed.collect { adapter.submitList(it) }
            }
        }
    }
}

Consejo para entrevista:
repeatOnLifecycle vs collect en onCreate directo. ESTADO mínimo para collectors. viewLifecycleOwner en Fragment obligatorio para UI.

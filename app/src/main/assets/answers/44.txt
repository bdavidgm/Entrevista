Concepto:
LifecycleOwner es cualquier objeto que tiene un Lifecycle: Activity, Fragment, View (en Compose), ProcessLifecycleOwner. Lifecycle representa el estado actual (INITIALIZED, CREATED, STARTED, RESUMED, DESTROYED) y notifica cambios mediante DefaultLifecycleObserver o lifecycleScope.

Permite acoplar trabajo asincrónico (coroutines, Flow.collect) al ciclo de vida del componente, evitando memory leaks y crashes al actualizar UI fuera de tiempo.

Ejemplo:
class MyFragment : Fragment() {
    override fun onViewCreated(view: View, savedInstanceState: Bundle?) {
        viewLifecycleOwner.lifecycleScope.launch {
            repeatOnLifecycle(Lifecycle.State.STARTED) {
                viewModel.state.collect(::render)
            }
        }
    }
}

Consejo:
"repeatOnLifecycle(STARTED) para colectar Flows en UI". En Fragments usa viewLifecycleOwner para evitar leaks tras onDestroyView.

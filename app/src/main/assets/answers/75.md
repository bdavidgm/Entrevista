## Concepto
Navigation Component es la solución de Jetpack para gestionar la navegación dentro de una app: define un grafo (XML para Views, código para Compose) con destinos, acciones y argumentos, gestiona el [back stack](pregunta:118) y centraliza los [deep link](pregunta:119)s. Para Compose existe la variante [Navigation Compose](pregunta:60).

Ventajas: argumentos tipados (Safe Args), animaciones de transición declarativas, [deep link](pregunta:119)s automáticos y soporte de nested graphs por feature.

Ejemplo:
// Compose
val navController = rememberNavController()
NavHost(navController, startDestination = "home") {
    composable("home") { HomeScreen { navController.navigate("detail/$it") } }
    composable("detail/{id}") { DetailScreen(it.arguments?.getString("id")!!) }
}

ExplicaciónKotlin:
val navController = rememberNavController() tipado NavHostController implícito. [NavHost](pregunta:60)(navController, startDestination = "home") { [composable](pregunta:46)("home") { HomeScreen { navController.navigate("detail/$it") } } } usa lambda trailing: HomeScreen recibe lambda sin nombre (it -> navigate) donde it es argumento inferido del callback onNavigate. [composable](pregunta:46)("detail/{id}") { DetailScreen(it.arguments?.getString("id")!!) } usa [safe call](pregunta:17) ?. sobre arguments nullable y !! para forzar String; alternativa idiomática sería requireNotNull o let. DetailScreen(...) recibe id forzado. Las rutas son strings interpolados "detail/$it". El DSL [composable](pregunta:46) anida lambdas [@Composable](pregunta:46). NavHostController.navigate es función regular Kotlin/Java de la librería Navigation.

Concepto:
AlarmManager es un servicio del sistema que dispara intents (broadcasts o activities) en un momento futuro o de forma periódica, incluso si la app está en segundo plano, sujeto a restricciones de ahorro de batería desde Android 6+.

Explicación detallada:
Permite programar alarmas con:
- setExact / setExactAndAllowWhileIdle: hora precisa (despertar dispositivo); requiere permisos SCHEDULE_EXACT_ALARM (API 31+) o USE_EXACT_ALARM para casos elegibles (reloj, alarma).
- setWindow / setAndAllowWhileIdle: ventana flexible; mejor para batería.
- setRepeating: deprecado en favor de ventanas; Doze puede agrupar ejecuciones.
- setAlarmClock: muestra icono de alarma en status bar; el usuario entiende que es crítico.

El disparador suele ser PendingIntent hacia BroadcastReceiver o Service. En Android 8+ muchos broadcasts implícitos están limitados; patrón común: AlarmManager → BroadcastReceiver → startForegroundService o encolar WorkManager.

Restricciones modernas:
- Doze y App Standby retrasan alarmas inexactas.
- Límites a alarmas exactas repetitivas (API 23+): usar una alarma y reprogramar en el receiver.
- Android 12+: permiso SCHEDULE_EXACT_ALARM; el usuario puede revocarlo en ajustes.
- Para trabajo diferible y con restricciones de red/carga, Google recomienda WorkManager en lugar de AlarmManager salvo alarmas de usuario visibles (despertador, calendario).

Cuándo usarlo:
- Alarmas de reloj/despertador, recordatorios de calendario con hora exacta.
- Notificación a una hora concreta acordada con el usuario.
- Casos legacy donde WorkManager no encaja (muy pocos hoy).

Evitar: sincronización periódica de datos, uploads en background, polling de red (usar WorkManager + NetworkCallback).

Ejemplo:
val alarmManager = context.getSystemService(AlarmManager::class.java)
val pending = PendingIntent.getBroadcast(
    context, 0,
    Intent(context, AlarmReceiver::class.java),
    PendingIntent.FLAG_UPDATE_CURRENT or PendingIntent.FLAG_IMMUTABLE
)
val triggerAt = System.currentTimeMillis() + 60_000
if (Build.VERSION.SDK_INT >= Build.VERSION_CODES.M) {
    alarmManager.setExactAndAllowWhileIdle(
        AlarmManager.RTC_WAKEUP, triggerAt, pending
    )
}

Consejo para entrevista:
Contrasta siempre con WorkManager. Di “exact alarm” vs “inexact”, Doze, y permisos API 31+. Buena respuesta: AlarmManager para UX de alarma real; WorkManager para sync en background.

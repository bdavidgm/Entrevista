## Concepto
Las clases y sus miembros son final por defecto en Kotlin: no pueden heredarse ni sobrescribirse. Debes marcarlos open de forma explícita. Una subclase declara el tipo base después de : y llama a su constructor.

Ejemplo:
open class Employee(val name: String) {
    open fun role() = "Empleado"
}

class AndroidDeveloper(name: String) : Employee(name) {
    override fun role() = "Android Developer"
}

ExplicaciónKotlin:
`open class Employee(val name: String)` marca la clase como heredable; sin `open`, Kotlin la haría final por defecto. `open fun role() = "Empleado"` permite sobrescritura en subclases. `class AndroidDeveloper(name: String) : Employee(name)` declara subclase que pasa `name` al constructor del padre con sintaxis `: Employee(name)`. `override fun role() = "Android Developer"` reemplaza implementación; `override` es obligatorio en Kotlin para evitar accidentes. No se usa `open` en la subclase, así que `AndroidDeveloper` sigue siendo final. La herencia es single: solo `: Employee`, no múltiples clases. Propiedad `name` heredada es `val` pública accesible en subclase. [Polimorfismo](pregunta:250): variable `Employee` puede referenciar `AndroidDeveloper` y llamar `role()` dinámicamente. Kotlin favorece composición, pero `open`/`override` modelan extensiones controladas de comportamiento.

Consejo:
El valor final por defecto evita jerarquías accidentales. Abre solo los puntos de extensión que formen parte del diseño.

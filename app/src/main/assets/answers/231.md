## Concepto
Flashear es escribir una imagen firmware (boot, system, recovery, vbmeta) en particiones del dispositivo mediante fastboot o herramientas OEM. Se usa para actualizar, downgrade, reparar bricks suaves o instalar ROMs.

Siempre verifica checksums y guías del modelo exacto.

Ejemplo:
fastboot flash system system.img && fastboot reboot

ExplicaciónKotlin:
Flashear `system.img` reemplaza la partición del sistema Android donde residen framework y apps preinstaladas, no el código Kotlin de tu proyecto en `/data/app`. Tras el flash, tu app se instala de nuevo con `adb install` o Play Store; el bytecode Kotlin se recompila en ART en cada actualización de app, independiente del system image. `fastboot flash system system.img && fastboot reboot` es shell; en automatización Kotlin usarías procesos externos, no APIs del SDK. Verificar checksums del `.img` es buena práctica antes de escribir particiones. Si el system image no coincide con el vendor/boot, el dispositivo puede no arrancar aunque tu Kotlin compile bien. Para desarrolladores de apps, `./gradlew` y variantes `debug`/`release` son la vía normal; fastboot flash es recuperación o ROM custom. Conocer la diferencia evita intentar "flashear" un APK como si fuera partición de sistema.

Consejo:
vbmeta/avb pueden requerir --disable-verity en flasheos no oficiales (riesgoso). Haz backup de partición crítica si el OEM lo permite.

Concepto:
Parcelable es la interfaz de Android para serializar objetos de forma eficiente al pasarlos entre procesos o componentes (extras de Intent, argumentos de Bundle, SavedState). Es más rápida que Serializable de Java y se considera la forma idiomática en Android.

Se puede implementar a mano o, mucho más cómodo, generarse automáticamente con el plugin kotlin-parcelize y la anotación @Parcelize.

Ejemplo:
@Parcelize
data class User(val id: String, val name: String) : Parcelable

intent.putExtra("user", user)
val u = intent.getParcelableExtra<User>("user")


ExplicaciónKotlin:
`@Parcelize data class User(val id: String, val name: String) : Parcelable` — plugin `kotlin-parcelize` genera implementación `Parcelable` en compile time. `data class` proporciona `copy`, `equals`, componentes. `: Parcelable` interfaz Android para serialización inter-proceso. `intent.putExtra("user", user)` — overload para Parcelable empaqueta en Intent extras. `intent.getParcelableExtra<User>("user")` — función genérica con reified type parameter en extensiones KTX modernas; en API antigua retorna nullable. `@Parcelize` elimina boilerplate manual de `writeToParcel`. Propiedades `val` se serializan por orden del constructor primario.

Consejo:
Prefiere @Parcelize a implementar Parcelable a mano. A partir de Android 13 usa getParcelableExtra(name, clazz) en vez del genérico deprecado.

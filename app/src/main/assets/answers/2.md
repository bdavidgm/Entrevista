## Concepto
El operador Elvis ?: devuelve el operando de la izquierda si no es null y, en caso contrario, evalúa y devuelve el de la derecha. Sirve para dar valores por defecto, lanzar excepciones o hacer early return de forma concisa.

Puede combinarse con throw o return porque ambos son expresiones de tipo Nothing, lo que el compilador interpreta como “si el lado izquierdo es null, sal de aquí”.

Ejemplo:
val name = user?.name ?: "Anónimo"
val id = bundle?.getString("id") ?: return
val token = prefs.getToken() ?: throw IllegalStateException("Sin sesión")

ExplicaciónKotlin:
Cada línea declara una propiedad local con `val` (referencia inmutable) y combina el [safe call](pregunta:17) `?.` con Elvis `?:`. En `user?.name ?: "Anónimo"`, si `user` o `name` son null, el resultado es el literal `"Anónimo"`; Kotlin infiere `String` no anulable. La segunda línea muestra el patrón `?: return`: si `bundle?.getString("id")` devuelve null, la función contenedora termina de inmediato; el tipo de retorno de `throw` y `return` es `Nothing`, que hace que el compilador acepte la expresión como válida en contextos que esperan un valor concreto. La tercera usa `?: throw IllegalStateException(...)`, otro caso de `Nothing`: o bien obtienes el token no nulo o la ejecución aborta con excepción. Estos tres usos demuestran que Elvis no solo asigna defaults, sino que también expresa control de flujo de forma declarativa.

Consejo:
Diferéncialo del [safe call](pregunta:17) (?.). Menciona el patrón "?: return" para salir temprano y mantener el código plano.

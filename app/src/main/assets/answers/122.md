## Concepto
as es el operador de cast inseguro: convierte un valor al tipo indicado y lanza ClassCastException si no encaja. as? es el cast seguro: devuelve null en lugar de lanzar excepción cuando el tipo no coincide.

as? combinado con ?: es el patrón habitual para convertir y proporcionar un fallback sin riesgos.

Ejemplo:
val any: Any = "hola"

val s1 = any as String          // ok
val s2 = any as Int             // ClassCastException
val s3 = any as? Int            // null
val s4 = (any as? Int) ?: -1    // -1


ExplicaciónKotlin:
`val any: Any = "hola"` declara referencia de tipo base `Any` (equivalente Java Object) apuntando a String. `any as String` — cast inseguro: compila y ejecuta OK aquí. `any as Int` — fallaría en runtime con `ClassCastException`. `any as? Int` — safe cast devuelve `Int?`; si falla retorna `null` sin excepción. `(any as? Int) ?: -1` combina safe cast con [operador Elvis](pregunta:2): si null, usa `-1`. Precedencia: paréntesis agrupan safe cast antes de [Elvis](pregunta:2). `as` y `as?` son operadores de Kotlin, no funciones. Preferir `as?` cuando el tipo no está garantizado por [smart cast](pregunta:123) previo.

Consejo:
Preferir as? + [Elvis](pregunta:2) sobre as. Mencionar [smart cast](pregunta:123) como alternativa cuando ya hay un is previo.

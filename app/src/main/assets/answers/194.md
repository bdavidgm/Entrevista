## Concepto
Android Studio es el IDE oficial: integra el Android [Gradle](pregunta:109) Plugin, emulador, Layout Inspector, Profiler (CPU/memoria/red/energía), Database Inspector, Logcat y asistentes para Jetpack/Compose. Está optimizado para el ciclo completo compile-deploy-debug de apps Android.

Alternativas existen, pero pierdes tooling de primera parte y actualización al día con cada API.

Ejemplo:
./gradlew assembleDebug desde la terminal integrada; Breakpoints en código Kotlin/Java y en layouts híbridos.

ExplicaciónKotlin:
El ejemplo mezcla un comando [Gradle](pregunta:109) en shell con referencia a código Kotlin en el IDE; no hay snippet Kotlin ejecutable aquí. `./gradlew assembleDebug` invoca el wrapper [Gradle](pregunta:109) para compilar la variante debug, proceso que compila archivos `.kt` con el compilador Kotlin integrado en Android [Gradle](pregunta:109) Plugin. Los breakpoints en Kotlin funcionan igual que en Java porque el bytecode JVM/DEX conserva información de depuración con nombres de funciones, lambdas y variables locales generadas por el compilador Kotlin. En layouts híbridos (XML + Compose), el depurador puede alternar entre código declarativo Kotlin (`@Composable`) y vistas tradicionales. Desde Kotlin, `./gradlew` se suele lanzar desde la terminal integrada de Android Studio asociada al módulo raíz del proyecto. El flujo profesional implica escribir Kotlin, compilar con AGP y depurar con source maps que mapean lambdas y [funciones inline](pregunta:20) a líneas reales del `.kt`.

Consejo:
Menciona [Baseline Profile](pregunta:98), Lint y version catalog como parte del flujo profesional. El emulador con snapshots acelera iteraciones.

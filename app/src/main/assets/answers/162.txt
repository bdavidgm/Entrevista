Concepto:
NotificationChannel es la categoría de notificaciones (Android 8+, API 26) que agrupa comportamiento de sonido, vibración, importancia y visibilidad en pantalla de bloqueo; el usuario controla cada canal en ajustes del sistema.

Explicación detallada:
Antes de Oreo, el comportamiento se configuraba solo en la notificación. Ahora debes crear el canal antes de notificar.

Atributos clave:
- channelId: String único estable.
- name y description: visibles al usuario en ajustes.
- importance: IMPORTANCE_MIN, LOW, DEFAULT, HIGH, MAX — afecta heads-up, sonido, interrupción.
- enableVibration, setSound, enableLights.
- lockscreenVisibility.

Una app puede tener varios canales: "mensajes", "promociones", "descargas". Así el usuario silencia marketing sin perder alertas críticas.

Creación:
- Una vez por instalación; createNotificationChannel es idempotente.
- No se puede bajar importancia programáticamente tras creación (usuario manda); puedes abrir ACTION_CHANNEL_NOTIFICATION_SETTINGS.

Cuándo usarlo:
Siempre al usar NotificationCompat en API 26+. Planifica canales en diseño de producto, no un solo canal genérico.

Ejemplo:
companion object {
    const val CHANNEL_ALERTS = "alerts"
}

fun ensureAlertChannel(context: Context) {
    if (Build.VERSION.SDK_INT < Build.VERSION_CODES.O) return
    val channel = NotificationChannel(
        CHANNEL_ALERTS,
        context.getString(R.string.channel_alerts_name),
        NotificationManager.IMPORTANCE_HIGH
    ).apply {
        description = context.getString(R.string.channel_alerts_desc)
        enableVibration(true)
    }
    context.getSystemService(NotificationManager::class.java)
        .createNotificationChannel(channel)
}

Consejo para entrevista:
Explica que sin canal la notificación no se muestra bien en O+. Menciona importancia y que el usuario puede bloquear canal. Varios canales = mejor UX.

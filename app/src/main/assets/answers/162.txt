Concepto:
Las notificaciones se construyen con NotificationCompat, se asocian a un canal (Android 8+) y se muestran con NotificationManager.notify. Desde Android 13 hace falta el permiso en runtime POST_NOTIFICATIONS.

Foreground services muestran notificación obligatoria al iniciar el servicio.

Ejemplo:
if (Build.VERSION.SDK_INT >= 33) {
    requestPermissions(arrayOf(Manifest.permission.POST_NOTIFICATIONS), 0)
}

val channel = NotificationChannel("msg", "Mensajes", NotificationManager.IMPORTANCE_DEFAULT)
nm.createNotificationChannel(channel)

val notif = NotificationCompat.Builder(this, "msg")
    .setSmallIcon(R.drawable.ic_stat)
    .setContentTitle("Nuevo mensaje")
    .setContentText("Hola")
    .build()

nm.notify(1, notif)


ExplicaciónKotlin:
`if (Build.VERSION.SDK_INT >= 33)` compara SDK int con literal; rama condicional para permiso runtime Android 13+. `requestPermissions(arrayOf(Manifest.permission.POST_NOTIFICATIONS), 0)` usa `arrayOf` vararg factory y constante de permiso; `0` es requestCode legacy. `val channel = NotificationChannel(...)` construye canal con constructor posicional de tres argumentos. `NotificationCompat.Builder(this, "msg")` encadena `.setSmallIcon`, `.setContentTitle`, `.setContentText`, `.build()` — builder pattern Java interop. `nm.notify(1, notif)` publica con id entero. `this` en builder referencia Context/Activity. Kotlin no requiere punto y coma; cada `val` es local inmutable. Mezcla APIs framework con compat library.

Consejo:
Canal con IMPORTANCE adecuada (no todo HIGH). Deep link con PendingIntent; agrupa notificaciones relacionadas con setGroup.

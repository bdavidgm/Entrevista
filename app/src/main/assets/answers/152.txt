Concepto:
Singleton garantiza una única instancia de una clase en todo el proceso. En Kotlin suele implementarse con object; en Android, dependencias globales se registran en Hilt/Koin con scope @Singleton o equivalente.

Evita singletons con estado mutable compartido sin disciplina: complica tests y concurrencia.

Ejemplo:
object Analytics {
    fun track(event: String) { /* ... */ }
}

@Singleton
class AppGraph @Inject constructor() { /* ... */ }

Consejo:
Prefiere DI con scope de aplicación antes de object con estado. Los singletons ocultan dependencias: el constructor inyectado es más claro.

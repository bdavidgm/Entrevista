## Concepto
Tipo::class obtiene el [KClass](pregunta:256) de un tipo conocido en compilación. objeto::class obtiene el [KClass](pregunta:256) del tipo real de una instancia en runtime. Tipo::class.java convierte el [KClass](pregunta:256) a java.lang.Class<T>, necesario para APIs Java como [Intent](pregunta:33), [Retrofit](pregunta:207) o algunas librerías de serialización.

Ejemplo:
open class Animal
class Dog : Animal()

val animal: Animal = Dog()

val declared = Animal::class       // KClass<Animal>
val runtime = animal::class        // KClass<out Animal>, representa Dog
val javaType = Dog::class.java     // Class<Dog>

val intent = Intent(context, DetailActivity::class.java)

ExplicaciónKotlin:
`open class Animal` permite subclases; `class Dog : Animal()` hereda con constructor vacío implícito. `val animal: Animal = Dog()` demuestra [polimorfismo](pregunta:250): tipo declarado `Animal`, runtime `Dog`. `Animal::class` devuelve `KClass<Animal>` del tipo estático declarado, no el runtime. `animal::class` obtiene `KClass<out Animal>` reflejando clase real (`Dog`); útil en logging o serializers. `Dog::class.java` convierte [KClass](pregunta:256) a `Class<Dog>` Java requerido por APIs legacy. `Intent(context, DetailActivity::class.java)` es patrón estándar Activities: segunda parameter es `Class<?>`. Alternativa Kotlin pura: `DetailActivity::class` en APIs que acepten [KClass](pregunta:256). `javaClass` en instancia (`animal.javaClass`) equivale a `animal::class.java`. Variance `out` en [KClass](pregunta:256) refleja que subtipo es válido donde se espera base.

Consejo:
Usa ::class en APIs Kotlin y ::class.java cuando una API Java solicite Class<T>. javaClass también obtiene la clase runtime de una instancia.

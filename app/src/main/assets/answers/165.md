## Concepto
Las notificaciones push llegan desde un servidor a través de Firebase Cloud Messaging (FCM). Tu app obtiene un token de registro por instalación; el backend envía mensajes a ese token (o a topics). Hay mensajes de tipo notification (Play Services pueden mostrar la bandeja si la app está en segundo plano) y data (tú construyes la Notification en código; en background suele requerir FirebaseMessagingService).

En Android 13+ hace falta permiso POST_NOTIFICATIONS para mostrar notificaciones al usuario. El canal ([NotificationChannel](pregunta:163)) sigue siendo obligatorio desde Android 8.

Ejemplo:
class AppMessagingService : FirebaseMessagingService() {
    override fun onMessageReceived(message: RemoteMessage) {
        message.notification?.let { /* título/cuerpo del payload notification */ }
        message.data["orderId"]?.let { /* payload data */ }
        val notif = NotificationCompat.Builder(this, CHANNEL_ID)
            .setSmallIcon(R.drawable.ic_stat)
            .setContentTitle(message.notification?.title ?: "Pedido")
            .setContentText(message.notification?.body ?: message.data["status"])
            .build()
        NotificationManagerCompat.from(this).notify(orderId.hashCode(), notif)
    }

    override fun onNewToken(token: String) {
        // Enviar token a tu backend para dirigir pushes a este dispositivo
    }
}


ExplicaciónKotlin:
`class AppMessagingService : FirebaseMessagingService()` hereda servicio de Firebase; métodos marcados `override`. `onMessageReceived(message: RemoteMessage)` procesa payload entrante. `message.notification?.let { ... }` [safe call](pregunta:17) + scope `let` solo si notification no es null. El [Elvis](pregunta:2) `?:` en `setContentTitle` y `setContentText` provee fallback desde `message.data["status"]` — indexación de mapa devuelve nullable. `NotificationCompat.Builder(this, CHANNEL_ID)` usa `this` como [Context](pregunta:127) del servicio. `orderId.hashCode()` convierte id a int para tag de notificación. `NotificationManagerCompat.from(this).notify(...)` API compat. `onNewToken(token: String)` recibe token FCM actualizado como parámetro no nullable para sincronizar con backend.

Consejo:
No confíes solo en el payload notification para lógica crítica: en iOS/Android el comportamiento en background difiere; combina data + tu propia UI. Renueva el token en backend y maneja revocaciones. Para alto volumen, colas y reintentos van en el servidor, no en el cliente.

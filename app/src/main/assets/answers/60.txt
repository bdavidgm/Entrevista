Concepto:
Navigation Compose es la librería oficial para navegar entre Composables. Define un NavGraph con rutas (strings o, mejor, type-safe en versiones recientes) y un NavHost que muestra el destino actual según el back stack. NavController expone navigate/popBackStack.

Soporta argumentos tipados, deep links y nested graphs para feature modules.

Ejemplo:
val navController = rememberNavController()
NavHost(navController, startDestination = "home") {
    composable("home") {
        HomeScreen(onUser = { id -> navController.navigate("user/$id") })
    }
    composable(
        route = "user/{id}",
        arguments = listOf(navArgument("id") { type = NavType.StringType })
    ) { backStackEntry ->
        UserScreen(id = backStackEntry.arguments!!.getString("id")!!)
    }
}

ExplicaciónKotlin:
val navController = rememberNavController() crea y recuerda un NavController en Compose. NavHost(navController, startDestination = "home") { ... } usa trailing lambda con NavGraphBuilder DSL. composable("home") { HomeScreen(onUser = { id -> navController.navigate("user/$id") }) } registra ruta string; la lambda interna pasa onUser como lambda (id) -> Unit que interpola id en template string "user/$id" y llama navigate. composable(route = "user/{id}", arguments = listOf(navArgument("id") { type = NavType.StringType })) define argumento tipado; navArgument("id") { type = NavType.StringType } es lambda con receiver sobre NavArgumentBuilder. { backStackEntry -> UserScreen(id = backStackEntry.arguments!!.getString("id")!!) } usa doble operador !! (not-null assertion) sobre arguments y getString, forzando no-null en runtime con riesgo de NPE si falta el argumento. UserScreen(id = ...) usa parámetro nombrado.

Consejo:
"Single Activity + NavHost". Para argumentos prefiere tipos seguros (sealed routes o la nueva API type-safe). Maneja deep links registrándolos en composable(deepLinks = ...).

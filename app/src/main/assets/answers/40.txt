Concepto:
dp (density-independent pixel) y sp (scale-independent pixel) son unidades de medida en Android que escalan con la densidad de pantalla y, en el caso de sp, también con las preferencias de tamaño de fuente del usuario. Garantizan que layouts y textos se vean proporcionales en distintos dispositivos.

Explicación detallada:
1 dp equivale aproximadamente a un pixel físico en una pantalla mdpi (160 dpi). En xhdpi, 16dp ocupan más píxeles reales pero el mismo espacio "visual" relativo. px absolutos se evitan salvo cálculos de bajo nivel o bitmaps.

sp se usa para texto y respeta Accessibility font scale: si el usuario aumenta texto en ajustes, sp crece; dp para márgenes, iconos y contenedores que no deben inflarse con fuente (aunque a veces se debate usar sp en contenedores de solo texto).

En Compose: Modifier.padding(16.dp), fontSize = 14.sp. Conversión desde px: TypedValue.applyDimension o recursos dimen. En entrevistas piden fórmula conceptual: px = dp * (dpi / 160).

Cuándo usarlo:
dp para dimensiones, elevaciones visuales, iconos, radios. sp para textSize y lineHeight de texto legible. Nunca hardcodees px en layouts de producción.

Ejemplo:
<!-- res/values/dimens.xml -->
<dimen name="padding_standard">16dp</dimen>

<TextView
    android:layout_width="match_parent"
    android:layout_height="wrap_content"
    android:textSize="16sp"
    android:padding="@dimen/padding_standard" />

@Composable
fun Title(text: String) {
    Text(
        text = text,
        fontSize = 18.sp,
        modifier = Modifier.padding(horizontal = 16.dp, vertical = 8.dp)
    )
}

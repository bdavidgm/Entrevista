## Concepto
getSystemService(Class) devuelve el servicio del sistema tipado (ConnectivityManager, [AlarmManager](pregunta:175), ClipboardManager, etc.) asociado al [Context](pregunta:127). Es la forma recomendada frente a getSystemService(String) con cast manual.

Cada servicio encapsula hardware o políticas del sistema.

Ejemplo:
val cm = getSystemService(ConnectivityManager::class.java)
val nm = getSystemService(NotificationManager::class.java)
val imm = getSystemService(InputMethodManager::class.java)

ExplicaciónKotlin:
Cada línea declara una variable local con `val`, lo que garantiza [inmutabilidad](pregunta:115) de la referencia. Las tres invocaciones comparten el mismo patrón: `getSystemService(T::class.java)`, donde `T::class` es la referencia de clase de Kotlin y `.java` la convierte al `Class<T>` que exige la API Android basada en Java. El type parameter se infiere en cada llamada (`ConnectivityManager`, `NotificationManager`, `InputMethodManager`), de modo que el retorno ya viene tipado sin cast explícito — ventaja sobre la variante antigua con `String` y `(X) getSystemService(...)`. Estas propiedades suelen invocarse desde una `Activity`, `Service` o `Application` que implementan `Context`. No hay lambdas ni extensiones personalizadas; es interop directa y repetible para cualquier servicio del sistema disponible en el SDK.

Consejo:
Usa ContextCompat o [applicationContext](pregunta:127) cuando el servicio no dependa de la [Activity](pregunta:31). Algunos servicios requieren API mínima: comprueba versiones.

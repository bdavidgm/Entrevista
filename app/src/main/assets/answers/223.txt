Concepto:
Un Foreground Service es un Service que el sistema considera "visible" para el usuario y al que da prioridad para no matarlo en segundo plano. Debe mostrar una notificación persistente y declararse en el manifest con un foregroundServiceType (location, mediaPlayback, dataSync, etc., obligatorio en Android 10+).

Se usa para tareas largas y visibles para el usuario: reproductor, navegación GPS, llamadas VoIP, descargas grandes. No es para trabajo invisible: para eso, WorkManager.

Ejemplo:
class LocationService : Service() {
    override fun onStartCommand(intent: Intent?, flags: Int, startId: Int): Int {
        val notif = NotificationCompat.Builder(this, CHANNEL_ID)
            .setSmallIcon(R.drawable.ic_loc)
            .setContentTitle("Rastreando ubicación")
            .build()
        startForeground(NOTIF_ID, notif)
        return START_STICKY
    }
    override fun onBind(intent: Intent?): IBinder? = null
}

Consejo:
"Visible + notificación obligatoria + foregroundServiceType". A partir de Android 14 hay restricciones aún más estrictas para iniciar foreground services desde background.

Concepto:
El ProcessLifecycleOwner expone un Lifecycle a nivel de proceso que refleja cuándo la app entra y sale de primer plano, agregando los eventos de todas las activities. Pasa a ON_STOP cuando ninguna está visible y a ON_START cuando alguna lo está.

Es útil para inicializar/desligar recursos globales como WebSocket, sincronización o analytics de sesión sin atarse a una Activity concreta.

Ejemplo:
ProcessLifecycleOwner.get().lifecycle.addObserver(object : DefaultLifecycleObserver {
    override fun onStart(owner: LifecycleOwner) { wsClient.connect() }
    override fun onStop(owner: LifecycleOwner)  { wsClient.disconnect() }
})

ExplicaciónKotlin:
`ProcessLifecycleOwner.get().lifecycle.addObserver(...)` encadena llamadas: `get()` devuelve el singleton del owner de proceso; `.lifecycle` expone el `Lifecycle`; `.addObserver` registra un observador. `object : DefaultLifecycleObserver { ... }` es un object expression anónimo que implementa la interfaz `DefaultLifecycleObserver` (API con métodos default vacíos, solo sobrescribes los necesarios). `override fun onStart(owner: LifecycleOwner)` y `onStop` reciben el owner como parámetro nombrado. Las funciones de cuerpo único con llaves contienen llamadas a `wsClient.connect()` y `disconnect()`. No hay corrutinas ni lambdas con receptor: es el patrón Observer clásico expresado con sintaxis Kotlin concisa. El object expression evita crear una clase nombrada separada para lógica de una sola instancia.

Consejo:
"Foreground/background a nivel de proceso", no de Activity. Útil para sesiones, analytics y conexiones persistentes.

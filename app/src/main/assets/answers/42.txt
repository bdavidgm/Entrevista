Concepto:
Process Lifecycle (ProcessLifecycleOwner) expone el ciclo de vida del proceso completo de la aplicación en primer plano o background, no solo de una Activity. Permite observar cuando el usuario está interactuando con cualquier pantalla de tu app versus cuando la app está en segundo plano.

Explicación detallada:
ProcessLifecycleOwner.get().lifecycle dispara ON_START cuando la primera activity pasa a visible y ON_STOP cuando la última deja de serlo (con pequeño delay para cambios rápidos entre activities). Útil para analytics de sesión, pausar polling global, flush de logs, o iniciar/detener servicios ligeros conscientes de foreground.

No sustituye el lifecycle de cada Activity: cosas específicas de pantalla siguen en activity/fragment lifecycle. Librería androidx.lifecycle.ProcessLifecycleOwner.

En entrevistas contrastan con Activity lifecycle y preguntan casos: "app en background" global. Cuidado: ON_STOP del proceso no significa que el proceso fue matado.

Cuándo usarlo:
Métricas de sesión, SDK que necesita saber foreground, coordinar trabajo que debe pausarse cuando toda la app está background. No para lógica de UI de una pantalla concreta.

Ejemplo:
class MyApp : Application() {
    override fun onCreate() {
        super.onCreate()
        ProcessLifecycleOwner.get().lifecycle.addObserver(
            object : DefaultLifecycleObserver {
                override fun onStart(owner: LifecycleOwner) {
                    analytics.appForeground()
                }
                override fun onStop(owner: LifecycleOwner) {
                    analytics.appBackground()
                }
            }
        )
    }
}

Consejo para entrevista:
Primera activity visible vs última oculta. No confundir con onDestroy de activity. Alternativa: ActivityLifecycleCallbacks manual (más verboso).

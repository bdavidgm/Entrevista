## Concepto
Animatable es una API de Compose para animar un valor de forma imperativa dentro de una [coroutine](pregunta:78), ofreciendo control sobre el ciclo de la animación (animateTo, snapTo, stop). Es complemento de las APIs declarativas como animate*AsState y útil cuando necesitas combinar gestos con animaciones.

Ejemplo:
@Composable
fun BouncyBox() {
    val offsetX = remember { Animatable(0f) }
    LaunchedEffect(Unit) {
        offsetX.animateTo(200f, animationSpec = spring())
    }
    Box(
        Modifier
            .offset { IntOffset(offsetX.value.toInt(), 0) }
            .size(48.dp)
            .background(Color.Red)
    )
}


ExplicaciónKotlin:
`remember { Animatable(0f) }` memoriza un `Animatable<Float>` entre [recomposiciones](pregunta:47); el lambda de `remember` solo corre en la primera composición. `LaunchedEffect(Unit)` lanza [corrutina](pregunta:78) asociada al [composable](pregunta:46) para llamar `animateTo`, [función suspend](pregunta:79) que interpola hacia `200f` usando `animationSpec = spring()` (parámetro nombrado con valor por defecto sobrescrito). `offsetX.value` lee el valor actual del animatable dentro del lambda de `Modifier.offset`, que recibe `@Composable`-aware lambda `Density.() -> IntOffset`. `toInt()` convierte el `Float` animado a entero para posicionamiento en píxeles. La cadena `Modifier.offset(...).size(48.dp).background(Color.Red)` encadena extensiones que retornan `Modifier` (patrón fluent de Compose). `48.dp` usa propiedad de extensión `Int.dp` del tipo `Dp`. `Box` es contenedor [composable](pregunta:46) cuyo único hijo es el modificador aplicado al propio contenedor.

Consejo:
"Animatable = animación imperativa controlable". Para casos declarativos simples, prefiere animate*AsState; usa Animatable cuando necesitas pausar, reiniciar o combinar con gestos.

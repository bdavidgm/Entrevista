## Concepto
override indica explícitamente que una propiedad o función reemplaza un miembro open o abstract del padre. super permite invocar la implementación heredada. Un miembro sobrescrito continúa abierto, salvo que se marque final override.

Ejemplo:
open class Logger {
    open fun log(message: String) {
        println(message)
    }
}

class TaggedLogger : Logger() {
    final override fun log(message: String) {
        super.log("[APP] $message")
    }
}

ExplicaciónKotlin:
`open class Logger { open fun log(message: String) { println(message) } }` expone método sobrescribible; cuerpo con llaves usa bloque, no expresión. `class TaggedLogger : Logger()` hereda sin argumentos adicionales. `final override fun log(message: String)` combina `override` con `final` para prohibir nueva sobrescritura en descendientes de `TaggedLogger`. `super.log("[APP] $message")` invoca implementación del padre; template string inserta prefijo. `println` es función Kotlin estándar. Si omitieras `override`, el compilador error; si omitieras `open` en padre, también. Miembros sobrescritos son `open` implícitamente salvo `final override`. Este patrón decorador es común en logging Android (Timber, wrappers). `super` desambigua cuando hay múltiples interfaces con mismo método usando `super<Logger>`.

Consejo:
override obligatorio evita sobrescrituras por accidente. Usa super solo si el contrato de la clase base exige conservar parte de su comportamiento.

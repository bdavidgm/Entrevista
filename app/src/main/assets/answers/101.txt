Concepto:
Los datos sensibles (tokens, contraseñas, claves API, PII) no deben almacenarse en texto plano en SharedPreferences sin cifrar, archivos externos legibles, logs ni código fuente. Se usan Android Keystore, EncryptedSharedPreferences/DataStore, servidor remoto y principio de mínimo privilegio.

Explicación detallada:
Keystore genera claves hardware-backed difíciles de extraer en dispositivos compatibles. EncryptedSharedPreferences (Security crypto) cifra valores. Para tokens OAuth: almacenar refresh token cifrado; access token corta vida en memoria si posible.

Nunca hardcodear API secrets en APK (reverse engineering con apktool). Usa backend proxy o Play App Signing + remote config con rotación. certificate pinning para MITM.

Logs: desactivar body en OkHttp producción. excludeFromBackup en manifest para backups cloud. BiometricPrompt antes de mostrar datos ultra sensibles.

En el contexto de proyectos Android reales, combina esta idea con las guías oficiales de Android Developers y con métricas de calidad (crashes, ANR, Vitals) cuando aplique. En code review, pregunta siempre por ciclo de vida, testabilidad y si la solución escala cuando el equipo crece.

Durante la entrevista, estructura la respuesta en definición, ejemplo concreto de tu experiencia o proyecto demo, y trade-offs frente a alternativas. Mencionar versiones recientes de API o biblioteca (Material3, Paging 3, Hilt 2.x) demuestra que sigues el ecosistema, sin necesidad de memorizar números de versión exactos.

Relaciona este concepto con los demás bloques del manifiesto Entrevista (arquitectura, corrutinas, seguridad, build) para demostrar visión de sistema completo, no conocimiento aislado de una API.

Cuándo usarlo:
Cualquier credencial, salud, financiero. DataStore normal para tema oscuro; Keystore para secrets.

Ejemplo:
val masterKey = MasterKey.Builder(context)
    .setKeyScheme(MasterKey.KeyScheme.AES256_GCM)
    .build()

val securePrefs = EncryptedSharedPreferences.create(
    context, "secure_prefs", masterKey,
    EncryptedSharedPreferences.PrefKeyEncryptionScheme.AES256_SIV,
    EncryptedSharedPreferences.PrefValueEncryptionScheme.AES256_GCM
)

Consejo para entrevista:
Keystore + encrypted prefs + no logs + no hardcode. OWASP Mobile mención breve. Root bypass: defensa en profundidad, no absoluta.

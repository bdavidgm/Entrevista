## Concepto
SideEffect ejecuta un bloque después de cada [recomposición](pregunta:47) exitosa, útil para publicar valores hacia código no-Compose (analytics, librerías que esperan estado actualizado). DisposableEffect ejecuta un bloque al entrar en la composition y permite definir un onDispose que se ejecuta al salir o al cambiar las keys, ideal para registrar/desregistrar listeners.

Ambos son side effects "manejados" por Compose, en contraste con escribir código no controlado en el cuerpo del [Composable](pregunta:46).

Ejemplo:
SideEffect {
    analytics.set("isPremium", user.isPremium)
}

DisposableEffect(systemUiController, useDark) {
    systemUiController.setStatusBarColor(if (useDark) Color.Black else Color.White)
    onDispose { /* limpieza si hace falta */ }
}

ExplicaciónKotlin:
SideEffect { analytics.set("isPremium", user.isPremium) } ejecuta el lambda después de cada [recomposición](pregunta:47) exitosa; accede a user.isPremium como propiedad de un objeto user del scope [composable](pregunta:46) externo. No devuelve valor útil (Unit) y no admite onDispose. DisposableEffect(systemUiController, useDark) recibe keys: si cualquiera cambia, ejecuta dispose del efecto anterior y vuelve a entrar. El bloque recibe systemUiController y useDark del scope; if (useDark) Color.Black else Color.White es expresión if de Kotlin que devuelve Color. onDispose { /* limpieza */ } es una lambda especial del DSL DisposableEffect que Compose invoca al salir o al cambiar keys. Ambos constructos son funciones composables de orden superior que reciben lambdas; SideEffect no es [suspend](pregunta:79), DisposableEffect tampoco en el bloque principal (salvo APIs internas).

Consejo:
"SideEffect = tras [recomposición](pregunta:47), sin limpieza"; "DisposableEffect = entrar/salir con onDispose". [LaunchedEffect](pregunta:53), para trabajo suspendido.

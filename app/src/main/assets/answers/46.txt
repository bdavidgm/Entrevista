Concepto:
Un Composable es una función Kotlin anotada con @Composable que describe parte del árbol de UI de Jetpack Compose. Al invocarse, el runtime de Compose registra qué se dibujó y puede volver a ejecutar la función cuando cambia el estado, de forma declarativa similar a re-renderizar según datos.

Explicación detallada:
Las funciones Composable deben llamarse solo desde otras Composables o desde setContent { }. No devuelven View sino que emiten UI al Composition. Pueden tener parámetros de estado y eventos callback. Recomposition re-ejecuta funciones cuyas entradas cambiaron.

Reglas: evitar side effects directos en el cuerpo (usar LaunchedEffect, rememberCoroutineScope). Estabilidad de parámetros (stable types) ayuda a optimizar skips. Preview en Android Studio con @Preview.

En entrevistas comparan con XML imperativo: UI = f(state). Preguntan pureza y por qué no hacer HTTP en el cuerpo del Composable.

Cuándo usarlo:
Toda UI nueva en Compose: pantallas, items, temas. Componentes pequeños y reusables con parámetros claros.

Ejemplo:
@Composable
fun GreetingCard(name: String, onDismiss: () -> Unit) {
    Card(modifier = Modifier.padding(16.dp)) {
        Column(Modifier.padding(16.dp)) {
            Text("Hola, $name", style = MaterialTheme.typography.titleLarge)
            TextButton(onClick = onDismiss) { Text("Cerrar") }
        }
    }
}

Consejo para entrevista:
@Composable restricciones. Recomposition explicada brevemente. Stateless vs stateful composables. No guardar estado mutable sin remember.

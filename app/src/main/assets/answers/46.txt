Concepto:
Un Composable es una función Kotlin anotada con @Composable que describe parte de la UI en Jetpack Compose. Solo puede invocarse desde otra Composable o desde setContent { }, y el runtime la vuelve a ejecutar (recomposition) cuando cambia el estado que lee. El modelo es declarativo: UI = f(state).

Reglas básicas: no hagas side effects en el cuerpo (usa LaunchedEffect, rememberCoroutineScope); pasa estado como parámetros para facilitar tests; previsualiza con @Preview.

Ejemplo:
@Composable
fun GreetingCard(name: String, onDismiss: () -> Unit) {
    Card(Modifier.padding(16.dp)) {
        Column(Modifier.padding(16.dp)) {
            Text("Hola, $name", style = MaterialTheme.typography.titleLarge)
            TextButton(onClick = onDismiss) { Text("Cerrar") }
        }
    }
}

Consejo:
Distingue stateless vs stateful composables. Recuerda usar remember para estado interno y nunca hagas IO ni HTTP en el cuerpo del Composable.

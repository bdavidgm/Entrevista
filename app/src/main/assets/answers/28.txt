Concepto:
internal es un modificador de visibilidad que limita el acceso al mismo módulo de Gradle/IDE. A diferencia de public (todos lados) o private (mismo archivo/clase), internal permite compartir código entre archivos de un módulo sin exponerlo al exterior.

Es especialmente útil en proyectos modulares y librerías para esconder detalles de implementación, sin sacrificar la posibilidad de testearlos desde dentro.

Ejemplo:
// módulo :feature:users
internal class UserMapperImpl : UserMapper { ... }
internal fun parseUserDto(dto: Dto): User = ...

// otro módulo: no compila
val mapper = UserMapperImpl() // error

Consejo:
"internal = mismo módulo". Combinado con módulos por feature ayuda a mantener APIs limpias. Recuerda que la unidad es el módulo Gradle, no el paquete.

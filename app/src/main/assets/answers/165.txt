Concepto:
Obtener la ubicación del usuario en Android requiere permisos de ubicación en runtime, elegir el proveedor adecuado (Fused Location Provider recomendado), respetar límites de background y consumo de batería.

Explicación detallada:
Permisos:
- ACCESS_COARSE_LOCATION / ACCESS_FINE_LOCATION (runtime desde API 23).
- ACCESS_BACKGROUND_LOCATION aparte en API 29+ si necesitas ubicación con app en background (revisión estricta en Play).

Google Play Services Location:
- FusedLocationProviderClient combina GPS, Wi-Fi, sensores.
- getCurrentLocation(priority, cancellationToken) para lectura puntual.
- requestLocationUpdates para seguimiento continuo (Foreground Service + tipo location en API 29+ si en background).

Prioridades:
- PRIORITY_HIGH_ACCURACY vs BALANCED_POWER_ACCURACY.

Sin Google Play:
- LocationManager legacy (menos recomendado).

Buenas prácticas:
- Pedir permiso en contexto (just-in-time) con rationale.
- Dejar de recibir updates en onPause/onStop si no es app de navegación.
- Comprobar isLocationEnabled y Settings para activar GPS.
- Minimizar frecuencia y usar lastLocation cuando baste precisión.

Cuándo usarlo:
Mapas, check-in, entregas, features contextuales. Evitar tracking continuo sin necesidad clara.

Ejemplo:
@AndroidEntryPoint
class MapActivity : AppCompatActivity() {
    @Inject lateinit var fusedClient: FusedLocationProviderClient

    private val requestPermission = registerForActivityResult(
        ActivityResultContracts.RequestMultiplePermissions()
    ) { grants ->
        if (grants[Manifest.permission.ACCESS_FINE_LOCATION] == true) {
            fetchLocation()
        }
    }

    private fun fetchLocation() {
        if (ActivityCompat.checkSelfPermission(this, Manifest.permission.ACCESS_FINE_LOCATION)
            != PackageManager.PERMISSION_GRANTED) {
            requestPermission.launch(arrayOf(Manifest.permission.ACCESS_FINE_LOCATION))
            return
        }
        fusedClient.getCurrentLocation(Priority.PRIORITY_BALANCED_POWER_ACCURACY, null)
            .addOnSuccessListener { location ->
                location?.let { updateMap(it.latitude, it.longitude) }
            }
    }
}

Consejo para entrevista:
Menciona permisos fine/coarse, background aparte, FusedLocationProviderClient y ahorro de batería. Foreground service para tracking continuo.

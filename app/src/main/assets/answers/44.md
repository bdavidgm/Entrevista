## Concepto
LifecycleOwner es cualquier objeto que tiene un Lifecycle: [Activity](pregunta:31), [Fragment](pregunta:32), View (en Compose), [ProcessLifecycleOwner](pregunta:42). Lifecycle representa el estado actual (INITIALIZED, CREATED, STARTED, RESUMED, DESTROYED) y notifica cambios mediante DefaultLifecycleObserver o [lifecycleScope](pregunta:130).

Permite acoplar trabajo asincrónico ([coroutines](pregunta:78), [Flow](pregunta:84).collect) al ciclo de vida del componente, evitando [memory leak](pregunta:117)s y crashes al actualizar UI fuera de tiempo.

Ejemplo:
class MyFragment : Fragment() {
    override fun onViewCreated(view: View, savedInstanceState: Bundle?) {
        viewLifecycleOwner.lifecycleScope.launch {
            repeatOnLifecycle(Lifecycle.State.STARTED) {
                viewModel.state.collect(::render)
            }
        }
    }
}

ExplicaciónKotlin:
MyFragment hereda de [Fragment](pregunta:32)() usando la sintaxis de clase Kotlin con constructor primario implícito vacío. [override](pregunta:247) fun onViewCreated indica que redefinimos un método de la superclase; los parámetros view: View y [savedInstanceState](pregunta:43): Bundle? muestran tipado explícito y nullabilidad (Bundle puede ser null en el primer arranque). viewLifecycleOwner es una propiedad del [Fragment](pregunta:32) que implementa LifecycleOwner y refleja el ciclo de la vista, no el del [Fragment](pregunta:32) entero. [lifecycleScope](pregunta:130) es una propiedad de extensión que expone un [CoroutineScope](pregunta:81) cancelado cuando el LifecycleOwner se destruye. launch { } es un builder de [coroutine](pregunta:78) que inicia trabajo suspendido dentro de ese scope. repeatOnLifecycle(Lifecycle.State.STARTED) { } es una [función suspend](pregunta:79) que reinicia el bloque cuando el estado alcanza STARTED y lo cancela al bajar de ese umbral. [viewModel](pregunta:68).state es presumiblemente un [Flow](pregunta:84); collect(::render) usa una referencia a método (method reference) para pasar cada valor emitido a la función render sin escribir una lambda explícita.

Consejo:
"repeatOnLifecycle(STARTED) para colectar Flows en UI". En Fragments usa viewLifecycleOwner para evitar leaks tras onDestroyView.

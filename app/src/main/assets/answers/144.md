## Concepto
Channel es una primitiva de comunicación entre corrutinas: send pone valores en una cola y receive los consume de forma suspendida. Útil para handoff explícito (actor, pipeline) o colas acotadas con backpressure (Channel.BUFFERED, CONFLATED, etc.).

En UI moderna muchos casos se resuelven con Flow/StateFlow; Channel brilla cuando quieres un consumidor único y orden estricto.

Ejemplo:
val channel = Channel<String>(Channel.BUFFERED)

launch {
    channel.send("ping")
}

launch {
    println(channel.receive())
}


ExplicaciónKotlin:
`Channel<String>(Channel.BUFFERED)` instancia canal genérico con capacidad por defecto del companion `BUFFERED`. `launch { channel.send("ping") }` suspende si el buffer está lleno hasta haber espacio. El segundo `launch { println(channel.receive()) }` consume suspendiendo hasta que haya elemento. Ambos `launch` requieren scope receptor implícito — en el ejemplo falta declararlo explícitamente pero en contexto real sería `coroutineScope.launch`. `send` y `receive` son funciones suspend simétricas del patrón productor-consumidor. `Channel` es clase final de kotlinx.coroutines; el tipo `String` fija el payload. A diferencia de `Flow`, un `Channel` típicamente tiene un consumidor activo; `receive()` es suspensión punto a punto.

Consejo:
Evita Channel infinito sin estrategia: puede crecer la memoria. Para eventos one-shot desde ViewModel, Channel + receiveAsFlow es un patrón habitual.

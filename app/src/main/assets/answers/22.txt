Concepto:
apply y also son scope functions que devuelven el objeto receptor tras ejecutar el lambda, pero difieren en el nombre del receptor (this vs it) y en la intención idiomática: apply para configurar el objeto; also para efectos secundarios adicionales manteniendo el objeto en la cadena.

Explicación detallada:
apply usa this implícito dentro del bloque, lo que favorece asignaciones directas a propiedades sin prefijo: name = "x", flags = .... also usa it, forzando referencia explícita al objeto si lo necesitas en el bloque, lo que a veces mejora claridad cuando el objeto no es "self-evident".

Ambas devuelven el receptor, permitiendo encadenamiento: crear().apply { }.also { log(it) }. En Android, apply domina en builders del framework (Intent, Bundle, View, AlertDialog). also domina para logs, métricas, o añadir a colección mientras devuelves el elemento.

Si solo quieres ejecutar código sin devolver el receptor, usa run o let según necesites valor de retorno. apply/also no deben contener lógica de negocio pesada: son azúcar sintáctico de organización.

En entrevistas piden elegir entre apply y also en snippet: si configuras propiedades del mismo objeto → apply; si "haces algo más" y sigues pasando el objeto → also.

Cuándo usarlo:
apply: inicialización y configuración de objetos recién creados. also: depuración, side effects no relacionados con configuración estructural, pipelines donde el siguiente paso necesita el mismo objeto.

Ejemplo:
val channel = NotificationChannel(ID, NAME, IMPORTANCE_DEFAULT).apply {
    description = "Pedidos"
    enableVibration(true)
}

val order = Order(id = "1")
    .also { cache.put(it.id, it) }
    .also { analytics.track("order_created", it.id) }

val view = TextView(context).apply {
    text = "Total"
    textSize = 16f
}

Consejo para entrevista:
apply = configurar (this); also = además (it). Preguntan retorno: ambos el objeto. Contrasta con let que retorna resultado del lambda.

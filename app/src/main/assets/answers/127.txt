Concepto:
Un Context es un objeto que representa el "entorno" actual de la app: da acceso a recursos, system services, assets, base de datos, SharedPreferences y permite lanzar activities, services o broadcasts. Casi todo en Android necesita un Context para funcionar.

Existen distintas variantes: ApplicationContext (vive toda la app), Activity Context (vive con la Activity), Service Context. Guardar un Activity Context fuera de su ciclo de vida es la causa más común de memory leaks.

Ejemplo:
class Analytics(private val context: Context) {
    fun log(event: String) {
        context.applicationContext // seguro de guardar
    }
}

val service = context.getSystemService(Context.CONNECTIVITY_SERVICE)
        as ConnectivityManager

Consejo:
"Usa applicationContext para singletons; Activity context solo para UI/diálogos". Memory leak clásico: guardar this de una Activity en un objeto estático.

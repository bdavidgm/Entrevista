## Concepto
Retrofit convierte una interfaz HTTP en cliente tipado mediante anotaciones @GET, @POST, @Body, @Query. Se apoya en [OkHttp](pregunta:208) para conexión, interceptores y timeouts, y en un converter ([Moshi](pregunta:262)/[Gson](pregunta:261)) para JSON.

[Suspend](pregunta:79) functions o Call/[Flow](pregunta:84) integran bien con [corrutinas](pregunta:78).

Ejemplo:
interface Api {
    @GET("users/{id}")
    suspend fun user(@Path("id") id: String): UserDto
}

val retrofit = Retrofit.Builder()
    .baseUrl("https://api.example.com/")
    .addConverterFactory(MoshiConverterFactory.create())
    .client(okHttp)
    .build()
    .create(Api::class.java)

ExplicaciónKotlin:
`interface Api` declara contrato Retrofit con anotaciones Java (`@GET`, `@Path`, `@Body`) aplicables en Kotlin. `@GET("users/{id}")` mapea la ruta; `suspend fun user(@Path("id") id: String): UserDto` combina [función suspend](pregunta:79)ida con parámetro nombrado en la URL — Retrofit 2.6+ genera código que suspende la [corrutina](pregunta:78) en lugar de bloquear. El tipo de retorno `UserDto` es no anulable; errores HTTP pueden lanzar `HttpException`. `Retrofit.Builder()` encadena configuración fluida: `baseUrl(...)` exige barra final, `addConverterFactory(MoshiConverterFactory.create())` registra [Moshi](pregunta:262) para JSON, `client(okHttp)` inyecta [OkHttp](pregunta:208) personalizado. `.build().create(Api::class.java)` materializa el proxy: `Api::class.java` pasa la clase al factory Java. Todo el snippet es Kotlin puro sobre APIs Java de Square, listo para inyectar en repos con [corrutinas](pregunta:78).

Consejo:
Centraliza auth, logging y reintentos en interceptores [OkHttp](pregunta:208). Usa Response<T> si necesitas leer códigos HTTP sin excepción.

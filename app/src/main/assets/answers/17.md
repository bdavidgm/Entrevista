## Concepto
El operador safe call ?. accede a un miembro de una referencia solo si esta no es null; en caso contrario devuelve null sin lanzar excepción. Permite encadenar accesos sobre tipos anulables de forma segura y compacta.

Combinado con ?: ([Elvis](pregunta:2)), .let { } o asignaciones, expresa "haz esto solo si el valor existe".

Ejemplo:
val length = user?.address?.street?.length
val name = user?.name ?: "Anónimo"

user?.let { sendWelcome(it) }

ExplicaciónKotlin:
`user?.address?.street?.length` encadena safe calls: si cualquier eslabón es null, toda la expresión devuelve null sin excepción; el tipo inferido de `length` es `Int?`. `user?.name ?: "Anónimo"` combina safe call con [Elvis](pregunta:2) para obtener un `String` no anulable. `user?.let { sendWelcome(it) }` aplica `?.` a la scope function `let`: solo si `user` no es null se ejecuta el bloque lambda con `it` como alias del receptor; si es null, no se llama a `sendWelcome` y el resultado es null. `let` devuelve el resultado del bloque (aquí probablemente `Unit` de `sendWelcome`). Este patrón sustituye `if (user != null) sendWelcome(user)` con una expresión más compacta y [composable](pregunta:46) con otras operaciones funcionales.

Consejo:
"?. cortocircuita en null". Distinguirlo de !!: ?. devuelve null sin crash, !! lo asume.

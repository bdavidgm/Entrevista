## Concepto
Una coroutine es una unidad de trabajo ligera y suspendible que Kotlin ejecuta de forma cooperativa, permitiendo código asíncrono secuencial sin bloquear hilos. En Android sustituye callbacks anidados y simplifica operaciones de red y BD junto con [ViewModelScope](pregunta:131).

## Explicación detallada
Las coroutines se programan en [dispatchers](pregunta:30) (Main, IO, Default). Una [función suspend](pregunta:79) puede pausarse sin bloquear el hilo del sistema; al reanudarse continúa. Miles de coroutines pueden coexistir frente a pocos threads.

[CoroutineScope](pregunta:81) define ciclo de vida: [viewModelScope](pregunta:131), [lifecycleScope](pregunta:130), applicationScope. launch inicia trabajo fire-and-forget; async devuelve Deferred con resultado. [Structured concurrency](pregunta:81): los scopes padre cancelan hijos.

Integración Android: Main dispatcher actualiza UI tras [withContext](pregunta:82)(IO). [Retrofit](pregunta:207) y [Room](pregunta:70) exponen [suspend](pregunta:79). [Flow](pregunta:84) para streams múltiples de valores.

Errores: CoroutineExceptionHandler en scope raíz; try/catch en [suspend](pregunta:79). No uses GlobalScope en producción (sin lifecycle). [runBlocking](pregunta:142) solo en tests o main bloqueado puntual.

## Cuándo usarlo
Toda operación async en Kotlin moderno: red, BD, combinación de fuentes, debounce en [ViewModel](pregunta:68). Callback APIs legacy: suspendCoroutine o adaptadores.

Ejemplo:
viewModelScope.launch {
    _uiState.update { it.copy(loading = true) }
    val data = withContext(Dispatchers.IO) { repository.fetch() }
    _uiState.update { it.copy(loading = false, items = data) }
}

ExplicaciónKotlin:
[viewModelScope](pregunta:131).launch { ... } inicia coroutine en scope del [ViewModel](pregunta:68) cancelado en onCleared(). _uiState.update { it.copy(loading = true) } usa update sobre [MutableStateFlow](pregunta:86): lambda recibe it (estado actual) y copy de [data class](pregunta:3) con parámetro nombrado loading = true. [withContext](pregunta:82)([Dispatchers.IO](pregunta:30)) { [repository](pregunta:63).fetch() } es [función suspend](pregunta:79) que cambia dispatcher al pool IO para fetch() y devuelve resultado al caller en dispatcher original. [repository](pregunta:63).fetch() presumiblemente [suspend](pregunta:79). Segundo _uiState.update { it.copy(loading = false, items = data) } asigna items = data donde data es valor retornado por [withContext](pregunta:82). Encadena corrutinas, scope, [dispatchers](pregunta:30), lambdas y [data class](pregunta:3) copy sin bloquear Main thread durante IO.

Consejo para entrevista:
Explica [suspend](pregunta:79), scope y [dispatchers](pregunta:30). No digas que coroutine es un hilo. Seguimiento: diferencia thread vs coroutine — muchas coroutines, pocos threads.

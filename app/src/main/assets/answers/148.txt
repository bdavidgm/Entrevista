Concepto:
MainDispatcherRule (JUnit4) o MainDispatcherExtension (JUnit5) sustituye Dispatchers.Main en tests por un dispatcher controlado (StandardTestDispatcher o UnconfinedTestDispatcher). Así las corrutinas que usan Main se ejecutan de forma determinista sin el hilo real de Android.

Imprescindible para testear ViewModels que usan viewModelScope o withContext(Main).

Ejemplo:
@get:Rule
val mainDispatcherRule = MainDispatcherRule()

@Test
fun `carga estado`() = runTest {
    val vm = HomeViewModel(repo)
    vm.load()
    advanceUntilIdle()
    assertIs<HomeState.Success>(vm.state.value)
}

Consejo:
Con StandardTestDispatcher avanza el tiempo con advanceUntilIdle; con UnconfinedTestDispatcher todo corre de inmediato (menos control, más simple).

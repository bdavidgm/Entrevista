Concepto:
catch es un operador de Flow que intercepta excepciones lanzadas en el upstream y permite emitir valores alternativos, relanzar o completar. Solo atrapa errores que ocurren antes en la cadena; no sustituye un try/catch alrededor de collect si el lambda del collector lanza.

Es el sitio idóneo para mapear fallos de red a estados de error en la UI.

Ejemplo:
repository.observeUsers()
    .catch { e ->
        emit(emptyList()) // o log + emitAll(fallbackFlow())
    }
    .onEach { /* ... */ }


ExplicaciónKotlin:
`repository.observeUsers()` devuelve `Flow` sobre el cual encadenas `.catch { e -> emit(emptyList()) }`: `catch` es operador inline que recibe lambda `(Throwable) -> Unit` con capacidad de emitir valores alternativos. `e` es parámetro implícito nombrado del throwable capturado del upstream. `emit(emptyList())` dentro de `catch` requiere que el tipo coincida con el flujo (`List<User>`); `emptyList()` infiere genérico por contexto. `.onEach { /* ... */ }` continúa la cadena con side effect por elemento sin transformar. Los operadores devuelven `Flow` encadenable (fluent API). `catch` solo atrapa excepciones lanzadas en operadores anteriores, no en el lambda del collector. El comentario en `onEach` indica placeholder de lógica UI.

Consejo:
Si necesitas manejar errores también en el collector, combina catch upstream con try/catch en collect o usa Result.

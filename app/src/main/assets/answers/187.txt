Concepto:
La SplashScreen API (Android 12+) unifica la pantalla de arranque: icono animado, fondo y transición a la app. Se configura con tema Theme.SplashScreen y SplashScreen.installSplashScreen() en onCreate antes de super.

Evita pantallas splash duplicadas: la primera ventana ya es la splash del sistema.

Ejemplo:
class MainActivity : AppCompatActivity() {
    override fun onCreate(savedInstanceState: Bundle?) {
        val splash = installSplashScreen()
        splash.setKeepOnScreenCondition { viewModel.isLoading.value }
        super.onCreate(savedInstanceState)
    }
}

ExplicaciónKotlin:
`class MainActivity : AppCompatActivity()` declara una clase que hereda de Activity AndroidX; los dos puntos expresan herencia en Kotlin. `override fun onCreate(savedInstanceState: Bundle?)` redefine el método del ciclo de vida: `savedInstanceState` es anulable (`Bundle?`) porque puede ser null en arranque frío. `installSplashScreen()` es una función de extensión del SplashScreen API que debe invocarse antes de `super.onCreate`. `val splash` captura la instancia devuelta con tipo inferido. `splash.setKeepOnScreenCondition { viewModel.isLoading.value }` pasa una lambda sin parámetros (`() -> Boolean`) que el sistema evalúa para decidir si mantener la splash; accede a `viewModel.isLoading.value`, probablemente un `StateFlow` o `LiveData` expuesto desde un ViewModel. `super.onCreate(savedInstanceState)` delega en la implementación base después de configurar la splash. El orden importa: splash primero, super después, siguiendo la guía oficial de Android 12+.

Consejo:
setKeepOnScreenCondition solo mientras cargas lo mínimo; no bloquees arranque con red síncrona. En versiones antiguas, windowBackground clásico.

## Concepto
WorkManager es la librería Jetpack para trabajo diferido y garantizado (sync, subidas, limpiezas) que respeta restricciones de batería y red. Sustituye en muchos casos a [AlarmManager](pregunta:175)/[JobScheduler](pregunta:189) para tareas persistentes.

Ejemplo:
class SyncWorker(
    context: Context,
    params: WorkerParameters
) : CoroutineWorker(context, params) {
    override suspend fun doWork(): Result {
        repository.sync()
        return Result.success()
    }
}

val request = PeriodicWorkRequestBuilder<SyncWorker>(15, TimeUnit.MINUTES)
    .setConstraints(
        Constraints.Builder()
            .setRequiredNetworkType(NetworkType.CONNECTED)
            .build()
    )
    .build()

WorkManager.getInstance(context).enqueueUniquePeriodicWork(
    "sync",
    ExistingPeriodicWorkPolicy.KEEP,
    request
)

ExplicaciónKotlin:
`CoroutineWorker` permite `doWork` [suspend](pregunta:79). `PeriodicWorkRequestBuilder` programa trabajo periódico (mínimo ~15 min). `Constraints` exige red. `enqueueUniquePeriodicWork` evita duplicados con una política `KEEP`. Ver [WorkManager](pregunta:72).

Consejo:
Usa `OneTimeWorkRequest` para tareas puntuales y cadenas (`beginWith().then()`) para pipelines.

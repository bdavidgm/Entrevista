## Concepto
NotificationChannel agrupa notificaciones con el mismo id y permite al usuario silenciarlas o cambiar importancia por categoría (mensajes, promos, media). Obligatorio desde API 26; sin canal, la notificación no se muestra.

Crea canales en arranque o antes de la primera notificación; los nombres visibles los ve el usuario en Ajustes.

Ejemplo:
if (Build.VERSION.SDK_INT >= Build.VERSION_CODES.O) {
    val ch = NotificationChannel(
        "sync",
        "Sincronización",
        NotificationManager.IMPORTANCE_LOW
    ).apply { description = "Progreso de subida de archivos" }
    notificationManager.createNotificationChannel(ch)
}


ExplicaciónKotlin:
`if (Build.VERSION.SDK_INT >= Build.VERSION_CODES.O)` guarda creación de canal solo en API 26+. `NotificationChannel("sync", "Sincronización", NotificationManager.IMPORTANCE_LOW)` instancia con id, nombre visible e importancia enum. `.apply { description = "..." }` usa scope function `apply`: receiver es el canal, retorna el mismo objeto tras configurar propiedad `description`. `notificationManager.createNotificationChannel(ch)` registra canal en sistema. `apply` es idiomático Kotlin para configuración inicial encadenada. Las propiedades dentro del bloque se resuelven como miembros del receptor sin calificar. Comparación con `VERSION_CODES.O` evita magic number 26.

Consejo:
No borres y recrees canales en cada arranque con distinto id. IMPORTANCE_LOW evita heads-up intrusivos para tareas rutinarias.

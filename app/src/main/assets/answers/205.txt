Concepto:
Retrofit es una biblioteca de Square que genera implementaciones de interfaces HTTP a partir de anotaciones (@GET, @POST, @Path, @Query, @Body), integrándose con convertidores (Moshi/Gson) y OkHttp.

Explicación detallada:
Defines ApiService como interfaz Kotlin/Java. Retrofit.Builder configura baseUrl, call adapters (CoroutineCallAdapter, RxJava), converter factories.

Ventajas: código declarativo, tipado, fácil mockear en tests, interceptors OkHttp para auth/logging, soporte suspend para coroutines sin Callback.

Componentes típicos:
- Retrofit.create(ApiService::class.java)
- OkHttpClient con timeouts, certificate pinning, logging HttpLoggingInterceptor (solo debug)
- MoshiConverterFactory para JSON Kotlin-friendly

Manejo errores: HttpException en códigos 4xx/5xx; IOException red; envolver en Result o sealed class en repositorio.

Cuándo usarlo:
- Casi todo cliente REST en Android profesional.
- Alternativas: Ktor client (Kotlin puro), Apollo para GraphQL.

Ejemplo:
interface ApiService {
    @GET("users/{id}")
    suspend fun getUser(@Path("id") id: Long): UserDto

    @POST("users")
    suspend fun createUser(@Body body: CreateUserRequest): UserDto
}

val okHttp = OkHttpClient.Builder()
    .addInterceptor(HttpLoggingInterceptor().apply {
        level = HttpLoggingInterceptor.Level.BODY
    })
    .build()

val retrofit = Retrofit.Builder()
    .baseUrl("https://api.example.com/")
    .client(okHttp)
    .addConverterFactory(MoshiConverterFactory.create())
    .build()

val api = retrofit.create(ApiService::class.java)

Consejo para entrevista:
Menciona OkHttp debajo, suspend, interceptors para token refresh, y capa Repository que no expone Retrofit a UI. Habla de manejo de errores y timeouts. Si preguntan GraphQL, cita Apollo.

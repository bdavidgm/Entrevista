Concepto:
MediaStore es el ContentProvider del sistema para imágenes, vídeo y audio en almacenamiento compartido. Expone URIs y columnas estándar; accedes vía ContentResolver. En Android 10+ con scoped storage es la vía principal para bibliotecas multimedia sin permisos de almacenamiento amplio.

Scoped access: lee/escribe con URIs y permisos granulares (READ_MEDIA_* / pick visual media).

Ejemplo:
val values = ContentValues().apply {
    put(MediaStore.Images.Media.DISPLAY_NAME, "foto_${System.currentTimeMillis()}.jpg")
    put(MediaStore.Images.Media.MIME_TYPE, "image/jpeg")
}
val uri = contentResolver.insert(MediaStore.Images.Media.EXTERNAL_CONTENT_URI, values)
contentResolver.openOutputStream(uri!!)?.use { /* escribir JPEG */ }

ExplicaciónKotlin:
`ContentValues()` instancia el contenedor clave-valor Java con el constructor por defecto; el bloque `.apply { ... }` es una función de ámbito estándar de Kotlin que configura el receptor y devuelve el mismo objeto, evitando una variable temporal intermedia. Dentro del `apply`, `put(...)` encadena dos entradas: `DISPLAY_NAME` con interpolación de string `"foto_${System.currentTimeMillis()}.jpg"` — aquí `${...}` inserta el valor numérico del timestamp en el literal — y `MIME_TYPE` con el literal `"image/jpeg"`. `contentResolver.insert(...)` devuelve `Uri?`; la asignación a `val uri` conserva la nullabilidad. El operador `!!` en `uri!!` fuerza el valor no nulo para pasarlo a `openOutputStream`, con el riesgo de `KotlinNullPointerException` si el insert falló; en código de producción conviene `uri?.let { ... }`. La cadena `openOutputStream(...)?.use { ... }` combina safe call (`?.`) con la función de extensión `use` de Kotlin sobre `Closeable`, que cierra el stream automáticamente al salir del bloque lambda incluso si hay excepción.

Consejo:
Usa Photo Picker cuando baste con selección de medios sin permisos amplios. Relaciona con SAF para carpetas arbitrarias.

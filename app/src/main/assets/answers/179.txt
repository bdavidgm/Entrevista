Concepto:
ClipboardManager gestiona el portapapeles del sistema: copiar y leer texto (y en APIs recientes, tipos enriquecidos) entre apps, obtenido con getSystemService(CLIPBOARD_SERVICE).

Explicación detallada:
ClipData encapsula uno o más items (texto plano, URI, intent). Métodos:
- setPrimaryClip(clip): reemplaza contenido (desde Android 10 el acceso en background está restringido).
- getPrimaryClip() / getPrimaryClipDescription(): leer si tu app está en foreground o cumples restricciones.
- addPrimaryClipChangedListener: notificación cuando cambia (uso limitado en background).

Privacidad Android 10+: las apps no pueden leer el portapapeles en segundo plano; solo cuando es foreground o hay foco. Android 13+ muestra toast al usuario cuando otra app pega contenido sensible si hay políticas del OEM.

Buenas prácticas:
- Copiar solo lo necesario (código de verificación, enlace); no contraseñas sin aviso.
- Para pegar, ofrece botón explícito en lugar de leer clipboard al abrir (mejor UX y privacidad).
- Usa ClipData.newPlainText(label, text) con label descriptivo para accesibilidad.

Cuándo usarlo:
- Copiar enlace, número de pedido, código promocional.
- Editores que soporten pegar desde portapapeles con acción del usuario.
- Evitar monitoreo continuo del clipboard (anti-patrón y bloqueado).

Ejemplo:
fun copyToClipboard(context: Context, label: String, text: String) {
    val clipboard = context.getSystemService(ClipboardManager::class.java)
    val clip = ClipData.newPlainText(label, text)
    clipboard.setPrimaryClip(clip)
}

fun readClipboardText(context: Context): String? {
    val clipboard = context.getSystemService(ClipboardManager::class.java)
    val clip = clipboard.primaryClip ?: return null
    if (clip.itemCount == 0) return null
    return clip.getItemAt(0).coerceToText(context)?.toString()
}

Consejo para entrevista:
Menciona restricciones de background desde Android 10 y enfoque “user-initiated paste”. No digas que leerás el clipboard al iniciar la app para “detectar enlaces” sin consentimiento.

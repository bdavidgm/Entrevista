Concepto:
Un BroadcastReceiver es un componente que escucha y reacciona a broadcast intents globales o locales (eventos del sistema o de la app). Puede registrarse en el Manifest (estático, limitado en versiones recientes) o dinámicamente en runtime con registerReceiver.

Explicación detallada:
Ejemplos históricos: BOOT_COMPLETED, CONNECTIVITY_CHANGE — muchos implicit broadcasts ya no se entregan a manifest receivers por batería y seguridad. Patrón moderno: LocalBroadcastManager deprecado; usar Flow, SharedFlow, o explicit broadcasts dentro de la app.

ordered broadcasts, sticky (obsoletos en práctica). Para trabajo pesado, onReceive debe delegar rápidamente a goAsync o WorkManager porque el sistema limita tiempo en main thread.

Cuándo usarlo:
Reaccionar a pocos eventos del sistema donde API moderna no basta, notificaciones internas legacy, plugins SDK. Preferir callbacks específicos y WorkManager cuando sea posible.

Ejemplo:
class DownloadCompleteReceiver : BroadcastReceiver() {
    override fun onReceive(context: Context, intent: Intent) {
        if (intent.action == ACTION_DOWNLOAD_DONE) {
            val pending = goAsync()
            CoroutineScope(Dispatchers.IO).launch {
                try { processDownload() } finally { pending.finish() }
            }
        }
    }
}

// Registro dinámico en Activity
override fun onStart() {
    super.onStart()
    registerReceiver(receiver, IntentFilter(ACTION_DOWNLOAD_DONE), RECEIVER_NOT_EXPORTED)
}

## Concepto
NetworkCallback es la interfaz que recibes al registrar un callback en ConnectivityManager (registerDefaultNetworkCallback o registerNetworkCallback). Te avisa cuando cambia la red activa, se pierde o cambian capacidades (validada, medida, VPN).

Desregistra en onStop/onDestroy para no filtrar el [Context](pregunta:127).

Ejemplo:
val callback = object : ConnectivityManager.NetworkCallback() {
    override fun onAvailable(network: Network) { _online.value = true }
    override fun onLost(network: Network) { _online.value = false }
}
cm.registerDefaultNetworkCallback(callback)
// onDestroy: cm.unregisterNetworkCallback(callback)


ExplicaciónKotlin:
`object : ConnectivityManager.NetworkCallback()` crea subclase anónima de callback Java abstracto con overrides. `override fun onAvailable(network: Network)` y `onLost` mutan `_online.value` — probablemente `MutableStateFlow` o [LiveData](pregunta:69) backing. `network: Network` tipa parámetro del framework. `cm.registerDefaultNetworkCallback(callback)` registra para red por defecto del app. El comentario `// onDestroy: cm.unregisterNetworkCallback(callback)` documenta cleanup necesario. Las lambdas object expression permiten múltiples métodos [override](pregunta:247) en un solo bloque. Kotlin interoperabilidad Java: `NetworkCallback` es clase Java extendida con sintaxis `object :`.

Consejo:
Combina onCapabilitiesChanged con NET_CAPABILITY_VALIDATED para distinguir “Wi‑Fi con portal cautivo” de Internet real.

## Concepto
Timber es un wrapper ligero sobre `Log` de Android. Añade árboles (`Tree`) por build type: debug imprime en Logcat; release puede enviar a Crashlytics. Evita logs olvidados en producción.

Ejemplo:
class MyApp : Application() {
    override fun onCreate() {
        super.onCreate()
        if (BuildConfig.DEBUG) {
            Timber.plant(Timber.DebugTree())
        } else {
            Timber.plant(CrashlyticsTree())
        }
    }
}

Timber.d("Usuario %s abierto", userId)
Timber.e(error, "Fallo al sincronizar")

ExplicaciónKotlin:
`Timber.plant(...)` registra un `Tree`. `DebugTree` formatea con clase/línea. `Timber.d` / `Timber.e` son APIs estáticas tipadas; el formato `%s` evita concatenar strings si el nivel está desactivado. En release, un `Tree` custom puede reenviar a [Firebase Crashlytics](pregunta:212). Ver [Timber](pregunta:265).

Consejo:
Planta el árbol una sola vez en `Application.onCreate`. No registres tokens ni PII.

## Concepto
Koin es un framework de DI en Kotlin puro: módulos declaran single/factory/viewModel y se arranca con startKoin { modules(...) }. Resuelve dependencias en runtime (sin anotaciones ni procesador de código como Dagger/Hilt).

Ventaja: simplicidad y multiplataforma. Desventaja: errores de grafo en runtime y algo menos de optimización que Dagger.

Ejemplo:
val appModule = module {
    single { Retrofit.Builder()...build().create(Api::class.java) }
    viewModel { HomeViewModel(get()) }
}

startKoin { androidContext(this@App); modules(appModule) }

ExplicaciónKotlin:
`val appModule = module { ... }` declara un módulo Koin con DSL: el lambda receiver es `Module` y permite definiciones encadenadas. `single { Retrofit.Builder()...build().create(Api::class.java) }` registra singleton: el bloque factory se ejecuta una vez; `create(Api::class.java)` usa referencia de clase Kotlin. `viewModel { HomeViewModel(get()) }` registra ViewModel de Android; `get()` es función inline del DSL que resuelve dependencias por tipo inferido (`Api`). `startKoin { androidContext(this@App); modules(appModule) }` arranca el contenedor: `androidContext(this@App)` pasa la Application con label calificado `@App`; `modules(appModule)` registra el módulo. El punto y coma en la misma línea es separador de sentencias Kotlin válido. Koin usa reflexión en runtime, sin anotaciones ni KAPT, aprovechando lambdas y DSLs idiomáticos de Kotlin.

Consejo:
"Hilt = compile-time + Android integrado; Koin = runtime + ligero". Para apps grandes muchos equipos prefieren Hilt por seguridad de grafo.

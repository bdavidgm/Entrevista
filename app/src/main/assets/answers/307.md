## Concepto
Ktor Client es el cliente HTTP multiplataforma de Jetpack/Kotlin. Configuras engines (OkHttp, CIO, Darwin) y plugins (ContentNegotiation, Auth, Logging) con un DSL idiomático.

Ejemplo:
val client = HttpClient(OkHttp) {
    install(ContentNegotiation) {
        json(Json { ignoreUnknownKeys = true })
    }
    install(Logging) {
        level = LogLevel.INFO
    }
    defaultRequest {
        url("https://api.example.com/")
        header("Accept", "application/json")
    }
}

suspend fun fetchNotes(): List<NoteDto> =
    client.get("notes").body()

ExplicaciónKotlin:
`HttpClient(OkHttp) { }` crea el cliente con engine Android. `install(ContentNegotiation)` usa [kotlinx.serialization](pregunta:263) para JSON. `defaultRequest` fija base URL y headers. `client.get("notes").body()` es [suspend](pregunta:79) y deserializa a `List<NoteDto>`. Ver [Ktor Client](pregunta:270).

Consejo:
Cierra el client (`client.close()`) o compártelo como singleton. En Android puro, [Retrofit](pregunta:207)+[OkHttp](pregunta:208) sigue siendo muy habitual.

## Concepto
El plugin de Kotlin para Compose (org.jetbrains.kotlin.plugin.compose) habilita el compilador de Compose: transforma funciones @Composable, aplica optimizaciones y es requisito para proyectos Compose con Kotlin 2.0+.

## Explicación detallada
Jetpack Compose necesita un compilador que procesa anotaciones @Composable, genera código de recomposición, skippers y agrupa lambdas estables. Antes se configuraba composeCompilerOptions con versiones alineadas manualmente al compilador de Kotlin. Desde Kotlin 2.0 el Compose Compiler se integra como plugin de Kotlin unificado.

En build.gradle.kts del módulo:
plugins {
    alias(libs.plugins.kotlin.compose)
}

El plugin valida funciones Composable en tiempo de compilación, genera grupos de recomposición y soporta métricas de estabilidad en debug. Sin el plugin, el código Compose no compila o muestra errores del tipo que las funciones Composable requieren el compiler.

Relación con buildFeatures { compose = true }: el flag del Android Gradle Plugin activa dependencias y tooling de UI; el plugin Kotlin Compose activa el compilador. Ambos son necesarios en módulos con UI declarativa.

El BOM de Compose (platform) alinea versiones de androidx.compose.*; el plugin alinea el compiler con Kotlin.

## Cuándo usarlo
Siempre en módulos que usen Jetpack Compose (app, features con UI). No añadas el plugin en módulos pure-Kotlin de dominio o data sin @Composable.

Ejemplo:
// libs.versions.toml
[plugins]
kotlin-compose = { id = "org.jetbrains.kotlin.plugin.compose", version.ref = "kotlin" }

// app/build.gradle.kts
plugins {
    alias(libs.plugins.android.application)
    alias(libs.plugins.kotlin.android)
    alias(libs.plugins.kotlin.compose)
}
android { buildFeatures { compose = true } }
dependencies {
    implementation(platform(libs.androidx.compose.bom))
    implementation(libs.androidx.material3)
}


ExplicaciónKotlin:
En TOML, `[plugins] kotlin-compose = { id = "org.jetbrains.kotlin.plugin.compose", version.ref = "kotlin" }` vincula plugin Compose al compilador Kotlin 2.0+. En `build.gradle.kts`, `plugins { alias(libs.plugins.kotlin.compose) }` lo aplica al módulo. `android { buildFeatures { compose = true } }` activa flag AGP para tooling Compose. `dependencies { implementation(platform(...)); implementation(libs.androidx.material3) }` combina BOM y artifact sin versión suelta. El plugin Kotlin procesa `@Composable` en compilación; sin él, funciones composables no compilan. Separación clara: AGP habilita feature Android; plugin Kotlin habilita compiler transform.

Consejo para entrevista:
Distingue AGP (application plugin), Kotlin Android plugin y Kotlin Compose plugin. Con Kotlin 2.0 reduces desalineación Kotlin/Compose que causaba crashes sutiles en runtime.

Concepto:
WorkManager es la API recomendada para programar trabajo en background que debe completarse aunque la app se cierre o el dispositivo se reinicie. Soporta constraints (red, batería, carga, almacenamiento), reintentos con backoff y cadenas de tareas dependientes.

Sustituye a JobScheduler, AlarmManager y FirebaseJobDispatcher para trabajo diferible. Para tareas que deben ejecutarse en un instante exacto (alarmas, calendario), usa AlarmManager.

Ejemplo:
class SyncWorker(ctx: Context, params: WorkerParameters) : CoroutineWorker(ctx, params) {
    override suspend fun doWork(): Result {
        return try { repository.sync(); Result.success() }
        catch (e: Exception) { Result.retry() }
    }
}

val request = OneTimeWorkRequestBuilder<SyncWorker>()
    .setConstraints(Constraints.Builder().setRequiredNetworkType(NetworkType.CONNECTED).build())
    .build()
WorkManager.getInstance(context).enqueue(request)

Consejo:
"Diferible y persistente; respeta Doze y battery savers". Usa OneTimeWork o PeriodicWork según el caso y enqueueUniqueWork para evitar duplicados.

## Concepto
Dagger 2 es un framework de inyección de dependencias con codegen. [Hilt](pregunta:74) es una opinión/capa sobre Dagger para Android que reduce boilerplate (componentes estándar, `@AndroidEntryPoint`, `@HiltViewModel`).

Ejemplo:
@Module
@InstallIn(SingletonComponent::class)
object AppModule {
    @Provides @Singleton
    fun provideApi(): Api = retrofit.create()
}

ExplicaciónKotlin:
`@Module` + `@InstallIn(SingletonComponent::class)` registran providers en el grafo [Hilt](pregunta:74) (Dagger bajo el capó). `@Provides` expone `Api`. `@Singleton` acota el ciclo de vida.

Consejo:
En apps nuevas suele usarse [Hilt](pregunta:74); saber Dagger ayuda a depurar el grafo generado.

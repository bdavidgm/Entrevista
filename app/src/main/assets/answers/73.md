## Concepto
DataStore es la API moderna para guardar pequeños datos clave-valor o tipados, sustituta de [SharedPreferences](pregunta:177). Es asíncrona (basada en [coroutines](pregunta:78) y [Flow](pregunta:84)), thread-safe y soporta dos variantes: Preferences DataStore (clave-valor parecido a Prefs) y Proto DataStore (esquema tipado con protobuf).

Sus ventajas frente a [SharedPreferences](pregunta:177): no bloquea el hilo principal, ofrece manejo de errores y propaga los cambios como [Flow](pregunta:84).

Ejemplo:
val Context.dataStore by preferencesDataStore("settings")
val DARK = booleanPreferencesKey("dark")

class SettingsRepository(private val context: Context) {
    val isDark: Flow<Boolean> = context.dataStore.data.map { it[DARK] ?: false }
    suspend fun setDark(value: Boolean) {
        context.dataStore.edit { it[DARK] = value }
    }
}

Consejo:
"Sustituye [SharedPreferences](pregunta:177)". Para esquemas complejos prefiere Proto DataStore por la validación de tipos.

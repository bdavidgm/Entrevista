## Concepto
Un TypeConverter en [Room](pregunta:70) indica cómo convertir tipos que no soporta SQLite (Date, Enum, List, Json, UUID) a tipos primitivos almacenables y viceversa. Se declaran como funciones marcadas con @TypeConverter en una clase, y se registran con @TypeConverters en la entidad, DAO o base de datos.

Permite usar tipos ricos del dominio dentro de las entidades sin renunciar a la persistencia.

Ejemplo:
class Converters {
    @TypeConverter fun fromDate(value: Long?): Date? = value?.let(::Date)
    @TypeConverter fun toDate(date: Date?): Long? = date?.time
    @TypeConverter fun fromTags(value: String?): List<String> = value?.split(",") ?: emptyList()
    @TypeConverter fun toTags(value: List<String>): String = value.joinToString(",")
}

@Database(entities = [Note::class], version = 1)
@TypeConverters(Converters::class)
abstract class AppDb : RoomDatabase()


ExplicaciónKotlin:
`class Converters` es clase normal con funciones marcadas `@TypeConverter`; [Room](pregunta:70) las invoca por reflexión/generación. `fromDate(value: Long?): Date?` acepta nullable y usa `value?.let(::Date)` — [safe call](pregunta:17) más referencia al constructor `Date` como función (`::Date` es callable reference equivalente a `{ millis -> Date(millis) }`). `toDate(date: Date?): Long?` usa propiedad `time` de `Date`. `fromTags` combina [Elvis](pregunta:2) `?: emptyList()` para devolver lista vacía cuando el string es null; `split(",")` devuelve `List<String>`. `toTags` usa `joinToString(",")` sobre `List<String>`. `@Database(entities = [Note::class], version = 1)` pasa array de clases [KClass](pregunta:256) con literal `Note::class`. `abstract class AppDb : RoomDatabase()` hereda de clase base generada; Kotlin exige `abstract` porque [Room](pregunta:70) implementa métodos concretos.

Consejo:
Mantén los conversores pequeños y deterministas. Para JSON evita acoplarte al parser dentro del converter: extrae la lógica para poder testearla aislada.

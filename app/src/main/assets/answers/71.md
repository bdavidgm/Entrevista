## Concepto
Un DAO (Data Access Object) en Room es una interface o clase abstracta donde declaras los métodos para leer y escribir en la base de datos mediante anotaciones (@Query, @Insert, @Update, @Delete, @Upsert). Room genera la implementación en tiempo de compilación y verifica el SQL contra el esquema.

Los métodos pueden ser suspend (operación puntual) o devolver Flow para observar cambios de la tabla.

Ejemplo:
@Dao
interface UserDao {
    @Query("SELECT * FROM User WHERE id = :id") suspend fun byId(id: String): User?
    @Query("SELECT * FROM User") fun observeAll(): Flow<List<User>>
    @Upsert suspend fun upsert(user: User)
    @Delete suspend fun delete(user: User)
}

ExplicaciónKotlin:
@Dao interface UserDao define operaciones sin implementación manual. @Query("SELECT * FROM User WHERE id = :id") suspend fun byId(id: String): User? devuelve nullable User: el ? indica que puede no existir fila; :id en SQL vincula parámetro id. observeAll(): Flow<List<User>> contrasta con suspend byId: Flow para observación continua. @Upsert suspend fun upsert(user: User) usa anotación agregada de Room. @Delete suspend fun delete(user: User) recibe entidad completa. Todas las funciones de escritura son suspend; las lecturas puntual suspend y la observación Flow. interface sin cuerpo es válido en Kotlin; Room genera implementación. Parámetros nombrados implícitos en @Query con sintaxis :nombre.

Consejo:
"DAO = interfaz que Room implementa". Recuerda que las queries se verifican en compile-time y que los Flow se reemiten al cambiar la tabla.

## Concepto
Un DAO (Data Access Object) en [Room](pregunta:70) es una interface o clase abstracta donde declaras los métodos para leer y escribir en la base de datos mediante anotaciones (@Query, @Insert, @Update, @Delete, @Upsert). [Room](pregunta:70) genera la implementación en tiempo de compilación y verifica el SQL contra el esquema.

Los métodos pueden ser [suspend](pregunta:79) (operación puntual) o devolver [Flow](pregunta:84) para observar cambios de la tabla.

Ejemplo:
@Dao
interface UserDao {
    @Query("SELECT * FROM User WHERE id = :id") suspend fun byId(id: String): User?
    @Query("SELECT * FROM User") fun observeAll(): Flow<List<User>>
    @Upsert suspend fun upsert(user: User)
    @Delete suspend fun delete(user: User)
}

ExplicaciónKotlin:
@Dao interface UserDao define operaciones sin implementación manual. @Query("SELECT * FROM User WHERE id = :id") [suspend](pregunta:79) fun byId(id: String): User? devuelve nullable User: el ? indica que puede no existir fila; :id en SQL vincula parámetro id. observeAll(): [Flow](pregunta:84)<List<User>> contrasta con [suspend](pregunta:79) byId: [Flow](pregunta:84) para observación continua. @Upsert [suspend](pregunta:79) fun upsert(user: User) usa anotación agregada de [Room](pregunta:70). @Delete [suspend](pregunta:79) fun delete(user: User) recibe entidad completa. Todas las funciones de escritura son [suspend](pregunta:79); las lecturas puntual [suspend](pregunta:79) y la observación [Flow](pregunta:84). interface sin cuerpo es válido en Kotlin; [Room](pregunta:70) genera implementación. Parámetros nombrados implícitos en @Query con sintaxis :nombre.

Consejo:
"DAO = interfaz que [Room](pregunta:70) implementa". Recuerda que las queries se verifican en compile-time y que los [Flow](pregunta:84) se reemiten al cambiar la tabla.

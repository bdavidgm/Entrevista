Concepto:
viewModelScope es un CoroutineScope ligado al ViewModel: las coroutines lanzadas en él se cancelan automáticamente cuando se llama a onCleared(), es decir, cuando el componente que lo posee desaparece del todo. Usa Dispatchers.Main.immediate por defecto.

Es el sitio adecuado para orquestar llamadas a repositorios y producir StateFlow para la UI, dejando que el cambio de configuración no cancele el trabajo.

Ejemplo:
class HomeViewModel(private val repo: Repository) : ViewModel() {
    private val _state = MutableStateFlow<HomeState>(HomeState.Loading)
    val state = _state.asStateFlow()

    fun load() = viewModelScope.launch {
        _state.value = HomeState.Success(repo.fetch())
    }
}


ExplicaciónKotlin:
`HomeViewModel` declara un constructor primario con parámetro `repo: Repository`, patrón habitual de inyección manual o DI. `_state` es `private val` sobre `MutableStateFlow<HomeState>`, donde el genérico fija el tipo sellado o de datos del estado UI; la inicialización usa el objeto `HomeState.Loading`. `val state = _state.asStateFlow()` expone solo lectura mediante conversión a `StateFlow`, ocultando `value` mutable desde fuera. La función `load()` devuelve un `Job` implícito gracias a `= viewModelScope.launch { ... }`: expresión de una sola línea con bloque trailing. Dentro del bloque suspendido, `_state.value = ...` asigna atómicamente un nuevo valor al `StateFlow`, lo que notifica a collectors; `repo.fetch()` debe ser `suspend` para poder invocarse sin bloquear. `viewModelScope` es una propiedad delegada del `ViewModel` que cancela corrutinas en `onCleared()`.

Consejo:
"viewModelScope = vida del ViewModel". No uses GlobalScope: pierdes la cancelación y filtras trabajo cuando el usuario sale.

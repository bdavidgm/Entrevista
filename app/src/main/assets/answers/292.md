## Concepto
Room es la librería oficial de Jetpack para persistencia sobre SQLite. Define tablas con `@Entity`, operaciones tipadas con `@Dao` y el punto de entrada con `@Database`. Genera el código en compile-time y se integra con [coroutines](pregunta:78) y [Flow](pregunta:84).

Ejemplo:
@Entity(tableName = "notes")
data class NoteEntity(
    @PrimaryKey(autoGenerate = true) val id: Long = 0,
    val title: String,
    val body: String
)

@Dao
interface NoteDao {
    @Query("SELECT * FROM notes ORDER BY id DESC")
    fun observeAll(): Flow<List<NoteEntity>>

    @Insert(onConflict = OnConflictStrategy.REPLACE)
    suspend fun upsert(note: NoteEntity): Long
}

@Database(entities = [NoteEntity::class], version = 1)
abstract class AppDatabase : RoomDatabase() {
    abstract fun noteDao(): NoteDao
}

val db = Room.databaseBuilder(context, AppDatabase::class.java, "app.db").build()

ExplicaciónKotlin:
`@Entity` marca la [data class](pregunta:3) como tabla; `@PrimaryKey(autoGenerate = true)` genera el id. `@Dao` declara el contrato que Room implementa. `observeAll(): Flow<List<NoteEntity>>` emite listas reactivas; `upsert` es [suspend](pregunta:79) para no bloquear el hilo principal. `Room.databaseBuilder(...).build()` crea el singleton de la base. Para más detalle conceptual ver [Room](pregunta:70) y [DAO](pregunta:71).

Consejo:
En producción define migraciones explícitas; evita `fallbackToDestructiveMigration` en release.

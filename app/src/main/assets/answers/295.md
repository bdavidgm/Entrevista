## Concepto
Retrofit convierte una interfaz Kotlin/Java en un cliente HTTP tipado. Usa anotaciones (`@GET`, `@POST`, `@Path`, `@Body`) y se apoya en [OkHttp](pregunta:208) y un converter JSON ([Moshi](pregunta:262), [Gson](pregunta:261) o [kotlinx.serialization](pregunta:263)).

Ejemplo:
interface NotesApi {
    @GET("notes")
    suspend fun getNotes(): List<NoteDto>

    @POST("notes")
    suspend fun createNote(@Body note: NoteDto): NoteDto
}

val api = Retrofit.Builder()
    .baseUrl("https://api.example.com/")
    .client(okHttpClient)
    .addConverterFactory(MoshiConverterFactory.create())
    .build()
    .create(NotesApi::class.java)

suspend fun load() = api.getNotes()

ExplicaciónKotlin:
La interfaz declara el contrato HTTP. `suspend fun` integra con [coroutines](pregunta:78). `@Body` serializa el DTO a JSON vía el converter. `Retrofit.Builder()` configura base URL, cliente [OkHttp](pregunta:208) y factory. `.create(NotesApi::class.java)` genera el proxy. Ver [Retrofit](pregunta:207) y [Retrofit + Coroutines](pregunta:209).

Consejo:
Centraliza auth y logging en interceptores de OkHttp; usa `Response<T>` si necesitas el código HTTP.

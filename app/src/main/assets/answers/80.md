## Concepto
launch y async son builders de coroutines en un CoroutineScope: launch inicia una coroutine sin devolver resultado útil (efectos secundarios); async devuelve Deferred<T> para obtener un valor más tarde con await(), útil para paralelizar tareas independientes.

## Explicación detallada
launch { } es fire-and-forget dentro del scope: si falla y no hay handler, puede propagar según el scope. Ideal para actualizar UI state tras operaciones, escribir en BD, o disparar eventos.

async { } calcula valor en paralelo; await() suspende hasta completar. Paralelizar dos redes: val a = async { api1() }; val b = async { api2() }; val result = a.await() to b.await() en el mismo scope.

Structured concurrency: async hijos se cancelan si el scope padre cancela antes de await. No uses async sin await (fugas y excepciones silenciosas).

Diferencia con runBlocking: builders para scopes con lifecycle, no bloquean el caller thread salvo runBlocking.

En ViewModel: launch en viewModelScope es lo más común; async para combinar repositorios. supervisorScope si un hijo falla y otros deben continuar.

## Cuándo usarlo
launch para trabajo sin retorno a caller; async cuando necesitas resultado combinado o paralelismo medible. Un solo suspend secuencial no necesita async.

Ejemplo:
viewModelScope.launch {
    val profile = async(Dispatchers.IO) { userRepo.profile() }
    val orders = async(Dispatchers.IO) { orderRepo.recent() }
    _uiState.value = HomeUiState(profile.await(), orders.await())
}

ExplicaciónKotlin:
viewModelScope.launch { val profile = async(Dispatchers.IO) { userRepo.profile() }; val orders = async(Dispatchers.IO) { orderRepo.recent() }; _uiState.value = HomeUiState(profile.await(), orders.await()) } lanza coroutine padre y dos async hijos en paralelo en IO. async devuelve Deferred<T>; await() es suspend que espera resultado. profile y orders se ejecutan concurrentemente hasta await. _uiState.value = HomeUiState(...) asigna data class HomeUiState con dos resultados; si _uiState es MutableStateFlow, .value es setter. HomeUiState(profile.await(), orders.await()) invoca constructor posicional. Structured concurrency: si el scope padre cancela, los async hijos también. Dispatchers.IO se pasa explícitamente a cada async.

Consejo para entrevista:
launch vs async = sin vs con Deferred. Menciona await y cancelación. Trampa: async en GlobalScope sin await.

## Concepto
launch y async son builders de [coroutines](pregunta:78) en un [CoroutineScope](pregunta:81): launch inicia una [coroutine](pregunta:78) sin devolver resultado útil (efectos secundarios); async devuelve Deferred<T> para obtener un valor más tarde con await(), útil para paralelizar tareas independientes.

## Explicación detallada
launch { } es fire-and-forget dentro del scope: si falla y no hay [handler](pregunta:159), puede propagar según el scope. Ideal para actualizar UI state tras operaciones, escribir en BD, o disparar eventos.

async { } calcula valor en paralelo; await() suspende hasta completar. Paralelizar dos redes: val a = async { api1() }; val b = async { api2() }; val result = a.await() to b.await() en el mismo scope.

[Structured concurrency](pregunta:81): async hijos se cancelan si el scope padre cancela antes de await. No uses async sin await (fugas y excepciones silenciosas).

Diferencia con [runBlocking](pregunta:142): builders para scopes con lifecycle, no bloquean el caller thread salvo [runBlocking](pregunta:142).

En [ViewModel](pregunta:68): launch en [viewModelScope](pregunta:131) es lo más común; async para combinar repositorios. supervisorScope si un hijo falla y otros deben continuar.

En el contexto de proyectos Android reales, combina esta idea con las guías oficiales de Android Developers y con métricas de calidad (crashes, [ANR](pregunta:95), Vitals) cuando aplique. ## Cuándo usarlo
launch para trabajo sin retorno a caller; async cuando necesitas resultado combinado o paralelismo medible. Un solo [suspend](pregunta:79) secuencial no necesita async.

Ejemplo:
viewModelScope.launch {
    val profile = async(Dispatchers.IO) { userRepo.profile() }
    val orders = async(Dispatchers.IO) { orderRepo.recent() }
    _uiState.value = HomeUiState(profile.await(), orders.await())
}

ExplicaciónKotlin:
[viewModelScope](pregunta:131).launch { val profile = async([Dispatchers.IO](pregunta:30)) { userRepo.profile() }; val orders = async([Dispatchers.IO](pregunta:30)) { orderRepo.recent() }; _uiState.value = HomeUiState(profile.await(), orders.await()) } lanza [coroutine](pregunta:78) padre y dos async hijos en paralelo en IO. async devuelve Deferred<T>; await() es [suspend](pregunta:79) que espera resultado. profile y orders se ejecutan concurrentemente hasta await. _uiState.value = HomeUiState(...) asigna [data class](pregunta:3) HomeUiState con dos resultados; si _uiState es [MutableStateFlow](pregunta:86), .value es setter. HomeUiState(profile.await(), orders.await()) invoca constructor posicional. [Structured concurrency](pregunta:81): si el scope padre cancela, los async hijos también. [Dispatchers.IO](pregunta:30) se pasa explícitamente a cada async.

Consejo para entrevista:
launch vs async = sin vs con Deferred. Menciona await y cancelación. Trampa: async en GlobalScope sin await.

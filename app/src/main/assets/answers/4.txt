Concepto:
val declara una referencia de solo lectura (no se puede reasignar tras inicializarla), mientras que var declara una referencia mutable que sí admite reasignación. val no implica inmutabilidad profunda: el objeto referenciado puede seguir cambiando su estado interno.

Por defecto se prefiere val en Kotlin para favorecer la inmutabilidad y reducir errores de concurrencia y de razonamiento.

Ejemplo:
val list = mutableListOf(1, 2)
list.add(3)        // permitido: el objeto cambia
// list = mutableListOf() // error: no se puede reasignar

var counter = 0
counter += 1       // permitido

Consejo:
"val == final reference, no deep immutable". Para inmutabilidad real usa val + tipos inmutables (List, data class con val).

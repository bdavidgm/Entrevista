Concepto:
En Kotlin, == compara igualdad estructural (equivale a llamar a equals), mientras que === compara igualdad referencial (mismo objeto en memoria). Es lo opuesto a Java, donde == compara referencias.

Para tipos sin equals personalizado, == acaba comparando referencias, pero en data classes equals compara los valores de las propiedades.

Ejemplo:
val a = "hola"
val b = String(a.toCharArray())

a == b   // true (mismo contenido)
a === b  // false (objetos distintos)

data class Point(val x: Int, val y: Int)
Point(1, 2) == Point(1, 2) // true

Consejo:
"== es equals; === es ==", al revés que Java. Útil para optimizaciones (chequeo rápido de identidad antes de equals costoso).

## Concepto
El operador :: crea una referencia a un símbolo (clase, función, propiedad o constructor) en lugar de invocarlo. Se usa para pasar funciones como valores (function references), obtener KClass (MyClass::class) o referenciar propiedades para reflexión.

Las referencias a funciones encajan donde se espera un tipo función (Lambda), pero suelen ser más legibles y reutilizables.

Ejemplo:
fun isOdd(n: Int) = n % 2 != 0

val odds = listOf(1, 2, 3).filter(::isOdd)

val classRef: KClass<User> = User::class
val nameProp = User::name

users.sortedBy(User::name)
button.setOnClickListener(::onButtonClick)

ExplicaciónKotlin:
`fun isOdd(n: Int) = n % 2 != 0` es una función de nivel superior con cuerpo de expresión única. `listOf(1, 2, 3).filter(::isOdd)` pasa una function reference: `::isOdd` es equivalente a la lambda `{ n -> isOdd(n) }` pero más concisa y reutilizable; el tipo esperado es `(Int) -> Boolean`. `User::class` produce una `KClass<User>` (metadatos Kotlin); `.java` obtiene `Class<User>` para APIs Java. `User::name` es una property reference usable donde se espera `(User) -> String`, por eso `sortedBy(User::name)` ordena extrayendo la propiedad. `::onButtonClick` adapta un método de instancia si está en scope, o referencia estática según contexto. Las referencias conservan la firma exacta y evitan allocations de lambdas en algunos escenarios.

Consejo:
"::funcion = function reference; Tipo::class = KClass". Para acceder a Class<T> Java usa Tipo::class.java.

## Concepto
Flow es un tipo frío (cold stream) de Kotlin que emite múltiples valores de forma secuencial y asíncrona, con soporte de operadores (map, filter, combine) y [coroutines](pregunta:78). En Android reemplaza a [LiveData](pregunta:69) para muchos casos reactivos con backpressure y composición.

## Explicación detallada
flow { emit(1); delay(100); emit(2) } no ejecuta hasta que un collector hace collect. Cada collector puede tener su propia ejecución (cold). [StateFlow](pregunta:86) y [SharedFlow](pregunta:86) son hot: mantienen valor sin collector activo.

Operadores: map, filter, debounce, distinctUntilChanged, catch, onEach, flatMapLatest (cancela emisiones previas al cambiar búsqueda). [flowOn](pregunta:145) cambia dispatcher upstream.

[Room](pregunta:70) observeAll(): Flow<List<Entity>> re-emite al cambiar tabla. [ViewModel](pregunta:68) expone [StateFlow](pregunta:86) derivado de Flow con stateIn SharingStarted.WhileSubscribed.

Lifecycle: collect en repeatOnLifecycle(STARTED) o [collectAsStateWithLifecycle](pregunta:134) en Compose. Nunca collect infinito en onCreate sin lifecycle.

Testing: [Turbine](pregunta:93) para assert emisiones. testScheduler para delay.

## Cuándo usarlo
Streams de datos, búsqueda con debounce, combinación de sensores, UI reactiva. Un solo valor async: [suspend](pregunta:79) basta.

Ejemplo:
fun searchFlow(query: Flow<String>): Flow<List<Result>> = query
    .debounce(300)
    .distinctUntilChanged()
    .flatMapLatest { q ->
        flow {
            emit(repository.search(q))
        }
    }

viewModelScope.launch {
    searchFlow(queryState).collect { results ->
        _uiState.update { it.copy(results = results) }
    }
}

ExplicaciónKotlin:
fun searchFlow(query: Flow<String>): Flow<List<Result>> = query.debounce(300).distinctUntilChanged().flatMapLatest { q -> flow { emit([repository](pregunta:63).search(q)) } } declara función que transforma Flow con operadores encadenados. debounce(300) suspende emisiones 300 ms; distinctUntilChanged() filtra repetidos consecutivos. flatMapLatest { q -> flow { emit([repository](pregunta:63).search(q)) } } cancela búsqueda anterior cuando query cambia; q es String emitido; flow { emit(...) } builder crea Flow frío que emite lista de Result desde [repository](pregunta:63).search(q) [suspend](pregunta:79) dentro del builder. [viewModelScope](pregunta:131).launch { searchFlow(queryState).collect { results -> _uiState.update { it.copy(results = results) } } } colecta en scope y actualiza estado con copy. queryState presumiblemente Flow o [StateFlow](pregunta:86) pasado como argumento.

Consejo para entrevista:
Cold vs hot, [flowOn](pregunta:145), flatMapLatest para búsqueda. Contrasta con [LiveData](pregunta:69). Seguimiento: stateIn y WhileSubscribed.

## Concepto
OkHttp es el cliente HTTP de bajo nivel que usa [Retrofit](pregunta:207) (y muchas apps) para conexiones, timeouts, cookies e interceptores. Permite añadir headers, logging y reintentos de forma centralizada.

Ejemplo:
val logging = HttpLoggingInterceptor().apply {
    level = HttpLoggingInterceptor.Level.BODY
}

val client = OkHttpClient.Builder()
    .connectTimeout(20, TimeUnit.SECONDS)
    .readTimeout(20, TimeUnit.SECONDS)
    .addInterceptor { chain ->
        val request = chain.request().newBuilder()
            .header("Authorization", "Bearer $token")
            .build()
        chain.proceed(request)
    }
    .addInterceptor(logging)
    .build()

ExplicaciónKotlin:
`OkHttpClient.Builder()` configura timeouts e interceptores. El lambda `addInterceptor { chain -> ... }` modifica la request (aquí añade Authorization) y llama `chain.proceed`. `HttpLoggingInterceptor` registra el cuerpo en debug. `apply { }` es una [scope function](pregunta:21). Este `client` se pasa a [Retrofit](pregunta:207) con `.client(client)`.

Consejo:
No dejes el logging BODY en release (puede filtrar tokens). Para pinning de certificados ver [certificate pinning](pregunta:154).

## Concepto
Un [Intent](pregunta:33) explícito indica exactamente qué componente (clase) debe atenderlo; un [Intent](pregunta:33) implícito describe una acción y deja que el sistema elija un componente capaz de ejecutarla mediante [intent](pregunta:33) filters declarados en el [AndroidManifest](pregunta:35).

Los explícitos se usan para navegar dentro de la propia app; los implícitos para acciones genéricas (compartir, abrir un enlace, capturar foto) que pueden ser atendidas por varias apps.

Ejemplo:
// Explícito
startActivity(Intent(this, DetailActivity::class.java))

// Implícito
startActivity(Intent(Intent.ACTION_VIEW, Uri.parse("https://example.com")))

ExplicaciónKotlin:
`startActivity(Intent(this, DetailActivity::class.java))` muestra un [intent](pregunta:33) explícito: el constructor recibe contexto y la clase concreta resuelta con `::class.java`, patrón estándar en Kotlin para navegación intra-app. El comentario `// Explícito` es documentación en código. La segunda llamada usa el constructor `Intent(action, uri)` para un [intent](pregunta:33) implícito: `Intent.ACTION_VIEW` es una constante String y `Uri.parse("https://example.com")` convierte el literal en `Uri`. Kotlin infiere tipos en la llamada a `startActivity`. No hay lambda ni scope function aquí: es API directa del framework, pero la obtención de la clase con `DetailActivity::class.java` es el idioma kotliniano frente a `DetailActivity.class` en Java. La diferencia explícito/implícito es de Android, no de Kotlin, pero la sintaxis de referencia a clase es propia del lenguaje.

Consejo:
Comprueba siempre resolveActivity antes de lanzar un implícito para evitar ActivityNotFoundException. Recuerda PackageVisibility (queries en manifest a partir de Android 11).

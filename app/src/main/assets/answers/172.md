## Concepto
CameraX es la capa Jetpack sobre Camera2 con casos de uso (Preview, ImageCapture, ImageAnalysis) y bindToLifecycle, que abre/cierra la cámara según STARTED/STOPPED. Oculta buena parte de la complejidad OEM y unifica comportamiento.

Incluye extensiones opcionales (HDR, noche) si el dispositivo las soporta.

Ejemplo:
val cameraProvider = ProcessCameraProvider.getInstance(context).get()
val preview = Preview.Builder().build().also { it.setSurfaceProvider(surfaceProvider) }
cameraProvider.bindToLifecycle(lifecycleOwner, CameraSelector.DEFAULT_BACK_CAMERA, preview)


ExplicaciónKotlin:
`ProcessCameraProvider.getInstance(context).get()` igual que variante anterior con `context` genérico. `Preview.Builder().build().also { it.setSurfaceProvider(surfaceProvider) }` separa surface provider como variable/lambda externa — útil en Compose con `PreviewView`. `cameraProvider.bindToLifecycle(lifecycleOwner, CameraSelector.DEFAULT_BACK_CAMERA, preview)` usa owner explícito en lugar de `this`, patrón recomendado cuando el composable no es LifecycleOwner directo. Argumentos vararg de use cases permiten añadir `ImageCapture` etc. Kotlin mantiene fluidez con builders Java de CameraX. `get()` sincrónico puede ANR si CameraProvider tarda; en código real usar `addListener`.

Consejo:
Menciona ImageAnalysis para ML/QR y que Camera2 directo sigue existiendo para control fino. En Compose envuelve PreviewView con AndroidView.

Concepto:
flowOn es un operador de Flow que cambia el CoroutineContext (típicamente el Dispatcher) en el que se ejecuta el upstream del operador, sin afectar al downstream que sigue en el contexto del collector.

Explicación detallada:
flow {
    // trabajo pesado
    emit(compute())
}.flowOn(Dispatchers.Default)
 .map { it.toUiModel() }  // puede seguir en Default si no hay otro flowOn
 .flowOn(Dispatchers.IO)   // upstream de este punto en IO
 .collect { }             // collector en el scope del llamador (Main en UI)

Reglas:
- flowOn solo afecta operadores anteriores en la cadena (upstream).
- Múltiples flowOn: el más cercano al emit define contexto de esa sección.
- No cambia el contexto de collect; para eso usa withContext en el collector o launch en Main.

Buenas prácticas Android:
- Repository: flow { emit(dao.getAll()) }.flowOn(Dispatchers.IO)
- ViewModel collect en viewModelScope (Main) y mapea a UiState.

No confundir con subscribeOn de Rx (afecta distinto). En Flow el collector define hilo final de entrega salvo operadores que cambien contexto downstream (raro).

Cuándo usarlo:
Operaciones de BD, red o CPU en Flow sin bloquear Main. Encadenar tras map pesado en capa de datos.

Ejemplo:
class NewsRepository @Inject constructor(private val api: NewsApi, private val dao: NewsDao) {
    fun headlines(): Flow<List<Article>> = flow {
        emit(api.fetchHeadlines())
    }.flowOn(Dispatchers.IO)
     .map { articles ->
         dao.insertAll(articles)
         articles
     }
     .flowOn(Dispatchers.IO)
}

Consejo para entrevista:
Dibuja upstream vs downstream. Menciona que collect en ViewModel suele estar en Main y flowOn protege el hilo UI.

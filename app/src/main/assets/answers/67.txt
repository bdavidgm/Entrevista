Concepto:
Inyectar dependencias por constructor (constructor injection) hace explícitas las colaboraciones de una clase y las convierte en parte de su API. Frente a singletons globales o service locators, mejora la testabilidad (puedes pasar fakes en tests) y evita acoplamientos ocultos.

Frameworks como Hilt o Koin generan o resuelven esos constructores, pero el patrón funciona incluso sin DI: si tu clase compila sin tocar nada estático, ya es inyectable.

Ejemplo:
class GetUser @Inject constructor(
    private val repo: UserRepository,
    private val analytics: Analytics
) {
    suspend operator fun invoke(id: String): User {
        analytics.track("get_user")
        return repo.get(id)
    }
}

Consejo:
"Constructor sobre setter, setter sobre Service Locator". Si tienes muchos parámetros, repiensa la responsabilidad de la clase (SRP).

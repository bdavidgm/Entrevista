## Concepto
AudioManager controla volumen y rutas de audio: STREAM_MUSIC, MODE_IN_CALL, solicitud de foco audio con requestAudioFocus para pausar o duck otros reproductores, y gestión de dispositivos Bluetooth/USB.

El foco es cooperativo: otras apps pueden ignorarlo, pero es buena ciudadanía.

Ejemplo:
val am = getSystemService(AudioManager::class.java)
am.adjustStreamVolume(AudioManager.STREAM_MUSIC, AudioManager.ADJUST_RAISE, AudioManager.FLAG_SHOW_UI)

val afChange = AudioManager.OnAudioFocusChangeListener { /* duck o pause */ }
am.requestAudioFocus(afChange, AudioManager.STREAM_MUSIC, AudioManager.AUDIOFOCUS_GAIN)

ExplicaciónKotlin:
`getSystemService(AudioManager::class.java)` sigue el patrón tipado de servicios. `adjustStreamVolume(...)` recibe constantes estáticas Java como argumentos enteros; `FLAG_SHOW_UI` muestra el diálogo de volumen. `val afChange = AudioManager.OnAudioFocusChangeListener { /* duck o pause */ }` usa la interfaz funcional SAM de Java: Kotlin traduce el listener en una lambda con un parámetro entero (el tipo de cambio de foco) cuyo cuerpo está comentado en el ejemplo. La lambda se convierte automáticamente en instancia de `OnAudioFocusChangeListener`. `requestAudioFocus(afChange, STREAM_MUSIC, AUDIOFOCUS_GAIN)` registra el listener y solicita foco permanente. Las variables `val am` y `val afChange` son inmutables; el listener captura el ámbito léxico donde se define (closure) para reaccionar a cambios de foco desde otras apps.

Consejo:
En Android 8+ AudioFocusRequest builder. Para VoIP/media, combina con [Foreground Service](pregunta:223) tipo mediaPlayback.

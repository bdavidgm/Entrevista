## Concepto
Paging 3 carga listas grandes por páginas (red o DB) y las muestra en [LazyColumn](pregunta:52)/RecyclerView sin traer todo a memoria. Integra [Room](pregunta:70), [Retrofit](pregunta:207) y Compose.

Ejemplo:
class NotesPagingSource(
    private val api: NotesApi
) : PagingSource<Int, NoteDto>() {
    override suspend fun load(params: LoadParams<Int>): LoadResult<Int, NoteDto> {
        val page = params.key ?: 1
        return try {
            val data = api.getNotes(page = page, size = params.loadSize)
            LoadResult.Page(
                data = data,
                prevKey = if (page == 1) null else page - 1,
                nextKey = if (data.isEmpty()) null else page + 1
            )
        } catch (e: Exception) {
            LoadResult.Error(e)
        }
    }
    override fun getRefreshKey(state: PagingState<Int, NoteDto>): Int? =
        state.anchorPosition
}

val pager = Pager(PagingConfig(pageSize = 20)) { NotesPagingSource(api) }.flow

ExplicaciónKotlin:
`PagingSource` define cómo cargar una página [suspend](pregunta:79). `LoadResult.Page` indica datos y claves prev/next. `Pager` + `PagingConfig` produce un [Flow](pregunta:84)`<PagingData<...>>` que en Compose se consume con `collectAsLazyPagingItems()`. Ver [Paging](pregunta:76).

Consejo:
Combina RemoteMediator + Room cuando quieras cache offline de páginas remotas.

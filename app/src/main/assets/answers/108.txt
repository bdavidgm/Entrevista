Concepto:
VectorDrawable es un recurso XML escalable que describe gráficos con paths en lugar de mapas de bits, ideal para iconos e ilustraciones simples en múltiples densidades sin assets png duplicados.

Explicación detallada:
Un solo archivo res/drawable/ic_add.xml escala sin pixelación. VectorDrawableCompat en versiones antiguas. AnimatedVectorDrawable para animaciones vectoriales ligeras.

En Compose: ImageVector desde Icons o vectorResource(R.drawable.ic_add). Material Icons extended. Limitación: paths muy complejos aumentan coste de render; fotos siguen siendo WebP/PNG.

app:srcCompat en ImageView en lugar de android:src para compatibilidad. tint para colorizar iconos con tema.

Cuándo usarlo:
Iconos UI, logos simples, ilustraciones flat. No para fotos ni gradientes complejos masivos.

Ejemplo:
// res/drawable/ic_launcher_foreground.xml — vector paths

@Composable
fun AddIcon() {
    Icon(
        imageVector = Icons.Default.Add,
        contentDescription = null
    )
}

// XML ImageView
<ImageView
    android:layout_width="24dp"
    android:layout_height="24dp"
    app:srcCompat="@drawable/ic_add"
    app:tint="?attr/colorPrimary" />


ExplicaciónKotlin:
`@Composable fun AddIcon()` declara composable sin parámetros. `Icon(imageVector = Icons.Default.Add, contentDescription = null)` pasa `null` explícito a `contentDescription` indicando icono decorativo (TalkBack lo ignora). `Icons.Default.Add` resuelve vector Material predefinido como `ImageVector`. El comentario XML `ImageView` no es Kotlin; en Views tradicionales `app:srcCompat` carga vector drawable. En Compose, equivalente es `Icon` o `Image(painterResource(...))`. `contentDescription = null` usa nullability del tipo `String?` para accesibilidad. Función de una expresión implícita si tuviera solo Icon, aquí es bloque con una llamada.

Consejo para entrevista:
Escalable, un XML, tint. Contrasta con PNG por densidad. Compose ImageVector equivalente.

Concepto:
Gradle es el sistema de build que orquesta compilación, empaquetado, dependencias y variantes de proyectos Android. En Android Studio, AGP (Android Gradle Plugin) añade tasks para compilar Kotlin/Java, generar R, dex, firmar APK/AAB y ejecutar tests.

Explicación detallada:
Estructura: settings.gradle.kts incluye módulos; build.gradle.kts por módulo define android {}, dependencies {}, plugins. Version catalog libs.versions.toml centraliza versiones.

Tasks: assembleDebug, bundleRelease, testDebugUnitTest. Configuraciones implementation vs api vs kapt/ksp. Build cache y configuration cache aceleran CI.

Kotlin DSL (build.gradle.kts) es estándar frente a Groovy. plugins { alias(libs.plugins.android.application) }.

Depuración build: --stacktrace, Build Analyzer, dependency insight. BOM alinea versiones (Compose, Firebase).

En el contexto de proyectos Android reales, combina esta idea con las guías oficiales de Android Developers y con métricas de calidad (crashes, ANR, Vitals) cuando aplique. En code review, pregunta siempre por ciclo de vida, testabilidad y si la solución escala cuando el equipo crece.

Durante la entrevista, estructura la respuesta en definición, ejemplo concreto de tu experiencia o proyecto demo, y trade-offs frente a alternativas. Mencionar versiones recientes de API o biblioteca (Material3, Paging 3, Hilt 2.x) demuestra que sigues el ecosistema, sin necesidad de memorizar números de versión exactos.

Para profundizar tras la entrevista, revisa documentación oficial, codelabs y el código fuente de tu propio proyecto Entrevista aplicando el concepto en un flujo mínimo reproducible. La práctica deliberada con tests unitarios refuerza la confianza al explicar el tema en viva voz.

Cuándo usarlo:
Cualquier cambio de dependencia, minSdk, signing, flavors. Entender Gradle es esencial para entrevista mid+.

Ejemplo:
// build.gradle.kts (app)
plugins {
    alias(libs.plugins.android.application)
    alias(libs.plugins.kotlin.android)
}
android {
    namespace = "com.example.entrevista"
    compileSdk = 35
    defaultConfig { minSdk = 24; targetSdk = 35 }
}
dependencies {
    implementation(libs.androidx.core.ktx)
}

Consejo para entrevista:
AGP, módulos, variantes, implementation vs api. Menciona catalog y kts. No confundas Gradle con Git.

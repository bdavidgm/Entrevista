Concepto:
DownloadManager es un servicio del sistema que gestiona descargas HTTP en segundo plano con notificación de progreso, reintentos y persistencia, accesible vía getSystemService(DOWNLOAD_SERVICE).

Explicación detallada:
Encolas descargas con Request(uri):
- setDestinationUri / setDestinationInExternalPublicDir (legacy) / setDestinationInExternalFilesDir
- setNotificationVisibility (VISIBLE, HIDDEN si no quieres notificación pública)
- setAllowedNetworkTypes (WIFI, MOBILE)
- setRequiresCharging, setRequiresDeviceIdle (API 24+)
- addRequestHeader para auth tokens (cuidado: no logs sensibles)

Devuelve downloadId (long). Consultas estado con query(Query().setFilterById(id)): COLUMN_STATUS (SUCCESSFUL, FAILED, PENDING), bytes descargados, reason failure.

Ventajas: sobrevive cierre de app, integración con notificaciones del sistema, manejo de red pausada.

Limitaciones: solo HTTP(S) simple; poca flexibilidad (no GraphQL streaming complejo); menos control que OkHttp + WorkManager; políticas de almacenamiento en scoped storage.

Alternativa moderna: WorkManager + OkHttp + ForegroundService para descargas grandes con lógica custom; o DownloadManager para archivos públicos simples (PDF, APK interno).

Cuándo usarlo:
- Descargar PDF, imagen, actualización OTA sin lógica compleja.
- Cuando quieres barra de progreso en shade sin implementar notificación propia.

Evitar: APIs autenticadas complejas, parsing en stream, múltiples formatos con transformación.

Ejemplo:
val dm = context.getSystemService(DownloadManager::class.java)
val request = DownloadManager.Request(Uri.parse(url))
    .setTitle("Manual")
    .setDescription("Descargando...")
    .setNotificationVisibility(
        DownloadManager.Request.VISIBILITY_VISIBLE_NOTIFY_COMPLETED
    )
    .setDestinationInExternalFilesDir(
        context, Environment.DIRECTORY_DOWNLOADS, "manual.pdf"
    )
val id = dm.enqueue(request)

Consejo para entrevista:
Contrasta con WorkManager + Retrofit/OkHttp. Menciona consulta por Cursor y permisos de almacenamiento según destino. Para entrevista senior: “DownloadManager para ficheros grandes simples; cola propia si necesitas cancelación fina o headers dinámicos”.

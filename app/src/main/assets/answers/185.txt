Concepto:
SplashScreen API (Android 12+, androidx.core:core-splashscreen) estandariza la pantalla de lanzamiento: icono de app, animación opcional y transición al contenido, sustituyendo temas splash ad-hoc y reduciendo parpadeos.

Explicación detallada:
Antes cada app usaba windowBackground con drawable en tema launchTheme; podía haber doble splash o flash blanco. Desde Android 12 el sistema muestra splash obligatorio con el icono adaptativo. La biblioteca androidx unifica comportamiento en versiones anteriores.

Pasos de integración:
1) Dependencia core-splashscreen
2) Tema Theme.SplashScreen (windowSplashScreenBackground, animatedIcon, postSplashScreenTheme)
3) En Activity.onCreate, antes de setContent: installSplashScreen()
4) Opcional: setKeepOnScreenCondition { viewModel.isLoading } para retener hasta datos listos
5) setOnExitAnimationListener para animación personalizada al salir

El splash del sistema no debe usarse como pantalla de marketing prolongada; Google desaconseja retardar más de lo necesario. Carga pesada en Application o Startup, no en splash artificial.

Compatibilidad: la lib replica splash en API < 31 con el mismo API installSplashScreen().

Cuándo usarlo:
- Toda app target SDK 31+ en Play debería migrar.
- Reemplazar launch screens custom que duplican el del sistema.
- Coordinar primera pintura de Compose/Views con condición de keep.

Ejemplo:
// themes.xml
<style name="Theme.App.Starting" parent="Theme.SplashScreen">
    <item name="windowSplashScreenBackground">@color/splash_bg</item>
    <item name="windowSplashScreenAnimatedIcon">@drawable/ic_launcher_foreground</item>
    <item name="postSplashScreenTheme">@style/Theme.App</item>
</style>

// MainActivity
override fun onCreate(savedInstanceState: Bundle?) {
    val splash = installSplashScreen()
    super.onCreate(savedInstanceState)
    splash.setKeepOnScreenCondition { viewModel.state.value is Loading }
    setContent { App() }
}

Consejo para entrevista:
Di installSplashScreen(), postSplashScreenTheme, y que no es lugar para anuncios. Menciona migración desde windowBackground legacy y alineación con políticas de Play sobre tiempo de inicio.

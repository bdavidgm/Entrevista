## Concepto
En Compose, el patrón [ViewModel](pregunta:68) mantiene el estado y la lógica fuera de las funciones [Composable](pregunta:46). El [ViewModel](pregunta:68) expone un [StateFlow](pregunta:86)<UiState> y métodos para procesar [intents](pregunta:33); la pantalla colecta el estado con [collectAsStateWithLifecycle](pregunta:134) y delega los eventos en métodos del [ViewModel](pregunta:68).

Esto permite que la UI sea declarativa y pura (UI = f(state)), mientras el [ViewModel](pregunta:68) sobrevive a la rotación y centraliza el side effect / [coroutine](pregunta:78) management.

Ejemplo:
@HiltViewModel
class HomeViewModel @Inject constructor(
    private val repo: HomeRepository
) : ViewModel() {
    val state: StateFlow<HomeUiState> = repo.observe()
        .map(HomeUiState::Success)
        .stateIn(viewModelScope, SharingStarted.WhileSubscribed(5_000), HomeUiState.Loading)
}

@Composable
fun HomeScreen(vm: HomeViewModel = hiltViewModel()) {
    val state by vm.state.collectAsStateWithLifecycle()
    HomeContent(state)
}

ExplicaciónKotlin:
[@HiltViewModel](pregunta:141) class HomeViewModel @Inject constructor(private val repo: HomeRepository) : [ViewModel](pregunta:68)() sigue el patrón DI estándar. repo.observe().map(HomeUiState::Success) encadena [Flow](pregunta:84) con map y method reference HomeUiState::Success, equivalente a { HomeUiState.Success(it) } si Success es constructor o función de un solo argumento. stateIn([viewModelScope](pregunta:131), SharingStarted.WhileSubscribed(5_000), HomeUiState.Loading) convierte a [StateFlow](pregunta:86) con valor inicial Loading. HomeScreen(vm: HomeViewModel = [hiltViewModel](pregunta:141)()) permite inyectar o usar default. val state by vm.state.[collectAsStateWithLifecycle](pregunta:134)() observa el [StateFlow](pregunta:86). HomeContent(state) es [composable](pregunta:46) puro que recibe el estado como parámetro, separando lógica de presentación. El genérico HomeUiState aparece en [StateFlow](pregunta:86) y en el valor inicial de stateIn.

Consejo:
"State down, events up". Usa [collectAsStateWithLifecycle](pregunta:134) para colectar con conciencia del ciclo de vida; evita lógica en el [Composable](pregunta:46).

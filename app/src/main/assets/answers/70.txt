Concepto:
Room es la capa de persistencia oficial de Android sobre SQLite: abstrae SQL con DAOs, entidades y compilación en tiempo de build que valida queries. Ofrece integración con Flow, LiveData, coroutines suspend y migraciones tipadas.

Explicación detallada:
@Database declara entidades y versión. @Dao define @Insert, @Update, @Delete, @Query con SQL. Room genera implementación en compile-time (KSP/KAPT). @TypeConverter mapea tipos no soportados (Date, List<String>). Las relaciones complejas usan @Relation, @Embedded o JOIN en Query.

Migraciones: Migration(1,2) { db.execSQL(...) } o AutoMigration en versiones recientes con reglas exportadas. Nunca version++ sin migración en producción o pierdes datos.

Room con coroutines: suspend fun insert; Flow<List<User>> observa cambios reactivos. Transacciones con @Transaction en funciones que llaman varios DAO.

Testing: inMemoryDatabaseBuilder para tests rápidos. No uses la misma instancia singleton entre tests sin cerrar.

Comparación DataStore: Room para datos estructurados relacionales; DataStore para preferencias clave-valor. Paging + Room es patrón común para listas grandes.

En el contexto de proyectos Android reales, combina esta idea con las guías oficiales de Android Developers y con métricas de calidad (crashes, ANR, Vitals) cuando aplique. En code review, pregunta siempre por ciclo de vida, testabilidad y si la solución escala cuando el equipo crece.

Durante la entrevista, estructura la respuesta en definición, ejemplo concreto de tu experiencia o proyecto demo, y trade-offs frente a alternativas. Mencionar versiones recientes de API o biblioteca (Material3, Paging 3, Hilt 2.x) demuestra que sigues el ecosistema, sin necesidad de memorizar números de versión exactos.

Cuándo usarlo:
Cualquier dato local estructurado, caché offline, historial. No para un solo flag booleano (DataStore). No para archivos grandes (almacenamiento de archivos).

Ejemplo:
@Entity(tableName = "products")
data class ProductEntity(@PrimaryKey val id: Int, val name: String)

@Dao
interface ProductDao {
    @Query("SELECT * FROM products")
    fun observeAll(): Flow<List<ProductEntity>>

    @Insert(onConflict = OnConflictStrategy.REPLACE)
    suspend fun insertAll(items: List<ProductEntity>)
}

@Database(entities = [ProductEntity::class], version = 1)
abstract class AppDatabase : RoomDatabase() {
    abstract fun productDao(): ProductDao
}

Consejo para entrevista:
Menciona DAO, migraciones, Flow, y diferencia con SQLite crudo. Pregunta: ¿Room en main thread? Por defecto no con suspend/Flow bien usados; allowMainThreadQueries solo tests.

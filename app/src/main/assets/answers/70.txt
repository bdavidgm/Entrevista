Concepto:
Room es la capa de persistencia oficial de Jetpack sobre SQLite. Genera el código de acceso a partir de anotaciones (@Entity, @Dao, @Database) y aporta verificación de queries SQL en tiempo de compilación, soporte para coroutines/Flow y migraciones explícitas.

Sus tres bloques son: Entity (tabla), DAO (operaciones tipadas) y Database (singleton que las une y configura las migraciones).

Ejemplo:
@Entity
data class User(@PrimaryKey val id: String, val name: String)

@Dao
interface UserDao {
    @Query("SELECT * FROM User") fun all(): Flow<List<User>>
    @Insert(onConflict = OnConflictStrategy.REPLACE) suspend fun upsert(user: User)
}

@Database(entities = [User::class], version = 1)
abstract class AppDb : RoomDatabase() { abstract fun userDao(): UserDao }

Consejo:
"Queries validadas en compilación + Flow reactivo". Para producción cuida las migraciones; nunca uses fallbackToDestructiveMigration en release.

## Concepto
El modo oscuro (dark theme) es una variante de tema UI con fondos oscuros y texto claro que reduce fatiga visual en poca luz, puede ahorrar batería en pantallas OLED y respeta la preferencia del sistema o del usuario en la app.

## Explicación detallada
Android 10+ soporta forceDark y configuración uiMode UI_MODE_NIGHT_YES. Material3: darkColorScheme() vs lightColorScheme(). En Compose: isSystemInDarkTheme() o estado guardado en DataStore para override manual (Sistema / Claro / Oscuro).

themes.xml: Theme.Material3.DayNight sin parent fijo light. values-night/ para recursos alternativos. Status bar y navigation bar icons contrast con WindowInsetsController.

Evita imágenes solo para light sin variante. Prueba contraste accesibilidad en ambos modos. dynamic color adapta paleta en ambos.

## Cuándo usarlo
Toda app moderna debe soportar dark o DayNight. Ofrece toggle en ajustes si producto lo requiere además del sistema.

Ejemplo:
@Composable
fun EntrevistaTheme(
    darkTheme: Boolean = isSystemInDarkTheme(),
    content: @Composable () -> Unit
) {
    val colors = if (darkTheme) darkColorScheme() else lightColorScheme()
    MaterialTheme(colorScheme = colors, content = content)
}


ExplicaciónKotlin:
`@Composable fun EntrevistaTheme(darkTheme: Boolean = isSystemInDarkTheme(), content: @Composable () -> Unit)` define parámetro `darkTheme` con valor por defecto evaluado en call site; permite override manual además del sistema. `val colors = if (darkTheme) darkColorScheme() else lightColorScheme()` selecciona paleta Material3. `MaterialTheme(colorScheme = colors, content = content)` aplica tema con un solo parámetro explícito además de content. Parámetros con default en Kotlin permiten llamar `EntrevistaTheme { Screen() }` omitiendo `darkTheme`. `@Composable () -> Unit` es tipo funcional especializado del compilador Compose. La función es pure UI setup sin side effects.

Consejo para entrevista:
DayNight, darkColorScheme, preferencia usuario + sistema. Menciona OLED battery. No hardcodear colores fuera de tokens.

Concepto:
Cloud Firestore es una base de datos de documentos en la nube en tiempo real: colecciones, documentos, consultas indexadas y listeners que emiten cambios. Las reglas de seguridad controlan lectura/escritura por usuario.

Alternativa a Realtime Database con modelo de datos más flexible.

Ejemplo:
Firebase.firestore.collection("users").document(uid).get()
    .addOnSuccessListener { doc -> println(doc.getString("name")) }

ExplicaciónKotlin:
`Firebase.firestore` es entry point KTX al cliente Firestore. La cadena `.collection("users").document(uid).get()` construye referencia a documento y lanza lectura asíncrona; `uid` es variable Kotlin del ámbito (probablemente `String`). `.addOnSuccessListener { doc -> ... }` registra callback con lambda: `doc` es `DocumentSnapshot` inferido. `println(doc.getString("name"))` usa función estándar Kotlin `println` y getter Java que devuelve `String?` — en producción conviene safe call `doc.getString("name") ?: "Anónimo"`. No hay listener de error ni corrutinas; alternativa idiomática: `Firebase.firestore...get().await()` dentro de `viewModelScope.launch`. El encadenamiento fluido sobre referencias Firestore es estilo Java/KTX compatible con Kotlin sin wrappers adicionales.

Consejo:
Diseña el esquema para consultas (índices compuestos). Offline persistence está habilitada por defecto en móvil; vigila tamaño de cache.

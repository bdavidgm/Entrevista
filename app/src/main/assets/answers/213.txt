Concepto:
Apollo Kotlin (antes Apollo Android) es un cliente GraphQL tipado que genera modelos y operaciones desde el schema, ejecutando queries/mutations/subscriptions sobre HTTP o WebSocket.

Explicación detallada:
GraphQL: el cliente pide exactamente los campos necesarios en una sola petición; el servidor expone un schema con tipos y resolvers.

Apollo:
- Define .graphql queries en proyecto
- Gradle plugin genera clases Kotlin type-safe
- ApolloClient con OkHttp, cache normalizado opcional
- Coroutines: query.execute() o Flow para watch queries
- Subscriptions para tiempo real (WebSocket)

Ventajas vs REST: menos over/under-fetching, evolución de API con deprecations en schema, una versión para móvil/web si comparten schema.

Desventajas: caching más complejo que REST; curva de schema; tooling de archivo único.

Cuándo usarlo:
- Backend GraphQL (Hasura, Apollo Server, DGS).
- Pantallas que necesitan combinar muchos recursos sin N+1 REST calls.

Ejemplo:
// GetUserQuery.graphql
// query GetUser($id: ID!) { user(id: $id) { id name email } }

val apollo = ApolloClient.Builder()
    .serverUrl("https://api.example.com/graphql")
    .build()

suspend fun loadUser(id: String): GetUserQuery.Data? {
    val response = apollo.query(GetUserQuery(id)).execute()
    if (response.hasErrors()) throw ApolloException(response.errors)
    return response.data
}

Consejo para entrevista:
Contrasta GraphQL vs REST (flexibilidad vs simplicidad caché HTTP). Menciona codegen, normalized cache y que no reemplaza Firestore. Si solo conoces Retrofit, admite Apollo cuando el backend es GraphQL-first.

Concepto:
La null safety es la característica de Kotlin que distingue en el sistema de tipos las referencias anulables (String?) de las no anulables (String). El compilador exige comprobar o convertir antes de usar un valor anulable, convirtiendo muchas NullPointerException de runtime en errores de compilación.

Operadores clave: ?. (safe call), ?: (Elvis), !! (force, peligroso) y requireNotNull. Tras un if (x != null), el smart cast trata x como no anulable dentro del bloque. Los tipos provenientes de Java son platform types (T!), donde el compilador no conoce la nullabilidad.

Ejemplo:
fun showEmail(user: User?) {
    val email = user?.email ?: "Sin correo"
    textView.text = email
}

Consejo:
Evita !! salvo invariantes demostrables; prefiere ?., ?: o early return. Menciona los platform types como punto débil al interoperar con Java.

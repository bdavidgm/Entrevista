## Concepto
Encapsular significa ocultar el estado interno y permitir cambios solo mediante operaciones válidas. Kotlin ofrece public (por defecto), internal (mismo módulo), protected (clase y subclases) y private (clase o archivo, según dónde se declare).

Ejemplo:
class BankAccount(initialBalance: Long) {
    var balance: Long = initialBalance
        private set

    fun deposit(amount: Long) {
        require(amount > 0)
        balance += amount
    }
}

ExplicaciónKotlin:
`class BankAccount(initialBalance: Long)` recibe parámetro de constructor no expuesto como propiedad pública; solo existe durante inicialización salvo que lo promuevas a `val`/`var`. `var balance: Long = initialBalance` crea propiedad mutable con backing field. `private set` genera setter privado y getter público: lectura externa permitida, escritura solo dentro de la clase. `fun deposit(amount: Long)` es método público que valida con `require(amount > 0)` antes de `balance += amount`. `require` es función inline estándar Kotlin para precondiciones. Encapsulación en Kotlin se logra con modificadores en propiedades, no solo métodos getter/setter verbosos de Java. `Long` evita problemas de coma flotante en dinero (mejor aún sería un [value class](pregunta:29) o `BigDecimal` según dominio). Nadie fuera de `BankAccount` puede hacer `balance = -100` gracias a `private set`. Este patrón es idiomático para DTOs y modelos de dominio en Android.

Consejo:
Expón la menor visibilidad necesaria. private set permite lectura pública sin permitir modificaciones arbitrarias.

## Concepto
suspend es un modificador de Kotlin que marca una función que puede pausar y reanudar de forma segura dentro de una [coroutine](pregunta:78), sin bloquear el hilo subyacente. Solo puede llamarse desde otra función suspend o desde un [coroutine](pregunta:78) builder como launch.

## Explicación detallada
El compilador transforma suspend en máquina de estados (continuations). Cuando la función espera IO, libera el hilo para otras [coroutines](pregunta:78). [Retrofit](pregunta:207) con suspend fun, [Room](pregunta:70) con suspend DAO, y delay() son ejemplos idiomáticos.

Llamar suspend desde código no suspend requiere launch o [runBlocking](pregunta:142) (tests). Encadenar varias suspend es código lineal legible frente a callbacks hell.

suspendCancellableCoroutine convierte callbacks legacy a suspend. withTimeout y [withContext](pregunta:82) son suspend estándar.

Cancellation: las suspend deben ser cooperativas; revisar isActive o usar funciones cancelables de kotlinx.[coroutines](pregunta:78).

## Cuándo usarlo
APIs que esperan resultado único async. Para streams usar [Flow](pregunta:84), no suspend repetido en loop manual.

Ejemplo:
interface ApiService {
    @GET("users/{id}")
    suspend fun getUser(@Path("id") id: Long): UserDto
}

suspend fun loadUser(id: Long): User = withContext(Dispatchers.IO) {
    api.getUser(id).toDomain()
}

ExplicaciónKotlin:
interface ApiService declara contrato [Retrofit](pregunta:207). @GET("users/{id}") suspend fun getUser(@Path("id") id: Long): UserDto anota suspend fun: [Retrofit](pregunta:207) genera implementación que pausa [coroutine](pregunta:78) en red. @Path("id") vincula parámetro id: Long Long no nullable. suspend fun loadUser(id: Long): User = [withContext](pregunta:82)([Dispatchers.IO](pregunta:30)) { api.getUser(id).toDomain() } usa expresión body: [withContext](pregunta:82) cambia a IO, api.getUser(id) llama suspend de [Retrofit](pregunta:207), .toDomain() extensión convierte DTO a dominio User. El modificador suspend en loadUser permite llamarla solo desde [coroutine](pregunta:78); el compilador transforma en continuación. Contrasta función suspend de interfaz (generada) con función suspend propia que orquesta contexto.

Consejo para entrevista:
Define suspend como pausable, no como async automático en otro hilo (el dispatcher decide). Pregunta: ¿puedo llamar suspend desde onClick directo? No, necesitas scope.launch.

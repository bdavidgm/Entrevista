Concepto:
CoroutineScope define el ámbito y ciclo de vida de las coroutines hijas: agrupa jobs y aplica structured concurrency (cancelación en cascada). En Android, scopes típicos son viewModelScope, lifecycleScope y applicationScope.

Explicación detallada:
Un scope = CoroutineContext (Job + Dispatcher + opcional name). Al cancelar el Job del scope, todas las coroutines hijas reciben cancelación cooperativa. viewModelScope usa SupervisorJob en implementación que permite un hijo fallar sin cancelar hermanos según configuración.

lifecycleScope en Activity/Fragment cancela al destruir DESTROYED. repeatOnLifecycle(Lifecycle.State.STARTED) para collect Flow solo cuando visible.

No crear scopes globales sin Job vinculado a lifecycle. CoroutineScope(Dispatchers.Main + job) manual requiere cancelar en onDestroy.

Testing: TestScope y runTest controlan tiempo virtual. backgroundScope en runTest para trabajo async.

En el contexto de proyectos Android reales, combina esta idea con las guías oficiales de Android Developers y con métricas de calidad (crashes, ANR, Vitals) cuando aplique. En code review, pregunta siempre por ciclo de vida, testabilidad y si la solución escala cuando el equipo crece.

Durante la entrevista, estructura la respuesta en definición, ejemplo concreto de tu experiencia o proyecto demo, y trade-offs frente a alternativas. Mencionar versiones recientes de API o biblioteca (Material3, Paging 3, Hilt 2.x) demuestra que sigues el ecosistema, sin necesidad de memorizar números de versión exactos.

Cuándo usarlo:
Siempre que uses launch/async; nunca GlobalScope en features de app. Elige scope según duración del trabajo: pantalla vs app vs test.

Ejemplo:
class MyViewModel : ViewModel() {
    fun save() = viewModelScope.launch {
        repository.save()
    }
}

// Activity
lifecycleScope.launch {
    repeatOnLifecycle(Lifecycle.State.STARTED) {
        viewModel.uiState.collect { render(it) }
    }
}

Consejo para entrevista:
Lista scopes Android y cuándo se cancelan. Pregunta viewModelScope vs lifecycleScope: ViewModel sobrevive rotación; lifecycleScope no para la misma Activity recreada igual... ViewModel persiste datos, lifecycleScope recolecta UI.

Concepto:
App Startup es una librería de Jetpack que permite inicializar componentes al arrancar la app de forma ordenada y sin tener que crear múltiples ContentProvider de inicialización. Defines clases Initializer<T> que declaran sus dependencias y App Startup las ejecuta en el orden adecuado.

Reduce el tiempo de arranque consolidando inicializaciones de librerías y dando control sobre cuáles diferir.

Ejemplo:
class LoggerInitializer : Initializer<Logger> {
    override fun create(context: Context): Logger {
        return Logger().also { Timber.plant(it) }
    }
    override fun dependencies(): List<Class<out Initializer<*>>> = emptyList()
}

<provider
    android:name="androidx.startup.InitializationProvider"
    android:authorities="${applicationId}.androidx-startup"
    android:exported="false"
    tools:node="merge">
    <meta-data android:name="com.example.LoggerInitializer" android:value="androidx.startup"/>
</provider>

Consejo:
"Inicializa muchas libs en un único provider con orden". Útil para mejorar el cold start y para diferir inicializaciones pesadas.

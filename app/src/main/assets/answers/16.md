## Concepto
lateinit y by lazy son dos formas de retrasar la inicialización de una propiedad. lateinit se aplica a var de tipo no anulable y se inicializa manualmente antes de usarse; lanza UninitializedPropertyAccessException si no fue inicializada. by lazy se aplica a val y delega en una función que se ejecuta de forma thread-safe la primera vez que se accede.

Usa lateinit cuando el valor se inyecta desde fuera (DI, onCreate); usa by lazy cuando el valor depende solo de la propia clase y se calcula una vez.

Ejemplo:
class MyActivity : AppCompatActivity() {
    private lateinit var binding: ActivityMainBinding
    private val viewModel: MainViewModel by viewModels()
    private val analytics by lazy { Analytics.create(this) }

    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        binding = ActivityMainBinding.inflate(layoutInflater)
    }
}

ExplicaciónKotlin:
`class MyActivity : AppCompatActivity()` hereda de la Activity de AndroidX; los miembros usan visibilidad `private`. `lateinit var binding: ActivityMainBinding` declara una propiedad `var` no anulable cuya inicialización se retrasa: el compilador confía en que se asignará antes del primer uso; acceder sin inicializar lanza `UninitializedPropertyAccessException`. No admite tipos primitivos ni anulables. `private val viewModel: MainViewModel by viewModels()` es una propiedad delegada: el delegado `viewModels()` (extensión de AndroidX) crea/recupera el ViewModel ligado al ciclo de vida; `by` delega get/set al objeto delegate. `private val analytics by lazy { Analytics.create(this) }` usa el delegado estándar `lazy`: la lambda se ejecuta una vez, de forma thread-safe, al primer acceso; `this` dentro es la Activity. En `onCreate`, `override fun` sobrescribe el callback de la superclase; `super.onCreate(...)` delega la parte base antes de `binding = ActivityMainBinding.inflate(layoutInflater)`, que completa la inicialización de `lateinit`.

Consejo:
"lateinit = var asignada por fuera; lazy = val calculada al primer uso". Para tipos primitivos usa lazy o Delegates.notNull, no lateinit.

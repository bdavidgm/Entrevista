Concepto:
AudioManager gestiona el flujo de audio del dispositivo: volumen por stream (música, ring, alarma), modo (normal, vibración, silencio), audio focus y rutas (altavoz, auricular, Bluetooth).

Explicación detallada:
Streams principales (STREAM_*):
- STREAM_MUSIC: media, juegos.
- STREAM_RING / STREAM_NOTIFICATION / STREAM_ALARM.
- STREAM_VOICE_CALL: llamadas.

Métodos:
- setStreamVolume / getStreamVolume / getMaxVolume
- adjustStreamVolume(ADJUST_RAISE/LOWER, flags)
- getRingerMode / setRingerMode (puede estar restringido)
- requestAudioFocus / abandonAudioFocus (OnAudioFocusChangeListener): pausar cuando otra app reproduce o hay llamada.

Modo comunicación: MODE_IN_COMMUNICATION para VoIP; setSpeakerphoneOn; routing Bluetooth SCO para headsets.

MediaPlayer/ExoPlayer integran audio focus automáticamente si configuras AudioAttributes (USAGE_MEDIA, CONTENT_TYPE_MUSIC).

Do Not Disturb: NotificationManager.Policy puede afectar si suena notificación.

Cuándo usarlo:
- App de música/podcast: focus y pausa en llamadas entrantes.
- Ajustar volumen solo de tu stream con UI custom (raro).
- VoIP: gestión de modo y altavoz.

Evitar: manipular ringer global sin permiso o sin UX clara; ignorar audio focus (dos apps sonando).

Ejemplo:
val am = context.getSystemService(AudioManager::class.java)
val focusRequest = AudioFocusRequest.Builder(AudioManager.AUDIOFOCUS_GAIN)
    .setAudioAttributes(
        AudioAttributes.Builder()
            .setUsage(AudioAttributes.USAGE_MEDIA)
            .setContentType(AudioAttributes.CONTENT_TYPE_MUSIC)
            .build()
    )
    .setOnAudioFocusChangeListener { change ->
        when (change) {
            AudioManager.AUDIOFOCUS_LOSS -> player.pause()
            AudioManager.AUDIOFOCUS_LOSS_TRANSIENT -> player.pause()
            AudioManager.AUDIOFOCUS_GAIN -> player.play()
        }
    }
    .build()
am.requestAudioFocus(focusRequest)

Consejo para entrevista:
Explica audio focus como concepto central, streams distintos, y ExoPlayer para reproducción real. Menciona que no necesitas AudioManager para sonidos UI cortos (SoundPool). Conecta con foreground service tipo mediaPlayback.

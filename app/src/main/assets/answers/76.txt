Concepto:
Paging 3 es la librería de Jetpack para cargar y mostrar grandes conjuntos de datos por páginas de forma eficiente, con placeholders, estados de carga/error y integración con RecyclerView, Compose LazyColumn y fuentes Room o red.

Explicación detallada:
PagingSource load(params) devuelve LoadResult.Page con datos y keys para siguiente/anterior. Pager construye Flow<PagingData<T>>. En UI, collect con collectLatest y LazyPagingItems en Compose o submitData en Adapter.

RemoteMediator sincroniza red + Room: REFRESH carga API y guarda en BD; Paging lee solo de Room. Separación clara offline-first.

LoadState: Loading, Error, NotLoading para footer y retry. prefetchDistance y pageSize tunan rendimiento. header/footer con LoadStateAdapter.

Evita listas gigantes en memoria: solo páginas visibles más prefetch. Invalida PagingSource tras filtros cambiando Pager o invalidate().

Cuándo usarlo:
Feeds infinitos, catálogos, historial de mensajes. No para listas de 10 elementos fijos en memoria.

Ejemplo:
@Composable
fun ProductList(pager: Flow<PagingData<Product>>) {
    val items = pager.collectAsLazyPagingItems()
    LazyColumn {
        items(items.itemCount) { index ->
            items[index]?.let { ProductRow(it) }
        }
        when (items.loadState.append) {
            is LoadState.Loading -> item { CircularProgressIndicator() }
            else -> Unit
        }
    }
}

Concepto:
Paging 3 es la librería de Jetpack para cargar y mostrar listas grandes en páginas a medida que el usuario hace scroll, evitando cargar todo en memoria. Funciona con red, base de datos o ambas (RemoteMediator para cache offline-first) y se integra con coroutines, Flow, RecyclerView y LazyColumn.

Sus piezas: PagingSource (cómo cargar una página), Pager (configuración) y PagingData (stream observable).

Ejemplo:
class UsersPagingSource(private val api: Api) : PagingSource<Int, User>() {
    override suspend fun load(params: LoadParams<Int>): LoadResult<Int, User> {
        val page = params.key ?: 1
        val users = api.users(page, params.loadSize)
        return LoadResult.Page(users, prevKey = if (page == 1) null else page - 1, nextKey = users.takeIf { it.isNotEmpty() }?.let { page + 1 })
    }
    override fun getRefreshKey(state: PagingState<Int, User>): Int? = state.anchorPosition?.let { state.closestPageToPosition(it)?.prevKey?.plus(1) }
}

val pager = Pager(PagingConfig(pageSize = 20)) { UsersPagingSource(api) }.flow

// Compose
val items = vm.pager.collectAsLazyPagingItems()
LazyColumn { items(items.itemCount) { items[it]?.let { UserRow(it) } } }

Consejo:
"Carga incremental + estados de carga". Para offline-first combina con RemoteMediator y Room.

## Concepto
Una clase es el molde que define propiedades y funciones; un objeto es una instancia concreta creada a partir de esa clase. Cada instancia mantiene su propio estado. La declaración object, en cambio, crea directamente una única instancia singleton.

Ejemplo:
class User(val id: String, var name: String) {
    fun rename(newName: String) {
        name = newName
    }
}

val ana = User("1", "Ana")
val leo = User("2", "Leo")
ana.rename("Ana María")

ExplicaciónKotlin:
`class User(val id: String, var name: String)` define constructor primario con dos propiedades: `id` inmutable (`val`) y `name` mutable (`var`). `fun rename(newName: String)` es método de instancia que asigna `name = newName`, permitido por `var`. `val ana = User("1", "Ana")` y `val leo = User("2", "Leo")` crean dos instancias distintas en memoria; cada una tiene su propio estado. `ana.rename("Ana María")` muta solo el objeto `ana`, no `leo`. Kotlin distingue la palabra clave `class` (molde) de `object` (singleton, no usado aquí). No hay `new` en Kotlin: instanciación con llamada a constructor como función. Si `User` fuera `data class`, obtendrías `copy`, `equals` y `hashCode` automáticos. El ejemplo aclara que múltiples `val` pueden referenciar objetos diferentes aunque compartan tipo `User`.

Consejo:
Clase = definición; objeto = instancia. Usa object solo cuando realmente deba existir una única instancia.

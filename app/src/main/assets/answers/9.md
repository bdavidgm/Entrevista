## Concepto
Las extension functions permiten añadir nuevos métodos a clases existentes sin modificarlas ni heredarlas, declarándolos con la sintaxis fun Tipo.nombre(...). Bajo el capó se compilan como funciones estáticas que reciben el receptor como primer parámetro.

Como son estáticas, no son polimórficas: se resuelven por el tipo declarado, no por el tipo en runtime. Si hay un método miembro con la misma firma, el miembro gana.

Ejemplo:
fun String.toSlug(): String =
    trim().lowercase().replace(" ", "-")

fun Int.dpToPx(context: Context): Int =
    (this * context.resources.displayMetrics.density).toInt()

"Hola Mundo".toSlug() // "hola-mundo"

ExplicaciónKotlin:
`fun String.toSlug(): String` declara una extension function: bajo el capó es una función estática que recibe el receptor `String` como primer parámetro implícito (`this`). Encadena llamadas a `trim()`, `lowercase()` y `replace(...)` sobre el receptor. `fun Int.dpToPx(context: Context): Int` extiende `Int` (el receptor es `this`, el valor dp) y usa `context.resources.displayMetrics.density`; el cuerpo termina con `.toInt()` para convertir el `Float` resultante. `"Hola Mundo".toSlug()` invoca la extensión con sintaxis de método de instancia. Las extension functions no son polimórficas: se resuelven en compile-time según el tipo declarado del receptor, no el runtime. No modifican la clase original; solo añaden API en el namespace donde se declaran.

Consejo:
Resalta que no son polimórficas y que ayudan a construir DSLs y APIs idiomáticas. Útiles para extender SDK/Android (View.show(), [Context](pregunta:127).toast()).

## Concepto
mutableStateOf crea un State<T> observable por Compose: cada vez que se asigna a su .value, todas las composables que lo lean se invalidan y se programan para [recomposición](pregunta:47). Es la base del modelo reactivo de Compose.

Para tipos primitivos existen variantes optimizadas como mutableIntStateOf, mutableLongStateOf y mutableFloatStateOf que evitan autoboxing.

Ejemplo:
@Composable
fun NameInput() {
    var name by remember { mutableStateOf("") }
    TextField(value = name, onValueChange = { name = it })
    Text("Hola, $name")
}

ExplicaciónKotlin:
NameInput demuestra el patrón básico de estado en Compose. var name by [remember](pregunta:48) { mutableStateOf("") } inicializa un State<String> vacío; el tipo String es no nullable. TextField(value = name, onValueChange = { name = it }) enlaza el campo: value es lectura del estado actual y onValueChange recibe una lambda con parámetro it (el nuevo String), asignándolo a name mediante el setter delegado. Text("Hola, $name") recompone cuando name cambia gracias al sistema de snapshot de Compose. Las funciones composables TextField y Text provienen de Material; sus parámetros nombrados mejoran la legibilidad frente al orden posicional. El genérico implícito de mutableStateOf("") es String por inferencia del literal. Este ejemplo no usa [coroutines](pregunta:78) ni nullability explícita, pero sí lambdas, delegación de propiedad y funciones de orden superior.

Consejo:
"mutableStateOf = State observable de Compose; combínalo con [remember](pregunta:48) o [rememberSaveable](pregunta:58)". Para Int/Float, prefiere mutableIntStateOf y compañía.

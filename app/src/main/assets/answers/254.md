## Concepto
Una enum class representa un conjunto cerrado de instancias conocidas, cada una con nombre y ordinal. Puede tener propiedades, métodos e implementar interfaces. Es adecuada para estados simples sin datos distintos por caso; si cada caso necesita datos propios, suele convenir una [sealed class](pregunta:19).

Ejemplo:
enum class Priority(val level: Int) {
    LOW(1),
    MEDIUM(2),
    HIGH(3);

    fun isUrgent() = this == HIGH
}

fun color(priority: Priority) = when (priority) {
    Priority.LOW -> "gris"
    Priority.MEDIUM -> "amarillo"
    Priority.HIGH -> "rojo"
}

ExplicaciónKotlin:
`enum class Priority(val level: Int)` declara enum con constructor privado implícito que recibe `level`; cada constante `LOW(1)`, etc., pasa argumento. Punto y coma `;` separa constantes de miembros adicionales. `fun isUrgent() = this == HIGH` compara identidad enum con `==` (equals estructural). `fun color(priority: Priority) = when (priority) { ... }` usa `when` exhaustivo: compilador exige cubrir todos los casos de enum cerrado. Ramas `Priority.LOW -> "gris"` devuelven `String` sin llaves por ser expresiones. Enums son clases finales singleton por constante; no puedes instanciar fuera. `Priority.entries` (Kotlin 1.9+) reemplaza `values()` para listar sin array Java. Persistir `name` en [DataStore](pregunta:73) es más seguro que `ordinal`. Enums pueden implementar interfaces como cualquier class.

Consejo:
Usa entries para recorrer valores. No persistas ordinal: guarda name o un código estable, porque el orden puede cambiar.

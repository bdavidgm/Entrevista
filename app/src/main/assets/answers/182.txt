Concepto:
BiometricPrompt es la API unificada de AndroidX Biometric para autenticación biométrica (huella, rostro, iris) o credencial del dispositivo, con UI del sistema y callbacks de éxito/error.

Explicación detallada:
Sustituye FingerprintManager antiguo. Componentes:
- BiometricManager: canAuthenticate(authenticators) → BIOMETRIC_SUCCESS, ERROR_NONE_ENROLLED, ERROR_NO_HARDWARE, etc.
- BiometricPrompt: muestra diálogo con título, subtítulo, negative button (cancelar; no uses “Usar contraseña” como negative si rediriges a PIN — usa CryptoObject o flujo propio).

Authenticators (flags combinables):
- BIOMETRIC_STRONG / BIOMETRIC_WEAK
- DEVICE_CREDENTIAL (PIN/patrón) con setAllowedAuthenticators
- Clase 3 (strong) exigida para pagos según guía

BiometricPrompt.CryptoObject vincula cifrado (Cipher) a la autenticación para que la clave en Android Keystore solo se desbloquee tras biométrico válido.

Flujo típico app bancaria/login:
1) Comprobar canAuthenticate
2) Opcional: generar clave en Keystore con setUserAuthenticationRequired
3) Mostrar BiometricPrompt en main thread
4) onAuthenticationSucceeded → desbloquear token o navegar

Errores: ERROR_LOCKOUT, ERROR_LOCKOUT_PERMANENT (demasiados intentos), user cancel, sensor unavailable.

Cuándo usarlo:
- Login rápido, confirmación de pago, acceso a datos sensibles.
- Sustituir contraseña en cada apertura si hay biométrico enrolled.

No usar solo biométrico sin fallback si el usuario no enrolló hardware.

Ejemplo:
val promptInfo = BiometricPrompt.PromptInfo.Builder()
    .setTitle("Confirmar identidad")
    .setSubtitle("Usa huella o rostro")
    .setNegativeButtonText("Cancelar")
    .build()

val biometricPrompt = BiometricPrompt(
    activity,
    ContextCompat.getMainExecutor(activity),
    object : BiometricPrompt.AuthenticationCallback() {
        override fun onAuthenticationSucceeded(
            result: BiometricPrompt.AuthenticationResult
        ) { /* desbloquear */ }
        override fun onAuthenticationError(code: Int, errString: CharSequence) {}
    }
)
biometricPrompt.authenticate(promptInfo)

Consejo para entrevista:
Menciona androidx.biometric, diferencia weak vs strong, CryptoObject + Keystore, y que el diálogo lo dibuja el sistema (no falsificar UI de huella). Nunca almacenes contraseñas; desbloquea claves o tokens cifrados.

Concepto:
Flow es un tipo frío (cold stream) de Kotlin que emite múltiples valores de forma secuencial y asíncrona, con soporte de operadores (map, filter, combine) y coroutines. En Android reemplaza a LiveData para muchos casos reactivos con backpressure y composición.

Explicación detallada:
flow { emit(1); delay(100); emit(2) } no ejecuta hasta que un collector hace collect. Cada collector puede tener su propia ejecución (cold). StateFlow y SharedFlow son hot: mantienen valor sin collector activo.

Operadores: map, filter, debounce, distinctUntilChanged, catch, onEach, flatMapLatest (cancela emisiones previas al cambiar búsqueda). flowOn cambia dispatcher upstream.

Room observeAll(): Flow<List<Entity>> re-emite al cambiar tabla. ViewModel expone StateFlow derivado de Flow con stateIn SharingStarted.WhileSubscribed.

Lifecycle: collect en repeatOnLifecycle(STARTED) o collectAsStateWithLifecycle en Compose. Nunca collect infinito en onCreate sin lifecycle.

Testing: Turbine para assert emisiones. testScheduler para delay.

En el contexto de proyectos Android reales, combina esta idea con las guías oficiales de Android Developers y con métricas de calidad (crashes, ANR, Vitals) cuando aplique. En code review, pregunta siempre por ciclo de vida, testabilidad y si la solución escala cuando el equipo crece.

Durante la entrevista, estructura la respuesta en definición, ejemplo concreto de tu experiencia o proyecto demo, y trade-offs frente a alternativas. Mencionar versiones recientes de API o biblioteca (Material3, Paging 3, Hilt 2.x) demuestra que sigues el ecosistema, sin necesidad de memorizar números de versión exactos.

Cuándo usarlo:
Streams de datos, búsqueda con debounce, combinación de sensores, UI reactiva. Un solo valor async: suspend basta.

Ejemplo:
fun searchFlow(query: Flow<String>): Flow<List<Result>> = query
    .debounce(300)
    .distinctUntilChanged()
    .flatMapLatest { q ->
        flow {
            emit(repository.search(q))
        }
    }

viewModelScope.launch {
    searchFlow(queryState).collect { results ->
        _uiState.update { it.copy(results = results) }
    }
}

Consejo para entrevista:
Cold vs hot, flowOn, flatMapLatest para búsqueda. Contrasta con LiveData. Seguimiento: stateIn y WhileSubscribed.

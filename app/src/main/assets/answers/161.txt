Concepto:
Repository con caché expone una sola API que lee primero de fuente rápida/local (Room, memoria) y refresca con red cuando toca. Patrón típico “single source of truth”: la UI observa Flow desde Room; el repo escribe en Room tras fetch.

Mejora offline, latencia y consistencia si defines bien invalidación y conflictos.

Ejemplo:
class UserRepository(private val api: UserApi, private val dao: UserDao) {
    fun users(): Flow<List<User>> = dao.observeAll()

    suspend fun refresh() {
        val remote = api.fetchUsers().map { it.toEntity() }
        dao.replaceAll(remote)
    }
}

Consejo:
Combina con WorkManager para sync periódico y muestra banner “sin conexión” cuando sirves datos viejos. Evita dos fuentes de verdad sin estrategia.

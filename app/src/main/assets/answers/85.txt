Concepto:
StateFlow y SharedFlow son hot flows que emiten valores a múltiples collectors sin reiniciar la fuente desde cero. StateFlow mantiene siempre un valor actual (estado UI); SharedFlow es broadcast de eventos con configuración de replay y buffer.

Explicación detallada:
StateFlow requiere valor inicial; emisiones con value= o update; collectors reciben el último estado al suscribirse. Ideal para UiState en ViewModel. Comparación con LiveData: similar semántica de estado, pero API coroutines.

SharedFlow: replay=0 para eventos one-shot (navegación, toast); replay=1 si quieres último evento a nuevos suscriptores. extraBufferCapacity y onBufferOverflow DROP_OLDEST para bursts. tryEmit en productores no suspend.

MutableStateFlow expuesto como StateFlow de solo lectura al exterior. combine(StateFlow, StateFlow) para estados derivados.

SharingStarted.WhileSubscribed(5000) en stateIn cancela upstream sin observadores tras timeout, ahorrando recursos.

En el contexto de proyectos Android reales, combina esta idea con las guías oficiales de Android Developers y con métricas de calidad (crashes, ANR, Vitals) cuando aplique. En code review, pregunta siempre por ciclo de vida, testabilidad y si la solución escala cuando el equipo crece.

Durante la entrevista, estructura la respuesta en definición, ejemplo concreto de tu experiencia o proyecto demo, y trade-offs frente a alternativas. Mencionar versiones recientes de API o biblioteca (Material3, Paging 3, Hilt 2.x) demuestra que sigues el ecosistema, sin necesidad de memorizar números de versión exactos.

Cuándo usarlo:
StateFlow para estado de pantalla; SharedFlow para eventos únicos o bus de eventos entre capas con cuidado. Channel alternativa para eventos one-shot con consumo único.

Ejemplo:
private val _uiState = MutableStateFlow(HomeUiState())
val uiState: StateFlow<HomeUiState> = _uiState.asStateFlow()

private val _events = MutableSharedFlow<UiEvent>()
val events: SharedFlow<UiEvent> = _events.asSharedFlow()

fun onSaved() = viewModelScope.launch {
    _events.emit(UiEvent.ShowSnackbar("Guardado"))
}

Consejo para entrevista:
StateFlow vs SharedFlow = estado vs evento. replay y buffer. No uses SharedFlow con replay=1 para toast que se repite en rotación sin consumir.

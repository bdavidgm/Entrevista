Concepto:
Un Service "started" se arranca con startService/startForegroundService y vive hasta que él mismo se detiene (stopSelf) o lo detiene quien lo creó (stopService); ejecuta trabajo aunque el cliente desaparezca. Un Service "bound" se enlaza con bindService desde uno o varios clientes, expone un IBinder para llamarlo como API y se destruye cuando se desligan todos los clientes.

Un mismo Service puede ser ambos a la vez. El started es típico para procesos largos (reproductor, descargas); el bound, para comunicación cliente-servicio (música controlada desde UI).

Ejemplo:
// Started
startForegroundService(Intent(this, MusicService::class.java))

// Bound
val conn = object : ServiceConnection {
    override fun onServiceConnected(name: ComponentName?, binder: IBinder?) { /* ... */ }
    override fun onServiceDisconnected(name: ComponentName?) { }
}
bindService(Intent(this, MyService::class.java), conn, Context.BIND_AUTO_CREATE)

Consejo:
"Started vive por sí solo; bound vive mientras haya clientes". Conoce que startForegroundService obliga a llamar startForeground en 5 s.

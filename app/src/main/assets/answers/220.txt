Concepto:
R8 (sucesor de ProGuard en el Android Gradle Plugin) encoge, optimiza y ofusca el bytecode en release cuando minifyEnabled es true: elimina código no alcanzado, renombra símbolos y aplica optimizaciones. Las reglas en proguard-rules.pro conservan clases usadas por reflexión (Room, Retrofit, Parcelable, serialización JSON).

El mapping.txt enlaza nombres ofuscados con originales: súbelo a Play Console y Crashlytics para stack traces legibles.

Ejemplo:
android {
    buildTypes {
        release {
            isMinifyEnabled = true
            proguardFiles(
                getDefaultProguardFile("proguard-android-optimize.txt"),
                "proguard-rules.pro"
            )
        }
    }
}

ExplicaciónKotlin:
Este fragmento de `build.gradle.kts` configura el tipo de build `release` dentro del bloque `android`. `buildTypes { release { ... } }` usa una lambda con receptor de tipo `BuildType`, donde cada propiedad es parte del API del AGP. `isMinifyEnabled = true` activa R8 para shrink, optimize y obfuscate; el prefijo `is` sigue la convención JavaBean expuesta al DSL. `proguardFiles(...)` recibe varargs: primero el archivo por defecto obtenido con `getDefaultProguardFile("proguard-android-optimize.txt")`, función del Project, y segundo la ruta local `"proguard-rules.pro"`. En Kotlin, las llamadas a funciones Gradle pueden omitir paréntesis externos cuando el último argumento es una lambda, aunque aquí se usan paréntesis explícitos. Las reglas ProGuard son texto, pero su activación es puramente declarativa en el script. El mapping generado no aparece en este snippet, pero se produce en `build/outputs/mapping/release/` como efecto de `isMinifyEnabled`. Conocer esta sintaxis permite alinear reglas `-keep` con dependencias Kotlin (Room, Retrofit) sin tocar código de la app.

Consejo:
Evita -keep amplio “por si acaso”: anula el shrinker. Añade -dontwarn solo para dependencias opcionales conocidas. Prueba release builds antes de publicar.
